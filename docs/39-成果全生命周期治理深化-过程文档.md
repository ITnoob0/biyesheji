# 39-成果全生命周期治理深化-过程文档

## 1. 背景与目标

本轮在既有“成果录入中心”基础上做最小必要增强，重点把论文成果链路向“成果治理中心”推进，覆盖以下高价值能力：

- BibTeX 导入预览后的修订与重新校验
- BibTeX 导入失败原因分类细化
- 论文操作历史与版本轨迹
- 代表作批量运营
- 统计、筛选、排序、导出增强
- 论文结构化对比
- 异常数据提示与批量清洗辅助

本次坚持不重做成果模块，不改变 sessionStorage 登录态，不引入 Neo4j 强依赖，不推翻既有导入确认链路。

## 3. 关键设计决策

### 3.1 操作历史采用轻量日志，而不是完整版本快照表

原因：

- 当前最需要的是“可追溯治理动作”，而不是重型版本仓库
- 已有 `PaperOperationLog` 迁移残留可直接复用
- 以 `changed_fields + metadata_flags + snapshot` 的方式即可支撑当前阶段验收

边界：

- 当前不支持任意历史版本一键回滚
- 当前不保存全文字段 diff，只保留关键字段变更信息

### 3.2 BibTeX 纠错采用“预览后修订 + 重校验”，不改主导入链路

原因：

- 用户要求必须保留原有导入确认主链路
- 通过新增 `bibtex-revalidate` 就能实现修订闭环，而无需推翻原预览接口

边界：

- 当前仍只覆盖 BibTeX，不把 PDF 元数据解析伪装成已上线能力
- 当前不接入外部查重平台

### 3.3 批量清洗只做低风险标准化

已实现：

- DOI 统一小写并去空白
- 来源链接、卷期页、期刊级别去空白

未实现：

- 自动补 DOI
- 自动补来源链接
- 自动纠正期刊级别枚举

这样可以保证“有治理辅助”但不引入高误伤自动修改。

## 5. 最小人工验证步骤

1. 登录教师账号，进入成果管理页 `/entry`
2. 打开论文 `BibTeX 批量导入`
3. 上传包含缺题目/缺期刊/重复 DOI 的 BibTeX 文件
4. 在预览中选择异常条目，修订题目、期刊、时间等字段后点击“保存并重新校验”
5. 确认导入，观察成功/跳过/失败分类反馈
6. 导入单篇成功后，确认成果中心自动进入该论文编辑态
7. 在“论文治理面板”中：
   - 查看异常分布
   - 选中多篇论文执行“批量标记代表作”
   - 选中需要清洗的论文执行“批量标准化清洗”
   - 选择两篇论文进行结构化对比
   - 查看单篇论文操作历史
   - 点击导出 CSV

## 6. 当前边界

- 当前治理增强聚焦论文成果主链路，未对项目、知识产权、教学成果、学术服务同步做同等级治理面板
- 当前操作历史是轻量治理日志，不是全量版本回滚系统
- 当前批量清洗仅覆盖低风险标准化，不做高风险自动补录
- 当前导出为 CSV，不引入报表任务和复杂模板

## 7. 后续可扩展方向

- 论文历史版本快照与回滚
- 代表作上限规则与学院级推荐候选池
- 更多异常类型：时间异常、疑似同名不同刊、来源链接无效等
- 项目/知识产权/教学成果的同类治理面板
- 学院侧治理看板与教师侧治理动作联动
