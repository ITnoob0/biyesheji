# 84-科研画像去教学成果化与五维收口-过程文档

## 1. 背景

- 当前选题为“高校教师科研能力数字画像与智能辅助系统”。
- 原实现中同时保留了“教学成果”成果类型，并在科研画像评分里设置了“人才培养成效”维度。
- 该口径更接近“教师综合画像”，与“科研能力画像”题目存在偏移。

## 2. 本次目标

- 将教师画像从“六维综合画像”收口为“严格五维科研画像”。
- 删除教学成果对科研画像评分、解释、趋势、阶段对比和跨模块证据联动的参与。
- 下线前端教学成果录入/审核入口与画像相关展示。
- 保留数据库模型与历史数据，不做破坏性删表迁移，避免影响现有数据安全与迁移链路。

## 3. 改动范围

### 3.1 后端评分与画像口径

- 文件：
  - `backend/achievements/scoring_engine.py`
  - `backend/achievements/portrait_analysis.py`
  - `backend/achievements/views.py`
- 调整内容：
  - 删除 `talent_training` 画像维度。
  - 将科研画像权重收口为五维：
    - `academic_output`
    - `funding_support`
    - `ip_strength`
    - `academic_reputation`
    - `interdisciplinary`
  - 教学成果不再参与：
    - 综合画像评分
    - 雷达维度
    - 维度解释
    - 权重说明
    - 阶段对比
    - 趋势维度序列
  - `total_achievements` 改为仅统计论文、项目、知识产权、学术服务。
  - 画像说明与报告文案改为“严格科研画像”口径。

### 3.2 成果展示与统计

- 文件：
  - `backend/achievements/views.py`
  - `frontend/src/views/DashboardView.vue`
  - `frontend/src/views/dashboard/portrait.ts`
- 调整内容：
  - 全部成果与近期成果列表不再返回教学成果。
  - 画像首页成果概览不再暴露 `teaching_achievement_count`。
  - 近年结构图不再包含教学成果堆叠项。
  - 画像页摘要文案去除教学成果统计口径。

### 3.3 前端入口与联动

- 文件：
  - `frontend/src/views/AchievementEntryView.vue`
  - `frontend/src/views/AchievementReviewView.vue`
  - `frontend/src/utils/crossModuleLinking.ts`
  - `frontend/src/views/dashboard/PortraitDeepAnalysisPanel.vue`
- 调整内容：
  - 教学成果录入页签隐藏。
  - 成果审核页不再显示教学成果审核页签。
  - 成果到画像的跨模块映射不再将教学成果映射到画像维度。
  - 深度分析面板维度目录同步收口为五维。

### 3.4 图谱与联动说明

- 文件：
  - `frontend/src/views/AcademicGraph.vue`
  - `backend/ai_assistant/services.py`
  - `backend/project_guides/services.py`
- 调整内容：
  - 前端图谱视图不再展示教学成果节点统计。
  - 智能问答、项目推荐中的画像维度联动不再引用“人才培养/教学贡献”维度。
  - 推荐服务不再保留 `talent_training` 画像联动说明。

## 4. 兼容性处理

- 未删除 `TeachingAchievement` 模型、序列化器、路由与历史数据。
- 本次采取“产品层下线、数据层保留”的保守方案：
  - 优点：避免删表、迁移和历史数据回滚风险。
  - 代价：数据库内仍可能保留历史教学成果记录，但它们不再进入科研画像主链路。

## 5. 验证

- 执行命令：

```bash
cd backend
python manage.py test achievements.tests --keepdb --noinput
```

- 结果：
  - `Ran 51 tests`
  - `OK`

## 6. 当前结论

- 系统画像口径已从“综合画像”调整为“科研能力画像”。
- 教学成果不再作为科研画像输入，也不再参与五维科研画像评分。
- 教学成果功能本次按“前端下线 + 画像口径剔除 + 统计展示收口”处理，数据库实体暂保留。

## 7. 后续建议

- 若答辩前希望彻底做“功能级删除”，可继续进行第二步收口：
  - 删除教学成果路由注册
  - 删除教学成果审核后端入口
  - 删除图谱后端对教学成果节点/关系的生成
  - 删除学院级统计中教学成果相关字段与导出列
- 若采用当前方案，建议在论文中明确说明：
  - 系统早期存在综合成果建模能力；
  - 最终验收版本为严格科研画像口径，教学成果已退出画像主链路。
