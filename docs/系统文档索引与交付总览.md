# 系统文档索引与交付总览
最后更新时间：2026-03-22

## 1. 文档体系目标

本项目已经形成“统一现状文档 + 演示验收文档 + 多轮过程文档”的资料结构。当前整理目标不是继续堆新文档，而是确保：

- 看现状时能快速找到统一入口
- 做开发交接时能区分“当前状态”和“历史过程”
- 做答辩和论文写作时能拿到稳定口径
- 每轮增量开发后都有明确的同步更新规则

## 4. 统一维护要求

每轮功能变更完成后，至少同步检查以下资料：

- 接口变更：更新“系统接口清单-统一版”
- 模块完成度或边界变化：更新“模块边界与系统最终形态说明-统一版”
- 字段、统计或解释口径变化：更新“数据口径与风险记录-统一版”
- 新增风险、环境要求或回退说明：更新“遗留问题记录”
- 演示主路径、入口或讲解口径变化：更新“最终答辩演示步骤-第三轮”
- 每轮实现过程、验证命令和边界说明：新增对应序号的过程文档

## 5. 文档与代码的一致性说明

- 当前统一现状文档以代码现状为准，不以历史设想为准
- 历史过程文档保留阶段性语境，但不作为唯一现状依据
- 仍需坚持以下口径：
  - 不把预留能力写成已完整上线能力
  - 不把过程性尝试写成稳定交付能力
  - 不把可选 Neo4j 能力写成主链路强依赖
