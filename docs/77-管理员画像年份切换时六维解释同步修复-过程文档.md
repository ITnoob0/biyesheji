# 77-管理员画像年份切换时六维解释同步修复-过程文档

## 问题
- 系统管理员/学院管理员在“查看画像”弹窗中切换到历史年份后，六维解释区域消失或不随年份更新。

## 根因
- 前端弹窗将六维解释显示条件写成“仅当前年份显示”。
- 六维解释数据来源仍是 `/api/achievements/radar/<id>/`（实时口径），与年份切片接口 `/api/achievements/portrait/analysis/` 分离，导致切年后不同步。

## 修复方案
### 后端
- 文件：`backend/achievements/views.py`
- 在 `TeacherPortraitAnalysisView` 中新增年份同步解释输出：
  - `dimension_insights` 字段随 `year` 同步返回。
  - 快照年份使用快照维度值构建解释（保证不缺失）。
  - 运行时年份继续使用现有评分引擎解释逻辑。

### 前端
- 文件：`frontend/src/components/teacher/TeacherRadarPreviewDialog.vue`
- 取消“仅当前年显示解释”的限制。
- 删除当前年专用的 `/radar` 解释拉取逻辑。
- 改为统一使用 `/portrait/analysis/` 返回的 `dimension_insights`，与雷达图同源刷新。

### 测试补充
- 文件：`backend/achievements/test_portrait_snapshot_benchmark.py`
- 增加断言：`portrait/analysis` 在冷启动与热启动下都返回完整六维 `dimension_insights`。

## 验证结果
- `python manage.py check`：通过。
- `python manage.py test achievements.test_portrait_snapshot_benchmark --keepdb`：4/4 通过。
- `npm run type-check`：通过。
- `npm run build`：通过。
