# 46-画像快照与阶段对比-过程文档

## 1. 本次任务目标

本轮目标是在当前教师画像已具备雷达评分、维度解释、趋势展示和画像形成说明的基础上，补齐“快照口径 + 阶段对比 + 结构化变化解释”的一期最小闭环。

本次严格遵守以下边界：

- 不把“按成果年份回溯估算的趋势”包装成真正历史快照。
- 在冻结口径和版本语义未定义清楚前，不盲目新增大量快照表。
- 当前优先打通口径、接口、页面承接、导出和测试，再评估后续真正落库。

## 3. 当前基线与问题

### 3.1 现有基础

仓库当前已经具备：

- 雷达维度评分
- 维度形成说明
- 近年趋势图
- 阶段对比雏形
- 画像报告导出
- 快照边界提示

### 3.2 主要断点

当前断点主要有三类：

- “快照”只有边界说明，还没有明确的冻结口径、版本语义和生成时机表达。
- 阶段对比更多还是“分值变化”，缺少“为什么变化”的结构化说明。
- 报告和页面虽能承接阶段对比，但还没有把“当前运行时快照”和“阶段基线”区分得足够清楚。

## 4. 本次落地方案

### 4.1 快照的最小口径方案

本次没有新增快照表，而是先落地“当前运行时画像快照”的最小口径语义：

- 快照类型：
  - `runtime_snapshot_view`
- 版本号：
  - `portrait-runtime-v1`
- 版本语义：
  - 版本号描述的是“当前固定权重公式 + 运行时聚合口径”
  - 不是数据库中的正式历史存档版本
- 生成时机：
  - `dashboard_request`
  - `report_request`
- 冻结口径：
  - 只在单次接口响应 / 单次报告导出周期内保持一致
  - 不落库存储，不作为正式历史冻结快照

这意味着当前“快照”已经可以承接页面展示和报告导出，但仍然不是正式历史快照体系。

### 4.2 阶段对比能力增强

本次继续保留现有“按成果年份回溯”的阶段对比路线，但明确把它命名为：

- `runtime_snapshot_vs_projection_baseline`

并明确参考基线类型：

- `yearly_projection_baseline`

页面和接口现在会明确告诉用户：

- 当前运行时画像快照可以和阶段基线做辅助比较
- 但不能替代正式历史快照对比

### 4.3 结构化维度变化解释

每个关键维度变化现在不再只是数字差值，而是新增：

- `change_summary`
- `drivers`
- `interpretation`
- `metric_deltas`
- `boundary_note`

覆盖维度：

- 基础学术产出
- 经费与项目攻关
- 知识产权沉淀
- 人才培养成效
- 学术活跃与声誉
- 跨学科融合度

解释逻辑严格复用现有评分引擎输入，确保前后端口径一致：

- 论文数
- 被引数
- 项目数
- 经费额
- 知识产权数量 / 转化数
- 教学成果数
- 学术服务数
- 合作者数
- 关键词数

### 4.4 快照级摘要展示与导出承接

本次已打通两类承接：

- 页面承接：
  - 深度分析面板中展示快照语义、版本、生成触发、冻结口径、对比边界
- 导出承接：
  - 报告中新增 `snapshot_digest`
  - Markdown 导出新增“快照摘要”段落
  - 阶段对比导出中加入“变化说明”和“驱动因素”

## 5. 关键设计决策与权衡

### 5.1 不新增快照表

原因：

- 当前阶段最关键的是先把快照口径和接口语义说清楚。
- 如果在语义还不稳定时直接落库，会带来迁移、回填和回退成本。
- 现有页面和导出已经可以承接“运行时快照”这一最小闭环。

### 5.2 把“当前运行时快照”与“正式历史快照”分开命名

原因：

- 当前快照只是响应内冻结，不是数据库归档。
- 如果不拆开，会让用户误以为系统已经具备完整时序画像体系。
- 这也是本轮最重要的边界治理点。

### 5.3 阶段对比继续复用现有成果年份回溯口径

原因：

- 这是当前已有真实数据能稳定支撑的最小比较方案。
- 能为后续正式快照体系积累页面结构和接口边界。
- 同时通过 `coverage_note` 和 `boundary_note` 明确限制，避免伪装成正式历史快照。

## 7. 最小人工验证步骤

### 7.1 教师视角

1. 打开教师画像主页。
2. 查看“阶段对比”卡片，确认：
   - 有当前阶段与基线阶段对比
   - 每个维度不只是分数变化，还有变化说明、驱动因素和边界说明
3. 查看“快照边界”卡片，确认：
   - 有快照版本
   - 有生成触发
   - 有冻结口径
   - 有对比边界
4. 导出 Markdown 报告，确认：
   - 有“快照摘要”
   - 有“阶段对比”
   - 有“变化说明”

### 7.2 管理员视角

1. 管理员查看指定教师画像。
2. 确认管理员可以查看同样的快照与阶段对比说明。
3. 确认仍然只是查看分析结果，不会获得教师私有录入链路。

### 7.3 边界验证

1. 选择成果样本较少的教师。
2. 打开画像页。
3. 确认系统显示：
   - 阶段对比不可用或样本不足
   - 明确保留接口边界
   - 不伪造历史快照结论
