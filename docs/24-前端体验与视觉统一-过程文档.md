# 前端体验与视觉统一 - 过程文档

## 1. 任务背景

本轮目标是在不推翻现有页面结构、不破坏主链路的前提下，把系统前端从“功能页可用”进一步收口为“有统一视觉规则的工作台”。  
本次保持现有顶部导航的品牌风格不变，重点统一页面内部的标题区、操作区、卡片区、筛选区、工具区、图表说明区以及空态/异常态。

## 2. 扫描范围

本次重点扫描并落地了以下核心页面：

- 教师画像页
- 成果管理中心
- 学院级看板
- 项目推荐页
- 项目指南管理页
- 教师管理页
- 教师个人中心
- 智能问答页

扫描重点包括：

- 页面标题区是否存在风格碎片化
- 页面级操作按钮是否样式不一致
- 卡片头部和内容容器是否重复定义
- 筛选栏与表格工具区是否重复造型
- 图表说明文案、空态、异常态是否缺乏统一表达
- 小屏下标题区、按钮组、卡片留白是否稳定

## 3. 统一方案

### 3.1 设计策略

采用“共享页面样式层 + 页面最小接线”的方式，而不是大规模拆页面或重做组件树。

原因：

- 当前多页结构已经稳定，直接重做页面骨架风险高
- 大量页面已有相近布局，最适合抽公共样式而不是重写模板
- 共享样式可快速覆盖多个核心页面，收益明显且回滚成本低

### 3.2 共享样式入口

新增共享样式文件：

- `frontend/src/assets/workspace-system.css`

并在：

- `frontend/src/main.ts`

统一引入。

- 筛选区和工具区缺乏统一外观
- 图表说明、弱提示、空态与异常态表达口径不一致

## 4. 具体落地页面

本次已接入统一样式的核心页面包括：

- `DashboardView.vue`
- `AchievementEntryView.vue`
- `ProjectRecommendationView.vue`
- `ProjectGuideManagementView.vue`
- `AcademyDashboardView.vue`
- `TeacherManagementView.vue`
- `TeacherProfileEditorView.vue`
- `AssistantDemoView.vue`

统一内容主要包括：

- 页面标题区改为统一 Hero 风格
- 页面级按钮区统一为共享操作栏
- 主体卡片接入统一容器样式
- 主要分区标题接入统一 section header
- 空态、非管理员异常态接入统一状态容器

## 5. 与顶部导航的风格关系

本次没有重做顶部导航。  
共享页面样式在颜色、圆角、渐变和玻璃感上主动向现有顶部导航靠拢，避免出现“导航是一套、内容区是另一套”的割裂感。

重点保留了以下风格特征：

- 深蓝到青蓝的品牌渐变基调
- 较柔和的圆角与浅阴影
- 页面内容的半透明/浅玻璃感容器
- 管理后台工作台风格，而非营销页风格

## 6. 响应式处理

本次没有引入新的响应式框架，只在共享样式层做了轻量适配：

- 标题区在小屏下转为纵向堆叠
- 页面级操作按钮允许换行
- 通用工具区和筛选区在窄屏下保持单列或自适应换行
- 统一卡片和内容壳层边距，避免小屏贴边

## 7. 本次刻意不做的内容

为保持主链路稳定，本次明确没有做以下事项：

- 不重做页面路由结构
- 不重构为全量通用 Vue 组件库
- 不推翻现有各页业务模板
- 不重写顶部导航
- 不把所有内层小组件样式一次性全部改完

这意味着当前系统已经具备“统一视觉骨架”，但部分深层业务卡片仍保留局部页面样式，属于后续可继续收口的部分，而不是本次遗漏。

## 8. 验证计划

### 8.1 实际执行命令

- `npm run type-check`
- `npm run verify:ui`

### 8.2 最小人工验证步骤

1. 登录系统并确认顶部导航风格未被破坏。
2. 依次进入教师画像、成果中心、项目推荐、学院看板、教师管理、教师个人中心、智能问答页面。
3. 检查页面标题区是否统一为相近的 Hero 风格。
4. 检查页面级按钮区是否统一为相近布局和风格。
5. 检查卡片头部、筛选区、表格工具区是否较之前更一致。
6. 检查空态和权限不足等异常态是否具有统一外观。
7. 缩小浏览器宽度，确认标题区与按钮区不会明显错位。

## 9. 当前边界与后续建议

### 9.1 当前边界

- 当前统一的是“页面骨架和高频视觉元素”，不是完整 Design System 重构
- 当前未统一全部业务子组件的内部细节
- 当前未新增独立主题切换机制

### 9.2 后续建议

若后续继续增强，优先级建议如下：

1. 继续把图谱页和更深层的业务卡片接入共享 section/card 规范
2. 把筛选栏和表格工具区进一步抽成轻量通用组件
3. 为页面说明文案、空态文案建立更固定的文案模板
4. 在不影响现有主链路前提下，逐步收敛局部按钮和表单控件尺寸
