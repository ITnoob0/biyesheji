# 75-教师画像持久化快照与同侪基准增强-过程文档

## 1. 目标与范围
- 将教师画像历史年份从运行时快照升级为可持久化快照（`PortraitSnapshot`）。
- 优化同侪基准计算性能，避免每次请求全表实时聚合。
- 在权限边界上区分教师/学院管理员/系统管理员的可见基准范围。
- 升级前端雷达图为多图层展示，并支持年份切片、基准切换与平滑过渡。
- 为关键业务场景补齐后端测试并完成前后端联调验证。

## 2. 后端改动
### 2.1 模型
- 文件：`backend/achievements/models.py`
- 新增模型：`PortraitSnapshot`
  - 字段：`user`、`year`、`dimension_scores`、`total_score`、`created_at`
  - 约束：`(user, year)` 唯一
  - 索引：`(user, year)`、`(year,)`

### 2.2 迁移
- 文件：`backend/achievements/migrations/0014_portraitsnapshot.py`
- 生成并应用迁移，确保本地数据库与测试数据库结构一致。

### 2.3 画像分析与缓存
- 文件：`backend/achievements/portrait_analysis.py`
- 新增/增强内容：
  - `resolve_portrait_payload`：历史年份优先读快照；缺失时回溯计算并落库。
  - `_resolve_or_backfill_snapshot`：历史快照回填逻辑。
  - 并发防重：`cache.add` 5 秒短锁，避免同年并发重复回溯。
  - `calculate_benchmark_scores`：院级/校级基准统一计算并缓存（24h）。
  - `calculate_college_comparison_scores`：系统管理员横向学院基准数据。
  - `invalidate_benchmark_score_cache`：审批通过时失效缓存版本。
  - `build_stage_comparison`、`build_snapshot_boundary`：支持按 `year` 返回历史冻结口径。

### 2.4 视图与权限分层
- 文件：`backend/achievements/views.py`
- 新增接口：
  - `GET /api/achievements/portrait/analysis/`（`TeacherPortraitAnalysisView`）
- 数据分层：
  - 教师/学院管理员：仅返回院级基准（`college_average_data`）。
  - 系统管理员：额外返回全校基准（`university_average_data`）与学院横向对比。
  - 非系统管理员请求 `scope=university`：后端强制降级为 `college` 并记录告警日志。
- 缓存失效挂钩：
  - 成果审核通过时触发 `invalidate_benchmark_score_cache()`。

### 2.5 路由
- 文件：`backend/achievements/urls.py`
- 增加：
  - `path('portrait/analysis/', TeacherPortraitAnalysisView.as_view(), ...)`

### 2.6 测试
- 文件：`backend/achievements/test_portrait_snapshot_benchmark.py`
- 新增 4 组测试用例并全部通过：
  1. 冷启动 vs 热启动：首次生成快照，二次命中快照/缓存。
  2. 角色越权隔离：教师与学院管理员不可获得全校基准，系统管理员可获得。
  3. 跨年冻结效应：历史快照不被后续补录回写污染。
  4. 单教师学院边缘场景：学院均值计算安全且与教师本人重合。

## 3. 前端改动
### 3.1 雷达图组件
- 文件：`frontend/src/components/RadarChart.vue`
- 升级点：
  - 支持多图层输入：`seriesData`。
  - 视觉分层：
    - 当前教师：蓝色实线 + 半透明填充。
    - 基准层：灰色虚线 + 淡填充。
  - Tooltip 升级：
    - 展示教师分值、基准分值与差值（领先/落后）。
  - View Transitions：
    - 雷达容器设置 `view-transition-name: portrait-radar-canvas`。
    - 补充 `::view-transition-old/new(...)` 动画时长。

### 3.2 深度分析面板
- 文件：`frontend/src/views/dashboard/PortraitDeepAnalysisPanel.vue`
- 新增交互：
  - 年份切片下拉（`selectedYear`、`availableYears`）。
  - 系统管理员基准切换（本院/全校）。
  - 显示当前实际叠加基准（处理后端降级后的真实 scope）。

### 3.3 教师画像主页面
- 文件：`frontend/src/views/DashboardView.vue`
- 新增逻辑：
  - 对接 `GET /api/achievements/portrait/analysis/`。
  - 维护年份与基准切换状态。
  - 切换年份/基准时使用 `document.startViewTransition` 包裹雷达更新。
  - 将 `RadarChart` 改为 `radarData + radarSeriesData` 双输入。

### 3.4 类型定义
- 文件：`frontend/src/views/dashboard/portrait.ts`
  - 新增 `PortraitAnalysisResponse`、`RadarSeriesItem`、基准相关类型定义。
- 文件：`frontend/src/types/achievements.ts`
  - 新增 `PortraitSnapshot` TypeScript 接口。
