# 73-校内合著者成果认领机制-过程文档

## 1. 需求背景
- 目标：解决校内多位教师合作论文重复录入问题。
- 核心机制：一位教师录入后，系统自动识别同名校内合著者并发起“待认领邀请”；被邀请教师确认后，成果纳入其画像统计。
- 角色边界：
  - 教师：处理本人待认领邀请（确认/拒绝）。
  - 学院管理员：仅查看本院长期未认领数据并发送提醒。
  - 系统管理员：不参与教师侧认领动作。

## 2. 本次改动范围

### 2.1 后端模型与迁移
- 新增模型：`backend/achievements/models.py`
  - `AchievementClaim`
  - 字段：`achievement`、`initiator`、`target_user`、`status(PENDING/ACCEPTED/REJECTED)`、`created_at`
  - 约束：`(achievement, target_user)` 唯一
  - 索引：`(status, created_at)`、`(target_user, status)`
- 新增迁移：`backend/achievements/migrations/0011_achievementclaim.py`

### 2.2 后端服务层
- 新增文件：`backend/achievements/services.py`
  - `sync_paper_claim_invitations(paper)`：
    - 解析论文合著者
    - 按姓名匹配有效校内教师（排除录入者本人）
    - 自动生成/更新 `PENDING` 认领邀请
    - 回填 `CoAuthor.is_internal/internal_teacher`
    - 清理已不匹配合著者的陈旧 `PENDING` 邀请
  - `build_teacher_related_paper_queryset(...)`：
    - 支持“本人论文 + 已认领论文”聚合查询（去重）

### 2.3 后端接口与权限
- 更新：`backend/achievements/views.py`、`backend/achievements/serializers.py`、`backend/achievements/urls.py`
- 新接口：
  - `GET /api/achievements/claims/pending-count/` 教师待认领数量
  - `GET /api/achievements/claims/pending/` 教师待认领列表
  - `POST /api/achievements/claims/<id>/accept/` 教师确认认领
  - `POST /api/achievements/claims/<id>/reject/` 教师拒绝认领
  - `GET /api/achievements/claims/college-unclaimed/` 学院管理员查看本院长期未认领记录
  - `POST /api/achievements/claims/college-unclaimed/remind/` 学院管理员一键发送提醒
- 自动识别挂钩位置：
  - 论文手工录入 `perform_create`
  - 论文编辑更新 `perform_update`
  - BibTeX 确认导入 `bibtex_confirm`
- 权限边界：
  - 教师仅能操作自己收到的邀请
  - 管理员不可执行教师认领动作
  - 学院管理员未认领追踪严格按本院过滤

### 2.4 画像统计联动
- 更新：`backend/achievements/scoring_engine.py`、`backend/achievements/views.py`
- 已认领论文并入：
  - 教师画像评分指标（论文数、被引、关键词、合作作者）
  - 教师“全部成果”接口聚合
  - 画像数据更新时间口径
- 前端画像论文请求补充参数 `include_claimed=1`，确保画像视角可读已认领论文。

### 2.5 前端页面与交互
- 新增页面：`frontend/src/views/personal-center/AchievementClaimView.vue`
  - 待认领列表展示论文摘要
  - 操作按钮：确认认领、拒绝认领
- 路由接入：`frontend/src/router/workspaceRoutes.ts`
  - 个人中心新增二级菜单“成果认领”（教师可见）
- 个人中心徽标：`frontend/src/views/TeacherProfileEditorView.vue`
  - 新增“待认领成果”Badge，点击进入认领列表
- 学院管理员追踪区块：`frontend/src/views/AcademyDashboardView.vue`
  - 新增“异常/未认领数据追踪”
  - 支持按超时天数查看并“一键发送系统提醒”

## 4. 风险与边界说明
- 当前“系统提醒”为接口级提醒回执，不依赖站内信中心；后续如接入通知中心可扩展为真实消息下发。
- 姓名匹配基于 `real_name` 精确匹配；同名教师场景会分别生成邀请，建议后续可叠加工号/单位等辅助判定策略。
- 本次遵循最小改动原则，未重构既有权限系统，仅在现有架构上增量扩展。
