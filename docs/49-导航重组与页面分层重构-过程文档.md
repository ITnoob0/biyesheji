# 49-导航重组与页面分层重构-过程文档

## 1. 本次改造目标

本轮前端改造不再继续沿用“一个页面塞满所有信息”的长页面工作台模式，而是在现有 Vue 3 + TypeScript + Element Plus 工程基础上，做一次轻量的信息架构级重构：

- 左侧改为模块级一级菜单为主
- 左侧展开区承接模块内稳定二级菜单
- 顶部承接品牌、页面上下文、用户区和辅助操作
- 右侧保持独立内容区与独立滚动
- 长页面优先收口为“摘要页 + 子视图”

本次没有引入 Vben Admin、vue-element-admin 或其他后台模板，也没有引入 tags-view、keep-alive、多页签体系。

## 2. 路由重组方式

### 2.1 主布局

- `App.vue` 继续保持纯入口壳，仅保留 `router-view`
- 登录后工作台统一挂在 `MainLayout`
- `/login`、`/register`、`/forgot-password` 继续保持独立，不进入主布局

### 2.2 模块级路由

工作台路由已从“扁平 children”重组为“模块父路由 + 子视图 children”，并尽量保留原有 URL 主路径：

- `/profile-editor`
- `/dashboard`
- `/entry`
- `/graph`
- `/project-recommendations`
- `/assistant-demo`
- `/teachers`
- `/academy-dashboard`
- `/project-guides`

其中旧 URL 主路径继续可用，模块内新增的二级子视图采用子路由承接，例如：

- `/profile-editor/public-profile`
- `/dashboard/dimensions`
- `/entry/manage`
- `/graph/collaboration`
- `/project-recommendations/feedback`
- `/assistant-demo/sources`
- `/academy-dashboard/ranking`

## 3. 一级菜单与二级菜单生成规则

### 3.1 一级菜单

一级菜单不再按旧页面分组硬编码，而是基于 `workspaceRoutes.ts` 中的模块父路由生成：

- 使用 `route.meta.moduleRoot = true`
- 使用 `title / icon / order / moduleKey / requiresAdmin / menuRoles`
- 结合当前登录用户身份做过滤

### 3.2 二级菜单

二级菜单由当前模块的可见子路由生成：

- 使用子路由 `title / order`
- `hiddenInMenu` 的路由不显示
- 管理员专属模块与子路由仅管理员可见
- 教师不可见管理员专属模块

### 3.3 权限边界

本次仍沿用现有轻量权限语义：

- `requiresAuth`
- `requiresAdmin`
- `guestOnly`
- `sessionStorage` 登录态

未引入新的 RBAC 平台或后端驱动菜单机制。

## 4. 默认首页语义

本轮已恢复并明确默认落地规则：

- 管理员默认首页：`/dashboard`
- 教师默认首页：`/profile-editor`

同时同步修正了：

- `workspaceNavigation.js`
- `sessionFlow.js`
- `verify-third-round-ui.mjs`

确保登录后和访问根路径时的默认去向与当前任务要求一致。

## 5. 页面分层结果

本轮优先处理了当前最容易形成长页面堆叠的模块页面。

### 5.1 个人中心

在 `TeacherProfileEditorView.vue` 增加 `sectionMode`，形成：

- 总览
- 公开资料
- 账户安全
- 快捷入口 / 代表成果

总览页保留摘要、状态和主要入口，详细资料与密码安全不再强制与全部内容同时展开。

### 5.2 教师画像

在 `DashboardView.vue` 增加 `sectionMode`，形成：

- 画像总览
- 维度解释
- 趋势分析
- 关键词 / 主题
- 形成说明

重构后不再要求用户通过单页长滚动查看全部趋势、解释、阶段对比和证据区。

### 5.3 成果中心

在 `AchievementEntryView.vue` 增加 `sectionMode`，形成：

- 成果总览
- 成果列表 / 录入
- BibTeX 导入
- 统计摘要
- 代表成果

当前仍复用同一业务页的数据与录入逻辑，但把摘要与录入操作拆到不同子视图承接。

### 5.4 学术图谱

新增 `AcademicGraphWorkspaceView.vue`，将图谱从画像长页面中独立为一级模块，并形成：

- 图谱总览
- 合作网络
- 主题热点
- 图谱说明

图谱组件继续复用原有 `AcademicGraph.vue`，并新增 `initialFocusPreset` 来匹配不同子视图的默认焦点。

### 5.5 项目推荐

在 `ProjectRecommendationView.vue` 增加 `sectionMode`，形成：

- 推荐结果
- 收藏 / 反馈
- 推荐解释

当前仍保留原有推荐主链路、反馈弹窗和历史抽屉，但将结果、反馈摘要和解释区做了分层展示。

### 5.6 智能问答

在 `AssistantDemoView.vue` 增加 `sectionMode`，形成：

- 问答主页
- 常见问题
- 来源说明 / 证据入口

常见问题和来源说明采用轻量引导卡承接，不改动原有问答接口和来源卡片逻辑。

### 5.7 管理员页面

管理员侧本轮完成了轻量分层：

- 教师管理：总览 / 教师列表 / 账户操作
- 学院看板：总览 / 趋势与对比 / 排行与钻取 / 导出与说明
- 项目指南管理：指南总览 / 新增编辑 / 生命周期管理

仍复用现有管理页接口和核心操作逻辑。

## 6. 图表 resize 兼容

布局改为左侧主导航 + 右侧独立内容区后，继续保留本轮之前已完成的图表兼容方案：

- `ResizeObserver` 监听容器尺寸变化
- 组件卸载时清理 observer
- 组件卸载时销毁 ECharts 实例

重点覆盖页面仍包括：

- 教师画像
- 学院看板
- 学术图谱
- 雷达图组件

本轮新增图谱独立模块页后，图谱组件仍沿用同一套 `ResizeObserver` 兼容逻辑。

## 7. 旧 URL 兼容情况

本轮尽量保留旧 URL 主路径，不做大面积失效改造：

- `/dashboard` 保留
- `/profile-editor` 保留
- `/entry` 保留
- `/project-recommendations` 保留
- `/assistant-demo` 保留
- `/teachers` 保留
- `/academy-dashboard` 保留
- `/project-guides` 保留

新增的二级子视图使用子路由补充，不替换旧主路径。

## 8. 当前未做事项

本轮明确未做以下能力：

- tags-view
- keep-alive 页面缓存体系
- 顶部一级导航驱动左侧二级导航
- 后端驱动菜单
- 折叠侧栏
- 复杂移动端抽屉导航

这些能力都不属于当前“轻量模块化工作台重构”的最小闭环。

## 10. 风险与后续建议

### 10.1 当前边界

- 当前页面分层以 `sectionMode + 子路由` 为主，没有把所有模块拆成大量全新页面
- 这是有意保持增量改造，避免一次性重写业务逻辑

### 10.2 后续建议

- 若后续继续打磨，可优先补充移动端抽屉菜单
- 可继续把首页中的入口卡片做成更强的“最近变化 / 待办 / 快捷钻取”
- 若后续要进一步解耦，可再把少数子视图从 `sectionMode` 逐步抽成独立轻量组件，但当前阶段没有必要一口气拆太散
