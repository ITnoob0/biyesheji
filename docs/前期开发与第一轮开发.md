## 2. 项目定位

项目名称：高校教师科研画像与智能辅助系统

当前阶段定位：

- 核心目标已经从“大而全平台设想”收敛为“可运行、可演示、可验收的一期系统”。

## 5. 当前实际页面与路由

当前前端已落地路由如下：

- `/login`：登录页
- `/register`：教师注册页
- `/forgot-password`：忘记密码页
- `/dashboard`：当前登录教师画像主页
- `/profile/:id`：管理员查看指定教师画像页
- `/entry`：论文录入与论文记录管理页
- `/teachers`：教师基础信息管理页
- `/`：重定向到 `/dashboard`

当前路由规则：

- 除 `/login`、`/register`、`/forgot-password` 外，其余页面全部要求登录
- 每次路由跳转都会重新校验当前标签页会话
- 非管理员访问 `/profile/:id`` 时只能看自己的画像

## 7. 当前已打通的核心业务链路

### 7.1 认证链路

登录 -> 生成 JWT -> 写入当前标签页 `sessionStorage` -> 获取 `/api/users/me/` -> 进入业务页面

补充说明：

- 教师登录用户名已统一为“工号”
- 教师注册时必须自行设置密码
- 忘记密码按“工号 + 真实姓名 + 新密码”重置
- 退出登录通过清除当前标签页会话完成，不影响其他标签页

### 7.2 教师信息链路

教师/管理员维护基础信息 -> 写入 `users_customuser` 与 `achievements_teacherprofile` -> 画像页直接读取并展示

### 7.3 论文录入链路

论文表单提交 -> 写入 `Paper` -> 写入 `CoAuthor` -> 写入 `ResearchKeyword / PaperKeyword` -> 刷新论文列表 -> 画像页读取统计与标签

### 7.4 画像展示链路

进入画像页 -> 获取教师基础档案 -> 获取 dashboard 统计 -> 获取 radar 数据 -> 获取 papers -> 前端实时计算关键词画像、合作画像、趋势图 -> 获取 topology 数据并展示

### 7.5 图谱回退链路

优先尝试 Neo4j -> 如果 Neo4j 关闭或不可用 -> 自动回退到 MySQL 实时组装拓扑图 -> 保证画像页学术关系图可展示

## 8. 当前实际数据模型基线

当前代码中已经存在并实际使用的数据模型如下：

### 8.1 用户与教师基础信息

- `users.CustomUser`
- `achievements.TeacherProfile`

说明：

- 教师登录账号直接使用工号
- `TeacherProfile` 当前承担教师扩展信息角色，而不是“画像统计快照表”

### 8.2 论文及其关联实体

- `achievements.Paper`
- `achievements.ResearchKeyword`
- `achievements.PaperKeyword`
- `achievements.CoAuthor`

### 8.3 其他成果类型

- `achievements.Project`
- `achievements.IntellectualProperty`
- `achievements.TeachingAchievement`
- `achievements.AcademicService`

### 8.4 当前数据模型与原规划的差异

原规划中曾设想单独的“teacher_profile 画像统计表”“funding_call”“recommendation”等表。

当前代码实际情况是：

- 已有教师扩展档案表，但不是推荐算法专用画像快照表
- 推荐相关表尚未落地
- 项目指南表尚未落地

因此后续 Agent 在继续开发时，必须以现有模型为起点，不要假设相关推荐表已经存在。

## 11. 当前认证设计定稿

这一部分是后续 Agent 不应再随意改动的系统规则。

### 11.1 当前统一规则

- 教师登录用户名 = 六位工号
- 教师注册时必须设置密码
- 管理员创建教师账号时也必须设置初始密码
- 忘记密码按工号和真实姓名校验
- 登录状态仅保存在当前标签页 `sessionStorage`

### 11.2 当前严格路由策略

- 所有业务路由必须校验当前标签页会话
- 若 token 缺失或当前用户上下文失效，立即清空会话并跳转 `/login`
- 不允许依赖 `localStorage` 存储认证信息

### 11.3 当前退出登录策略

- 退出登录 = 清空当前标签页的 token 和用户上下文
- 不做全站全标签页强退
- 目的是保留多标签页多账号并行使用能力

## 14. 当前项目风险与注意事项

### 14.1 编码风险

当前部分存量中文文案在终端读取时存在乱码表现，后续建议统一排查文件编码并全部收敛到 UTF-8。

### 14.2 外部依赖风险

- Neo4j 不应阻断主链路
- AI 服务不可用时，论文录入也必须成功

### 14.3 功能边界风险

后续答辩或文档写作时，不能把“推荐系统、学院级决策看板、智能问答”表述为当前已完成模块，只能表述为后续扩展方向。

## 16. 面向后续 Agent 的硬约束

后续 Agent 开发必须遵守以下约束：

- 不得把认证从 `sessionStorage` 改回 `localStorage`
- 不得再把教师登录账号设计为 `teacher1/teacher2` 这类自动用户名
- 不得让 Neo4j 成为论文录入和画像展示的强依赖
- 所有新页面必须接入现有严格路由守卫
- 所有新增功能必须注明“是否纳入本阶段验收”
- 修改接口时必须同步更新本文档中的接口清单与验收口径
