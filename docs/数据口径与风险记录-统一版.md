# 数据口径与风险记录-统一版

最后更新时间：2026-03-21

## 1. 说明

本文件统一说明当前系统的主要数据口径、实时聚合边界、回退逻辑和风险记录。  
目标不是重复所有过程文档，而是给开发交接、论文写作和答辩讲解提供一致口径。

## 2. 核心数据口径

### 2.1 用户与身份

- 教师账号主认证字段：工号
- 登录态：前端 `sessionStorage`
- 管理员边界：当前主要依据 `is_staff / is_superuser`
- 当前用户资料：`CustomUser` + `TeacherProfile`

### 2.2 个人中心字段边界

#### 公开展示字段

- 头像
- 姓名
- 个人简介
- 邮箱
- 电话
- 院系
- 职称
- 研究领域 / 学科
- 研究方向 / 研究兴趣

#### 内部管理字段

- 工号
- 登录账号
- 角色身份
- 账户状态
- 密码状态
- 密码更新时间

说明：

- 当前“公开展示”主要指系统内展示口径
- 不代表匿名公开网页已经完整上线

### 2.3 成果统计口径

当前纳入成果聚合的类型包括：

- 论文
- 项目
- 知识产权
- 教学成果
- 学术服务

画像、问答、看板和推荐中用到的成果统计，默认均基于这些已入库的系统内数据。

### 2.4 画像口径

当前画像主要来自：

- 教师基础资料
- 论文及其他成果聚合
- 雷达评分引擎
- 成果年份回溯估算的趋势

说明：

- 当前趋势不等于历史快照
- 当前尚未引入冻结版画像快照表

### 2.5 图谱口径

- 默认优先使用 Neo4j
- 若 Neo4j 不可用，则回退到 MySQL 关系数据
- 回退后仍可展示：
  - 图谱主体
  - 合作网络概览
  - 合作者类型分析
  - 研究主题热点分析

说明：

- 当前图谱展示不以 Neo4j 为强依赖

### 2.6 推荐口径

当前推荐属于：

- 规则增强型、可解释推荐

主要依据包括：

- 教师研究方向
- 学科与院系
- 关键词匹配
- 当前成果活跃度
- 指南窗口与基础条件

说明：

- 当前不是机器学习推荐
- 当前不应表述为语义级政策智能推荐平台

### 2.7 问答口径

当前问答仅基于：

- 教师资料
- 成果聚合
- 推荐结果
- 学院统计

说明：

- 当前问答是模板化、受控、可回退的系统内辅助能力
- 当前不支持外部知识检索和开放领域回答

### 2.8 学院看板口径

当前学院看板以实时聚合为主，支持：

- 院系
- 年份
- 教师
- 职称
- 合作情况

说明：

- 当前仍是轻量管理分析口径
- 不是离线 BI 口径

## 3. 实时聚合与未来离线聚合边界

### 当前继续使用实时聚合的原因

- 当前数据规模和演示目标下仍可接受
- 当前尚未出现必须引入离线聚合的明确收益
- 保持实现克制，避免引入额外维护成本

### 哪些指标当前属于实时聚合

- 教师画像主页统计
- 教师画像雷达
- 推荐结果计算
- 学院看板统计
- 问答回答中的统计摘要

### 哪些方向未来更适合离线聚合

- 学院级长期趋势报表
- 跨年度横向对比看板
- 高并发推荐结果缓存
- 画像快照或审计型展示

## 4. 当前主要风险记录

### 4.1 依赖与环境一致性风险

- 仓库当前没有正式 Python 依赖清单文件
- 若迁移未执行，登录与资料链路容易暴露数据库字段不一致问题

### 4.2 测试环境状态风险

- 当前测试依赖现有 MySQL 测试库环境
- `test_biyesheji` 已存在时，需使用 `--keepdb --noinput`

### 4.3 文档分散风险

- 多轮过程文档较多
- 若只看历史过程文档，容易把阶段性口径误当作当前状态

### 4.4 展示与能力表述风险

- 推荐、问答、图分析和学院看板都容易在答辩中被误讲成“完整平台”
- 当前必须坚持使用“轻量”“规则增强”“受控”“回退可用”等口径

## 5. 当前遗留问题统一口径

### 当前仍未完成

- 正式 CI 工作流
- 完整前端测试框架
- 画像快照体系
- 推荐缓存体系
- 学院看板离线聚合
- RAG 问答体系
- 高级图挖掘能力

### 当前不应误表述为遗漏

- 未新增快照 / 缓存 / 离线聚合表
- 未引入复杂 RBAC
- 未引入复杂模型推荐

这些都是当前阶段有意保持克制的架构选择，不是“忘记做”。

## 6. 后续建议

建议后续统一沿以下顺序推进：

1. 先保证依赖清单、迁移执行和验证命令完整
2. 再继续增强文档、测试与演示基线
3. 最后依据真实需求决定是否进入快照、缓存、离线聚合或更强模型化能力
