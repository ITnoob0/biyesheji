# 41-推荐系统与项目指南深化-过程文档

## 1. 任务背景

本轮目标是在现有“规则增强推荐可演示”基础上，把推荐模块深化为可持续演进的业务模块。重点不是引入复杂模型，而是在当前项目指南推荐链路上补齐收藏持久化、推荐历史、轻量反馈、解释可视化、画像联动和管理员生命周期管理能力。

## 3. 本次落地内容

### 3.1 规则档位与配置增强

- 保留原有规则增强推荐主路线。
- 在原有规则档位基础上新增：
  - `PORTRAIT_FIRST` 画像联动优先
  - `FOUNDATION_FIRST` 申报基础优先
- 新增 `rule_config` 字段，用于按指南配置轻量加成项：
  - `keyword_bonus`
  - `discipline_bonus`
  - `activity_bonus`
  - `window_bonus`
  - `support_bonus`
  - `portrait_bonus`
- 管理端表单已支持维护这些规则细化参数。

### 3.2 收藏持久化

- 新增 `ProjectGuideFavorite` 表。
- 收藏状态不再只依赖前端本地状态，现已后端持久化。
- 推荐页刷新、跨会话访问后仍能保留收藏状态。

### 3.3 推荐结果历史记录

- 新增 `ProjectGuideRecommendationRecord` 表。
- 每次推荐请求会为当前推荐结果生成历史记录批次。
- 历史记录保存：
  - 指南标题快照
  - 规则档位快照
  - 推荐得分
  - 推荐理由和解释维度
  - 命中关键词/学科
  - 画像联动说明
  - 收藏状态快照
  - 反馈信号

### 3.4 教师反馈信号采集

- 当前先落地轻量显式反馈，不进入复杂反馈学习模型。
- 支持反馈信号：
  - 感兴趣
  - 暂不相关
  - 计划申报
  - 已申报
- 反馈会写入推荐历史记录，便于后续规则复盘与迭代。

### 3.5 推荐解释可视化

- 现有解释维度从纯文本扩展为结构化可视化信息。
- 每个解释维度增加：
  - 得分
  - 占比
  - 细节说明
- 前端已使用进度条和结构卡片展示解释贡献。

### 3.6 推荐与画像维度联动说明

- 推荐结果已补充 `portrait_dimension_links`。
- 当前会把指南命中与教师画像中的以下维度进行联动解释：
  - 跨学科融合度
  - 基础学术产出
  - 经费与项目攻关
  - 当前最强画像维度
- 页面新增画像联动说明区，帮助教师理解“推荐和画像为什么有关”。

### 3.7 项目指南生命周期管理增强

- 项目指南状态新增 `ARCHIVED`。
- 指南模型新增生命周期字段：
  - `published_at`
  - `closed_at`
  - `archived_at`
  - `lifecycle_note`
- 管理端新增：
  - 生命周期摘要卡片
  - 规则配置覆盖统计
  - 近 30 天到期提醒
  - 一键归档 / 重新开放

## 4. 为什么当前仍然是规则增强推荐

当前明确仍以规则增强推荐为主，不进入复杂模型化推荐，原因如下：

- 当前反馈样本量不足，难以支撑稳定的监督学习或排序模型。
- 指南文本结构化程度有限，直接上复杂模型容易放大噪声。
- 当前阶段业务更需要“可解释、可验证、可管理”的推荐结果。
- 教师和管理员都需要先积累收藏、反馈和历史，再决定是否进入更复杂建模。

因此本轮优先建设：

- 规则档位与规则配置
- 历史记录
- 收藏持久化
- 显式反馈
- 画像联动解释

而不是直接引入复杂推荐模型。

- 当前历史记录的是“规则增强推荐结果快照”，不是完整实验日志系统。
- 当前反馈只采集显式信号，不代表已形成推荐模型训练闭环。
- 当前推荐解释强调可追溯，不宣称具备高级智能推荐能力。

## 8. 最小人工验证步骤

1. 使用管理员账号进入“项目指南管理”页面。
2. 新建一条指南，设置规则档位和规则细化配置，确认列表中可看到规则细化标签。
3. 将一条指南归档，再重新开放，确认生命周期时间与摘要卡片同步变化。
4. 使用教师账号进入“项目指南推荐”页面。
5. 刷新推荐后，确认页面出现：
   - 收藏状态
   - 推荐历史预览
   - 画像联动说明
   - 反馈按钮
6. 点击收藏并刷新页面，确认收藏状态仍保留。
7. 点击“感兴趣”或“计划申报”，再打开历史抽屉，确认反馈信号已写入历史。

## 9. 后续建议

- 下一阶段如需更强推荐，可先基于当前反馈和收藏数据做规则调优，而不是直接跳到复杂模型。
- 当指南文本结构化和反馈样本积累更稳定后，再考虑引入更强的召回或排序模块。
- 如果学院侧需要管理视角进一步深化，可继续补“按院系 / 职称 / 研究方向”的推荐分析报表。
