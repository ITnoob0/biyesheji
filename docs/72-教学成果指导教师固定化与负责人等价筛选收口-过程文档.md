# 72-教学成果指导教师固定化与负责人等价筛选收口-过程文档

## 1. 需求说明
- 针对教学成果中的两类成果：
  - `学生竞赛 (COMPETITION)`
  - `优秀论文指导 (THESIS)`
- 角色要求调整为：
  - 教师端不允许手动填写承担角色；
  - 系统固定为“指导教师”语义；
  - 在成果筛选与排行口径上与“负责人”保持等价；
  - 筛选项中不新增“指导教师”单独选项。

## 2. 实施策略（最小改动）
- 不新增数据库角色枚举（不引入 `ADVISOR` 可选值），避免额外角色分叉。
- 存储层继续使用 `role='PI'` 承担“负责人等价”口径。
- 展示层在教学成果接口中对 `COMPETITION/THESIS` 返回 `role_display='指导教师'`，实现文案语义。
- 前端对这两类成果锁定角色输入（不可编辑），显示固定“指导教师”。

## 3. 代码改动

### 3.1 后端
文件：`backend/achievements/serializers.py`

- `TeachingAchievementSerializer`：
  - `role_display` 改为 `SerializerMethodField`。
  - 新增 `validate`：当 `achievement_type in {'COMPETITION', 'THESIS'}` 时，强制 `role='PI'`。
  - 新增 `get_role_display`：上述两类返回“指导教师”，其余返回模型默认 `get_role_display()`。

说明：
- 这样在“全部成果”筛选中仍按 `lead` 口径归入负责人，不会新增筛选维度。

### 3.2 前端
文件：`frontend/src/views/AchievementEntryView.vue`

- 教学成果录入表单：
  - 对 `COMPETITION/THESIS`，承担角色输入改为禁用输入框，固定显示“指导教师”。
  - 其他教学成果类型保留“负责人/参与人”下拉选择。
- 新增监听逻辑：
  - 当教学成果类型切换到 `COMPETITION/THESIS` 时，内部提交值自动锁定为 `role='PI'`。

## 4. 数据一致性处理
- 执行一次性规范化：
  - `TeachingAchievement` 中 `COMPETITION/THESIS` 且 `role != 'PI'` 的记录更新为 `PI`。
- 执行结果：`0` 条（当前数据已满足）。

## 5. 验证结果
- `python manage.py makemigrations --check --dry-run`：无迁移漂移
- `python manage.py check`：通过
- `python manage.py test achievements.tests.AchievementReviewFlowApiTests --keepdb --noinput`：通过
- `npm run type-check`：通过

## 6. 结果结论
- 教师在“学生竞赛/优秀论文指导”录入时不再可手填承担角色，系统固定为“指导教师”语义。
- 排名筛选与展示口径仍按“负责人等价”执行，筛选项无新增“指导教师”。
- 改动保持现有审批流与角色体系不变，满足最小变更要求。
