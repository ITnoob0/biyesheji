# 32-权限边界与角色治理继续收口-过程文档

## 1. 本轮目标

在当前教师 / 管理员双角色体系基础上，继续收口权限边界，重点解决三类问题：

- 越权判断分散，提示口径不一致
- 高风险越权场景测试覆盖不足
- 当前角色边界在现状文档中写得还不够直接

本轮不做复杂 RBAC 重构，也不改变工号登录和 `sessionStorage` 登录态方案。

## 2. 扫描结论

本轮重点扫描了以下位置：

- 后端统一权限入口：`backend/users/access.py`
- 后端账户、成果、推荐、问答、图谱相关视图与服务
- 前端路由守卫：`frontend/src/router/index.ts`
- 前端统一提示层：`frontend/src/utils/authPresentation.js`
- 管理员页面：教师管理、学院看板
- 教师个人中心页面
- 现有后端权限和回归测试

扫描后确认：

- 后端已经有统一权限 helper，但部分模块仍有散落消息
- 前端守卫和管理员页面提示还不完全一致
- 管理员通过教师自助成果入口维护成果的边界还不够明确
- 当前系统已具备未来多角色扩展入口雏形，但需要把 `permission_scope` 明确纳入统一口径

## 3. 本轮改动

### 3.1 后端权限边界收口

- 为教师 / 管理员常见权限场景补充统一消息常量：
  - 教师自助成果维护
  - 画像访问
  - 推荐访问
  - 问答访问
  - 图谱访问
  - 学院统计访问
  - 教师管理范围
- 抽出并复用：
  - `is_teacher_user`
  - `ensure_teacher_user`
  - `ensure_self_user`
- 管理员通过教师管理入口访问管理员账户时，改为明确拒绝
- 教师成果入口改为“教师本人维护，管理员当前仅查看验证”
- 图谱、推荐、问答、学院看板改为统一权限消息来源

### 3.2 前端守卫与角色说明收口

- 路由守卫改为统一使用权限提示 helper，而不是散落硬编码文案
- 教师管理页和学院看板页接入统一管理员入口提示
- 教师个人中心新增“当前权限边界”展示区，展示：
  - 当前角色摘要
  - 当前可执行操作
  - 当前受限边界
  - 多角色扩展预留说明
- `verify:baseline` 补充统一提示 helper 校验

### 3.3 越权测试补强

补充了以下高风险越权或正向权限测试：

- 教师不能创建教师账户
- 当前用户接口包含 `permission_scope`
- 管理员不能通过教师管理入口访问管理员账户详情
- 管理员不能通过教师自助成果入口新增、修改、删除论文
- 教师不能修改或删除项目指南
- 管理员可以查看指定教师图谱
- 管理员可以查看指定教师问答结果

## 4. 当前权限边界口径

### 教师当前可做什么

- 登录、注册、忘记密码、修改自己的密码
- 维护自己的个人资料
- 维护自己的成果
- 查看自己的画像、推荐、图谱和问答结果

### 管理员当前可做什么

- 管理教师账户
- 重置教师密码
- 查看指定教师的画像、推荐、图谱和问答结果
- 查看学院级统计看板
- 维护项目指南

### 当前明确不支持

- 教师访问管理员入口
- 教师查看其他教师的核心分析结果
- 管理员通过教师自助成果入口代替教师维护成果
- 教师管理入口把管理员账户当作教师资料来维护

## 5. 多角色扩展预留说明

当前未实现复杂 RBAC。  
后续若扩展多角色，建议继续沿用当前统一入口：

- `role_code`
- `role_label`
- `permission_scope`
- 后端统一权限 helper
- 前端统一守卫提示 helper

应优先做“新角色能力映射 + 统一提示 + 测试补齐”，不建议直接推翻当前双角色体系。

## 6. 验证命令

本轮实际执行：

- `cmd /c npm run type-check`
- `cmd /c npm run verify:baseline`
- `python manage.py check`
- `python manage.py test --keepdb --noinput users achievements graph_engine project_guides ai_assistant`
- `python scripts/run_validation_pipeline.py`

## 7. 最小人工验证步骤

1. 教师登录后手动访问 `/teachers`、`/academy-dashboard`，确认被统一提示拦回。
2. 教师访问 `/profile/其他教师ID`，确认自动切回自己的画像页。
3. 管理员登录后进入教师管理，确认管理员账户不作为“教师详情”编辑对象暴露。
4. 管理员查看指定教师画像 / 推荐 / 图谱 / 问答，确认可正常查看。
5. 教师个人中心确认能看到“当前权限边界”说明区。
