# 37-认证与账户生命周期完善-过程文档

## 1. 本轮目标

在 P0 已完成的前提下，继续把认证与账户体系从“可以登录”收口到“账户生命周期更完整、角色边界更清晰、管理员操作更集中、恢复提示更一致”的状态。

本轮重点不做认证体系重写，不改变工号登录和 `sessionStorage` 登录态，也不引入复杂 RBAC，仅在现有认证模块、教师管理入口、统一权限入口和提示层上做增量增强。

## 2. 扫描结论

本轮开始前，仓库已经具备：

- 工号登录、教师注册、忘记密码、主动修改密码
- 管理员单个重置教师密码
- 当前用户统一资料接口 `GET/PATCH /api/users/me/`
- 教师管理页与统一权限入口 `permission_scope`
- 统一失效回跳和基础错误提示

仍然存在的增强空间主要有：

- 管理员账户启停 / 恢复能力虽然已有字段基础，但缺少统一摘要和批量入口
- 教师管理页对密码生命周期、账户状态和恢复说明展示不够集中
- 当前文档还没有把“未来多角色扩展的正式口径”明确写成统一入口保留，而不是复杂 RBAC 已上线
- 批量账户操作、生命周期摘要和统一恢复说明缺少测试覆盖

## 3. 本轮实现

### 3.1 后端账户生命周期增强

在 `backend/users` 内继续复用现有认证和教师管理链路，新增了以下能力：

- 教师账户生命周期摘要接口：
  - `GET /api/users/teachers/summary/`
  - 返回教师账户总量、启用数量、停用数量、待修改密码数量、稳定密码数量、恢复说明和未来扩展提示
- 教师账户批量操作接口：
  - `POST /api/users/teachers/bulk-actions/`
  - 当前支持：
    - `activate`
    - `deactivate`
    - `reset_password`
- 教师账户序列化补充生命周期字段：
  - `account_status_label`
  - `password_status_label`
  - `next_action_hint`

同时补了轻量管理员账户操作审计：

- 当前仅记录为服务端日志，不新建审计表
- 记录场景包括：
  - 详情页启停教师账户
  - 单个教师密码重置
  - 批量恢复 / 停用 / 批量重置密码

### 3.2 前端教师管理与账户安全增强

在现有教师管理页基础上做了以下收口：

- 增加账户生命周期摘要卡片
- 增加按关键词、账户状态、密码状态筛选
- 增加批量恢复、批量停用、批量重置密码
- 统一展示：
  - 角色身份
  - 账户状态
  - 密码状态
  - 生命周期提示
- 批量操作后展示最近一次处理结果、临时密码和恢复说明
- 页面显式提示：
  - 当前未来多角色扩展只保留统一入口
  - 不代表复杂多角色平台已经完整上线

密码生命周期面板也做了小幅增强：

- 增加账户状态与密码状态展示
- 账户停用时禁止继续提交改密表单
- 会话失效后的登录页恢复说明更清晰

### 3.3 文档同步

统一现状文档本轮同步了以下内容：

- 接口清单增加：
  - `GET /api/users/teachers/summary/`
  - `POST /api/users/teachers/bulk-actions/`
- 模块边界文档明确：
  - 当前账户生命周期已覆盖启停、恢复、重置、修改和统一提示
  - 当前多角色扩展的正式口径是 `role_code / role_label / permission_scope`
  - 当前管理员账户操作审计仍是轻量日志级别
- 数据口径文档补充：
  - 账户生命周期口径
  - 多角色扩展不是“已完成平台”
  - 生命周期与密码覆盖风险说明

## 4. 测试与验证

### 4.1 新增 / 增强后端测试

在 `backend/users/tests.py` 增加了以下覆盖：

- 当前用户返回生命周期字段
- 管理员可查看教师账户生命周期摘要
- 非管理员不可查看生命周期摘要
- 管理员可批量停用 / 恢复教师账户
- 管理员可批量重置教师密码
- 批量账户操作会跳过管理员账号
- 非管理员不可执行批量账户操作

### 4.2 前端基线补充

在 `frontend/scripts/verify-third-round-ui.mjs` 增加了：

- 账户生命周期提示 helper 校验
- 会话恢复说明 helper 校验

## 5. 实际执行的验证命令

本轮实际执行如下命令：

- `cmd /c npm run type-check`
- `cmd /c npm run verify:baseline`
- `python manage.py check`
- `python manage.py test --keepdb --noinput users`
- `python scripts\run_validation_pipeline.py`

## 6. 最小人工验证步骤

1. 使用管理员账号登录
2. 进入“教师管理”页面
3. 查看页面顶部账户摘要是否显示总量、启用、停用、待修改密码数量
4. 勾选至少两条教师账户记录，分别验证：
   - 批量停用
   - 批量恢复
   - 批量重置密码
5. 检查批量操作结果区是否显示：
   - 处理数量
   - 跳过数量
   - 临时密码（仅重置密码时）
   - 恢复说明
6. 使用教师账号登录个人中心，检查：
   - 角色身份展示
   - 账户状态展示
   - 密码状态展示
   - 账户停用时改密表单不可继续提交

## 7. 本轮边界说明

- 本轮没有改变工号登录主流程
- 本轮没有改变 `sessionStorage` 登录态方案
- 本轮没有引入复杂 RBAC 或多角色平台
- 本轮管理员账户操作审计仍然只是轻量日志记录，不是独立审计系统
- 本轮批量账户操作当前只覆盖教师账户管理范围，不扩展到管理员账户
