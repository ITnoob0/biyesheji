## 3. 按使用场景查阅

### 3.1 开发交接

建议顺序：

1. README
2. 系统接口清单
3. 模块边界与系统最终形态说明
4. 数据口径与风险记录
5. 测试验证与工程化增强

### 3.2 论文写作

建议重点关注：

- 系统最终形态与完整版本路线图
- 模块边界与系统最终形态说明
- 数据口径与风险记录
- 多轮过程文档中的阶段性演进记录

### 3.3 答辩讲解

建议重点关注：

- 最终答辩演示步骤
- 最终验收清单
- 系统最终形态说明
- 接口清单中的主链路接口

## 4. 文档与代码的一致性说明

本轮统一文档以当前代码状态为准，重点对齐了：

- 后端真实路由
- 前端真实页面入口
- 当前已通过验证的功能口径
- 当前仍保留的边界与风险

历史过程文档保留其阶段性描述，不作为唯一的现状依据；需要看“当前状态”时，优先参考本轮统一文档。
