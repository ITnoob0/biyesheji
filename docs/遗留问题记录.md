# 遗留问题记录

最后更新时间：2026-03-18

## 当前说明

本文件仅保留在第三轮开发完成后仍然没有完全做完、但已经明确边界的事项。

以下方向已从遗留问题中移除，因为第三轮已经完成最小可展示落地：

- 推荐结果页的筛选、排序说明、标签分组和管理员查看入口
- 学院级看板的基础筛选、教师钻取和轻量 CSV 导出
- 轻量智能问答演示链路
- 图谱页的轻量图分析亮点展示
- 前端最小准自动化验证脚本

## 当前仍未完全完成的方向

### 1. 前端自动化测试仍不是完整框架化方案

- 现状：第三轮已补 `npm run verify:ui` 准自动化验证脚本，覆盖登录回跳、成果列表更新、图谱回退说明、推荐筛选排序等关键逻辑。
- 仍未完成：尚未接入完整组件级 / 页面级测试框架。
- 后续建议：如项目后续还要继续演进，可再评估是否引入正式前端测试框架。

### 2. 推荐系统仍属于规则增强路线

- 现状：第三轮已把推荐页做成更可解释、可筛选、可展示的亮点能力。
- 仍未完成：尚未进入复杂模型推荐，也没有外部政策语义理解能力。
- 边界说明：当前仍应表述为“规则增强型、可解释推荐”。

### 3. 学院级看板仍是轻量管理分析能力

- 现状：已支持学院 / 年份 / 教师筛选，支持教师画像与推荐结果钻取，也支持导出当前排行。
- 仍未完成：还不是完整 BI 平台，不支持复杂多层级钻取和深度报表体系。
- 边界说明：当前应表述为“轻量学院级科研统计与管理看板”。

### 4. 智能问答仍是单场景演示链路

- 现状：已支持基于真实系统数据生成画像总结、成果概括和推荐原因说明。
- 仍未完成：不支持复杂多轮问答、外部知识检索或通用开放问答。
- 边界说明：当前应表述为“轻量智能问答演示能力”。

### 5. 图谱分析仍是轻量展示增强

- 现状：已补合作热点和主题热点等轻量分析结果。
- 仍未完成：尚未进入社区发现、复杂路径推理、图学习等高级图挖掘阶段。
- 边界说明：当前应表述为“轻量图分析亮点”，不是完整图挖掘平台。

### 6. 扩展表 / 快照 / 缓存层仍未引入

- 现状：当前系统继续以实时聚合为主，没有新增快照表、推荐缓存表或离线聚合表。
- 原因：第三轮没有出现必须引入这些支撑层的真实性能或展示压力。
- 边界说明：当前保持实时聚合是有意选择，不是遗漏。

## 当前建议

如果后续还继续维护本项目，建议按以下顺序推进：

1. 评估是否需要正式前端测试框架
2. 继续增强推荐规则或解释能力
3. 补学院级看板更细粒度管理视角
4. 视实际需要再考虑快照 / 缓存 / 离线聚合
