# 70-教师端全成果审批状态展示与学院管理员审核菜单下线-过程文档

## 1. 背景
- 当前教师端只有论文列表明确展示审批状态，其他成果类型缺少一致的审批状态可视化。
- 学院管理员端仍保留“成果审核”一级菜单，不符合当前菜单收口要求。
- 需要将现有教师成果数据统一置为“已提交待审核”，用于人工验证审批流。

## 2. 目标
- 教师端五类成果列表统一展示审批状态标签（与论文样式一致）。
- 删除学院管理员“成果审核”一级菜单入口（仅菜单层面收口，不改其他已存在审核链路）。
- 将数据库内教师账号下的全部成果批量设置为 `PENDING_REVIEW`。

## 3. 实施内容

### 3.1 教师端成果列表补齐审批状态展示
文件：`frontend/src/views/AchievementEntryView.vue`

- 在以下四类成果表新增“审批状态”列，统一使用 `status + status_label` 和 `resolveAchievementStatusTagType` 渲染标签：
  - 科研项目
  - 知识产权
  - 教学成果
  - 学术服务
- 科研项目表保留业务状态信息，原“状态”列调整为 `project_status`（列名“项目状态”），审批状态单独展示。

### 3.2 删除学院管理员成果审核一级菜单
文件：`frontend/src/router/workspaceRoutes.ts`

- 移除顶级模块路由：
  - `path: 'achievement-review'`
  - `moduleKey: 'achievement-review'`
- 结果：学院管理员侧不再显示“成果审核”一级菜单。

### 3.3 数据库批量设置为待审核
执行命令（一次性）：
- 在 `backend` 目录执行 Django Shell 批量更新教师账号（`is_staff=False and is_superuser=False`）下五类成果状态为 `PENDING_REVIEW`。

执行结果：
- `papers`: 13
- `projects`: 3
- `intellectual_properties`: 4
- `teaching_achievements`: 5
- `academic_services`: 5

复核结果（非待审核数量）：
- `papers_non_pending`: 0
- `projects_non_pending`: 0
- `ips_non_pending`: 0
- `teaching_non_pending`: 0
- `services_non_pending`: 0

## 4. 验证结果
- 前端：`npm run type-check` 通过
- 前端：`npm run build` 通过
- 后端：`python manage.py check` 通过

## 5. 结论
- 教师端五类成果在“成果列表/录入”中已统一可见审批状态。
- 学院管理员“成果审核”一级菜单已下线。
- 当前数据库中教师成果已统一为“提交待审核”，可直接用于你后续的手工审批流测试。
