# 80-项目指南生命周期管理页独立化与时间线筛选-过程文档

## 1. 需求背景
- 原“生命周期管理”二级菜单与“指南总览”展示几乎一致，缺少独立治理价值。
- 本次目标：在保持现有 RBAC 与项目指南架构不重构的前提下，将生命周期管理改造成真实治理页，聚焦：
  - 状态流转管理（归档 / 重新开放）
  - 时间线筛选（截止、更新时间、发布时间、归档时间）
  - 生命周期统计（含超期未归档）

## 2. 后端改动

### 2.1 `backend/project_guides/views.py`
- 在 `ProjectGuideViewSet.get_queryset` 中新增日期区间筛选支持：
  - `deadline_from`, `deadline_to`
  - `updated_from`, `updated_to`
  - `published_from`, `published_to`
  - `archived_from`, `archived_to`
- 新增内部方法：
  - `_parse_iso_date_or_raise`
  - `_to_day_start`
  - `_to_day_end`
  - `_apply_date_range_filters`
- 日期参数非法时返回 `400`，并带字段级错误提示（`YYYY-MM-DD`）。

### 2.2 `backend/project_guides/services.py`
- `build_lifecycle_summary` 新增指标：
  - `overdue_active_count`（超期未归档）
  - 统计口径：`ACTIVE/URGENT/OPEN` 且 `application_deadline < today`

### 2.3 后端测试增强 `backend/project_guides/tests.py`
- 新增测试：
  - `test_lifecycle_filters_by_deadline_and_updated_at`
    - 验证截止时间 + 更新时间组合筛选生效
  - `test_lifecycle_summary_contains_overdue_active_count`
    - 验证生命周期摘要返回 `overdue_active_count`

## 3. 前端改动

### 3.1 类型定义 `frontend/src/types/projectGuides.ts`
- `GuideLifecycleSummary` 新增字段：
  - `overdue_active_count: number`

### 3.2 页面改造 `frontend/src/views/ProjectGuideManagementView.vue`
- 将 `sectionMode='lifecycle'` 从总览复用模式拆分为独立生命周期视图：
  - 顶部时间筛选条（截止 / 更新 / 发布 / 归档四组 daterange）
  - 生命周期专属统计卡（总数、治理中活跃、近30天到期、超期未归档）
  - 生命周期专属列表列（标题、范围、状态、截止、发布、归档、更新时间、生命周期说明）
  - 操作列收口为治理动作：
    - `查看详情`
    - `归档` / `重新开放`
  - 不再在生命周期页展示推荐运营动作（如定向推送、删除、编辑）
- 新增交互方法：
  - `resetLifecycleFilters`
  - `showLifecycleDetail`

## 5. 结果结论
- 生命周期管理二级菜单已从“总览复用壳”升级为可用治理页。

- 操作收口到生命周期治理动作，页面职责边界与“指南总览”已明确分离。
