# 画像首页多成果联动增强 过程文档

## 1. 任务背景
- 目标：在不推翻现有画像页结构的前提下，让教师科研画像首页不再主要依赖论文数据，而是联动：
  - `Project`
  - `IntellectualProperty`
  - `TeachingAchievement`
  - `AcademicService`
- 保留项：
  - 基础信息区
  - 统计卡片
  - 雷达图
  - 论文趋势图
  - 关键词画像
  - 合作画像
  - 拓扑图
- 约束：
  - 不引入大规模画像重算任务系统
  - 不设计复杂画像快照体系
  - 管理员查看指定教师画像能力不能被破坏
  - 无 Neo4j 时页面主体仍须正常展示

## 2. 改造前数据流梳理
### 前端已有调用
- 教师基础档案：
  - `GET /api/users/teachers/:id/`
- 统计卡片：
  - `GET /api/achievements/dashboard-stats/`
- 雷达数据：
  - `GET /api/achievements/radar/:id/`
- 论文列表：
  - `GET /api/achievements/papers/`
- 拓扑图：
  - `GET /api/graph/topology/:id/`

### 原有职责划分
- 后端负责：
  - 基础统计卡片
  - 雷达维度
- 前端负责：
  - 关键词画像
  - 合作画像
  - 论文趋势图
  - 论文类型分布

### 原有问题
- 统计卡片仍偏论文视角。
- 雷达图虽然已经部分使用多成果信息，但没有“来源说明”。
- 代表性成果区仍主要展示论文。
- 页面没有明确的数据更新时间与来源说明。

## 3. 最小可行改造方案
- 不新开复杂聚合体系。
- 继续复用：
  - [views.py](/e:/Project/bishe/backend/achievements/views.py) 中的 `dashboard-stats`
  - [views.py](/e:/Project/bishe/backend/achievements/views.py) 中的 `radar`
- 后端补充轻量聚合返回：
  - 多成果概览
  - 最近成果列表
  - 维度来源说明
  - 数据更新时间/来源说明
- 前端保留现有布局，只增强数据展示内容。

## 4. 实现过程
### 4.1 后端画像聚合增强
- 重写画像评分聚合逻辑：
  - [scoring_engine.py](/e:/Project/bishe/backend/achievements/scoring_engine.py)
- 聚合依据改为真实关系库数据：
  - 论文数、被引数
  - 项目数、经费
  - 知识产权数、转化数
  - 教学成果数
  - 学术服务数
  - 关键词数
  - 合作者数

### 4.2 现有接口增强
- 增强 `GET /api/achievements/dashboard-stats/`
  - 返回 `statistics`
  - 返回 `achievement_overview`
  - 返回 `recent_achievements`
  - 返回 `data_meta`
- 增强 `GET /api/achievements/radar/:id/`
  - 返回 `radar_dimensions`
  - 返回 `dimension_sources`
  - 返回 `data_meta`

### 4.3 前端画像页增强
- 新增前端画像计算辅助模块：
  - [portrait.ts](/e:/Project/bishe/frontend/src/views/dashboard/portrait.ts)
- 重建画像首页实现：
  - [DashboardView.vue](/e:/Project/bishe/frontend/src/views/DashboardView.vue)

### 4.4 页面增强点
- 统计卡片：
  - 总成果数
  - 论文成果
  - 项目与知识产权
  - 教学与学术服务
  - 综合画像评分
- 雷达区：
  - 保留雷达图
  - 增加每个能力维度的来源说明
- 代表性成果区：
  - 从“最近论文”改为“多成果混合最近成果”
- 说明区：
  - 数据更新时间
  - 数据来源说明
  - 当前阶段验收说明

## 8. 验证过程
### 自动化验证
- `npm run type-check`
- `python manage.py test achievements --keepdb`
- `python manage.py check`

### 新增验证点
- `dashboard-stats` 返回多成果概览
- `dashboard-stats` 返回最近成果混合列表
- `dashboard-stats` 返回数据说明
- `radar` 返回维度来源说明
- 管理员可通过 `user_id` 查看指定教师画像聚合

## 9. 手工验证建议
1. 使用教师账号登录进入 `/dashboard`
2. 检查统计卡片是否体现多成果结构
3. 检查雷达图下方是否出现维度来源说明
4. 检查“代表性成果”区是否出现项目、知识产权、教学成果、学术服务
5. 检查“画像说明”区是否显示数据更新时间和来源说明
6. 使用管理员进入 `/profile/:id` 查看指定教师画像
7. 在 Neo4j 关闭情况下确认页面主体仍正常展示

## 10. 结果与取舍
### 已达成
- 画像页已能体现多成果类型，不再只从论文视角理解教师能力
- 原有论文趋势、关键词、合作画像仍保持可用
- 管理员查看指定教师画像能力未破坏
- 无 Neo4j 情况下页面主体仍正常可用

### 主动未做
- 未引入画像快照表
- 未引入后台重算任务系统
- 未把关键词画像和合作画像扩展到非论文成果
- 未把趋势图扩展成完整多成果时间轴

## 11. 风险与遗留问题
- 关键词、合作画像、趋势图仍主要依赖论文数据，这一取舍是为保证当前一期结构稳定。
- 前端与后端部分旧文件仍存在中文编码噪音，后续维护时需要继续清理。

## 12. 是否纳入当前阶段验收
- 结论：`纳入当前阶段验收`
- 原因：该任务直接增强了当前一期教师画像主页主链路。
