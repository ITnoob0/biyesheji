# 48-全局布局重构-过程文档

## 1. 本次改了什么

本次前端布局从原先 `App.vue` 内嵌的“顶部扁平工作台壳”重构为“左侧主导航 + 顶部上下文工具栏 + 右侧独立内容区”的现代 B 端布局。

重构范围坚持轻量增量原则：

- 不引入 Vben Admin 或其他后台模板
- 不改后端接口与页面业务逻辑
- 不改 `sessionStorage` 登录态
- 不改现有守卫语义
- 不做 tags-view、keep-alive、多页签

## 2. 路由如何重组

### 2.1 认证页继续独立

以下页面仍然不进入主布局：

- `/login`
- `/register`
- `/forgot-password`

### 2.2 工作台页挂到 `MainLayout` 下

新增：

- `frontend/src/layout/MainLayout.vue`

并将原有工作台路由重组为 `MainLayout` 的 children，URL 保持不变：

- `/dashboard`
- `/profile/:id`
- `/profile-editor`
- `/entry`
- `/project-recommendations`
- `/assistant-demo`
- `/teachers`
- `/academy-dashboard`
- `/project-guides`

根路径 `/` 仍根据 `workspaceNavigation.js` 解析默认首页，当前教师和管理员都保持落到画像主页 `/dashboard`。

## 3. 左侧菜单如何生成

菜单不走后端驱动，也没有引入复杂动态菜单平台，而是基于前端静态路由 `meta` 自动生成。

新增文件：

- `frontend/src/router/workspaceRoutes.ts`
- `frontend/src/router/route-meta.d.ts`
- `frontend/src/layout/layoutMenu.ts`

当前路由 `meta` 统一使用以下字段：

- `title`
- `icon`
- `order`
- `section`
- `hiddenInMenu`
- `activeMenu`
- `requiresAuth`
- `requiresAdmin`
- `guestOnly`
- `menuRoles`

其中：

- `hiddenInMenu` 用于隐藏详情路由，例如 `/profile/:id`
- `activeMenu` 用于详情页高亮主菜单，例如 `/profile/:id` 高亮 `/dashboard`
- `menuRoles` 用于控制菜单可见性，但不重写原有路由访问语义

## 4. 权限如何过滤菜单

当前菜单过滤遵循已有教师 / 管理员边界：

- 管理员可见：
  - 画像主页
  - 个人中心
  - 项目推荐
  - 智能问答
  - 教师管理
  - 学院看板
  - 项目指南管理
- 教师可见：
  - 画像主页
  - 个人中心
  - 成果管理
  - 项目推荐
  - 智能问答

说明：

- `成果管理` 当前只在教师侧菜单显示
- `/profile/:id` 继续作为隐藏详情页存在
- 管理员专属页面继续依赖现有 `requiresAdmin` 守卫

## 5. 哪些旧 URL 被兼容保留

本次没有大面积改 URL，原有外部可访问路径保持兼容：

- `/dashboard`
- `/profile/:id`
- `/profile-editor`
- `/entry`
- `/project-recommendations`
- `/assistant-demo`
- `/teachers`
- `/academy-dashboard`
- `/project-guides`

只是这些路由现在通过 `MainLayout` 作为统一工作台壳承载。

## 6. 图表 resize 如何处理

由于主布局改为“左侧导航 + 右侧独立内容区”，图表不能再只依赖 `window.resize`。

本次新增：

- `frontend/src/utils/resizeObserver.ts`

并把以下图表改为 `ResizeObserver` 驱动：

- `frontend/src/components/RadarChart.vue`
- `frontend/src/views/DashboardView.vue`
- `frontend/src/views/AcademyDashboardView.vue`
- `frontend/src/views/AcademicGraph.vue`

处理原则：

- 不动图表数据逻辑
- 只替换 resize 监听策略
- 组件卸载时正确断开 observer 并释放图表实例

## 7. 页面壳兼容与滚动修正

为适配新布局，做了最小必要样式修正：

- `App.vue` 清理为纯 `router-view`
- `workspace-system.css` 收口全局高度与基础字体
- 工作台页面根节点由 `min-height: 100vh` 改为 `min-height: 100%`
- 右侧内容区改为独立 `overflow-y: auto`

已修正的典型问题：

- 旧壳头部留白被移除
- 避免 body 与内容区双滚动
- 页面根容器不再强占整个视口高度

## 8. 当前未做的事项

本次明确没有实现以下能力：

- tags-view
- keep-alive 页面缓存
- 多页签历史
- 左侧折叠侧栏
- 顶部一级导航驱动左侧二级导航
- 后端驱动菜单
- 复杂面包屑配置平台

## 9. 验证建议

建议执行：

1. `cd frontend && npm run type-check`
2. `cd frontend && npm run verify:ui`
3. `cd frontend && npm run verify:baseline`

建议人工检查：

1. 教师登录后是否进入 `/dashboard`
2. 左侧菜单是否按教师 / 管理员身份变化
3. 切换到画像、学院看板、推荐页后图表是否在新布局宽度下正常渲染
4. `profile/:id` 是否仍能正常访问并高亮画像主页菜单
