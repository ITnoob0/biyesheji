# V1.1 工程收口与验收基线过程文档

## 任务背景

当前项目版本为 `V1.1-portrait-baseline`。一期主链路已经具备以下能力：

- 认证与会话
- 教师基础信息维护
- 五类成果录入中的论文主链路
- 教师画像展示
- 学术关系拓扑展示

本轮任务不是继续铺新功能，而是对“已实现系统 + 最近增量功能”做一次工程收口，使其更适合：

- 后续持续开发
- agent 增量协作
- 课程答辩演示
- 验收边界说明

本轮不改变以下硬约束：

- 认证状态仍使用 `sessionStorage`
- 教师账号仍使用工号登录
- Neo4j 仍然不是论文录入和画像展示的强依赖
- 所有页面继续受现有路由守卫约束

## 本轮目标

1. 扫描并确认前后端测试、校验、类型与文档基线。
2. 为最近新增/改造模块补齐最小必要测试。
3. 补齐前端共享 API 类型定义，减少页面内联接口类型。
4. 同步更新接口清单、实现边界、验收口径、演示步骤。
5. 输出一份可直接用于验收的检查清单。

## 本轮改动范围

### 一、后端测试补强

补充了以下关键回归测试：

- `backend/users/tests.py`
  - 非管理员不能查看其他教师详情

- `backend/achievements/tests.py`

- `backend/graph_engine/tests.py`
  - 普通教师不能访问其他教师的拓扑图

这些测试都属于当前阶段验收的工程保障项，纳入本轮验收。

## 当前真实接口清单

以下清单以当前代码为准。

- `POST /api/token/`

- `POST /api/token/refresh/`

## 已实现 / 部分实现 / 未实现边界

### 已实现

- 工号登录、会话校验与路由守卫
- 教师自助注册
- 找回密码
- 教师个人基础档案编辑
- 管理员教师管理
- 论文录入、查询、删除
- 项目/知识产权/教学成果/学术服务录入、查询、删除
- 教师画像首页聚合展示
- 学术拓扑图展示
- BibTeX 论文批量导入
- 项目指南管理
- 项目指南推荐
- 学院级管理看板

### 部分实现

- 五类成果后端都支持 `PATCH`，但前端成果中心仍以“新增 + 删除”为主，尚未补完整编辑 UI
- JWT 提供了 refresh 接口，但前端当前主链路仍以当前会话 access token 为主
- 图谱同时存在增量同步与全量重建两条维护路径，功能可用，但工程一致性仍需后续继续收口
- 推荐系统和学院看板已可运行，但不属于当前一期硬验收主链路

### 未实现

- PDF 元数据导入
- RAG / 智能问答正式链路
- 复杂图分析与图算法能力
- 推荐结果持久化、反馈闭环
- 前端专用单元测试框架
- `ai_assistant` 对外正式 API

## 当前验收口径

### 纳入当前阶段验收

- 一期主链路功能稳定性
- 后端关键权限与查询链路回归测试
- 前端共享 API 类型收口
- 文档与接口状态同步
- Neo4j 非强依赖前提下的图谱回退能力

### 不纳入当前阶段硬验收

- 推荐系统算法深度优化
- 项目指南自动采集
- RAG / 问答能力
- 复杂 BI 与学院级钻取分析
- PDF 全自动解析导入

## 演示步骤

### 教师主链路演示

1. 打开登录页，使用工号登录教师账号。
2. 进入教师画像首页，展示基础档案、统计卡片、雷达图、近期成果和拓扑图。
3. 进入“基础档案编辑”，修改研究兴趣与简介，保存后返回画像页验证联动。
4. 进入“成果录入中心”，新增一条论文或其他成果。
5. 返回画像页，验证统计、近期成果、关键词/合作信息是否同步刷新。
6. 打开学术拓扑图，展示节点、关系说明与空态/失败态兼容逻辑。

### 管理员链路演示

1. 使用管理员账号登录。
2. 进入教师管理页，创建教师账号，确认登录账号仍为工号。
3. 编辑教师资料，确认教师详情与画像信息可被管理员查看。
4. 打开指定教师画像页，验证管理员可查看指定教师的画像数据与论文列表。
5. 进入项目指南管理页，新增一条 `OPEN` 指南。
6. 打开项目指南推荐页，验证管理员可通过指定教师视角查看推荐结果。
7. 进入学院级管理看板，展示统计卡片、趋势图、院系统计与活跃教师排行。
