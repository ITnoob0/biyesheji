# 56-学院管理员角色扩展与学院级权限收口-过程文档

## 目标

在既有 `TEACHER`、`SYSTEM_ADMIN` 基础上，补充 `COLLEGE_ADMIN` 角色，并明确学院级与全校级权限边界。

## 本次收口

- 新增角色常量与识别逻辑：`COLLEGE_ADMIN`
- 学院管理员继承教师本人自助能力
- 学院管理员新增“仅本学院教师可见”的管理视角
- 系统管理员保留全校教师与学院管理员的全局可见范围
- 前后端统一按学院范围控制教师相关列表、详情与菜单入口

## 范围说明

- `SYSTEM_ADMIN`：全校范围
- `COLLEGE_ADMIN`：本学院范围
- `TEACHER`：本人范围

## 验证

- 前端类型检查通过
- Django `check` 通过
- `users` 相关测试通过
