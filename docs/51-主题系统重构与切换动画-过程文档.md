# 51-主题系统重构与切换动画-过程文档

## 1. 本轮目标

本轮主题改造的目标不是局部换色，而是把当前系统从“Layout 已切、内容未全切、动画弱”的状态，提升为：

- light / dark 双主题由统一语义 token 驱动
- Layout、内容区、卡片、表单、弹层、图表统一跟随主题
- 主题切换时从右上角主题按钮附近发起 reveal 动画
- 保留当前业务逻辑、路由守卫、sessionStorage 登录态和角色边界不变

## 2. 当前问题与修复方向

### 2.1 录屏 3 暴露的问题

- 外壳切暗后，内容页仍残留大量固定白卡片
- 多个页面仍直接写死 `#f8fafc`、`#fff`、深色标题字和浅色边框
- Element Plus 表单、表格、弹层、输入框没有完整跟随主题
- 图表仍以浅色轴线、分割线和 tooltip 为主
- 主题切换缺少从按钮 origin 发起的过程感

### 2.2 本轮修复策略

- 统一 light / dark token
- 全局覆盖 Element Plus 高频组件
- 修复高频页面的硬编码浅色面板
- 为图表增加主题感知和主题变更后重绘
- 落地 reveal 式切换动画和降级策略

## 4. 切换动画说明

本轮动画采用轻量 reveal 方案：

- 在 `body` 上注入全屏遮罩层
- 切换时以主题按钮中心点作为 `clip-path: circle()` origin
- 新主题先应用，再由遮罩层做径向扩展覆盖
- 时长约 620ms
- 若浏览器 / 用户偏好不适合大动画，则退化为直接切换

## 8. 最小人工验证步骤

1. 登录进入工作台
2. 依次打开：
   - 个人中心
   - 教师画像
   - 成果中心
   - 项目推荐
   - 智能问答
   - 学术图谱
   - 学院看板
3. 在 light 模式检查：
   - sidebar
   - header
   - breadcrumb / 页面标题
   - hero
   - card / panel / meta block
   - input / select / textarea
   - dialog / drawer / dropdown
   - empty state
   - charts
4. 点击右上角主题按钮切到 dark
5. 确认 reveal 动画从右上角主题按钮附近扩散
6. 检查上述区域是否仍存在突兀白卡片、错误文字色、浅色边框残留
7. 再切回 light，确认动画、滚动和布局无异常
