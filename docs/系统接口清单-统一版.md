# 系统接口清单-统一版

最后更新时间：2026-03-22

## 1. 说明

本清单以当前 Django 路由配置为准，来源于：

- `backend/core/urls.py`
- `backend/users/urls.py`
- `backend/achievements/urls.py`
- `backend/project_guides/urls.py`
- `backend/graph_engine/urls.py`
- `backend/ai_assistant/urls.py`

当前接口体系主要面向：

- 认证与账户
- 成果管理
- 画像与看板
- 项目推荐
- 学术图谱
- 智能问答

### 1.1 维护规则

以下情况发生时，必须同步更新本清单：

- 新增或删除接口
- 请求参数变化
- 返回结构变化
- 权限边界变化
- 演示主链路依赖的关键接口变化

若接口行为与历史过程文档表述不一致，以当前代码路由和本清单为准。

## 2. 认证与账户接口

### 鉴权

- `POST /api/token/`
  - 工号/管理员账号登录
  - 返回 JWT access / refresh
- `POST /api/token/refresh/`
  - 刷新 access token

### 当前用户与账户安全

- `GET /api/users/me/`
  - 获取当前用户信息
  - 返回当前角色能力摘要 `permission_scope`
- `PATCH /api/users/me/`
  - 更新当前用户资料与教师档案联动字段
- `POST /api/users/me/change-password/`
  - 当前用户修改密码

### 注册与找回

- `POST /api/users/register/`
  - 教师自助注册
- `POST /api/users/forgot-password/`
  - 教师忘记密码重置

### 教师账户管理

- `GET /api/users/teachers/`
  - 管理员查看教师列表
  - 教师仅返回自己
- `POST /api/users/teachers/`
  - 管理员创建教师账户
- `GET /api/users/teachers/<user_id>/`
  - 查看教师详情
  - 教师仅可查看自己
  - 管理员当前仅可查看教师账户，不把管理员账户纳入教师管理详情
- `PATCH /api/users/teachers/<user_id>/`
  - 管理员更新教师详情
- `POST /api/users/teachers/<user_id>/reset-password/`
  - 管理员重置教师密码

## 3. 成果管理接口

### 论文

- `GET /api/achievements/papers/`
  - 论文列表
  - 支持教师隔离、管理员按 `teacher_id` 查看
  - 支持搜索、年份筛选、代表作筛选、排序
- `POST /api/achievements/papers/`
  - 新增论文
  - 仅教师本人可通过该入口维护
- `GET /api/achievements/papers/<id>/`
- `PATCH /api/achievements/papers/<id>/`
- `DELETE /api/achievements/papers/<id>/`
  - 当前管理员可按 `teacher_id` 查看成果，但不通过教师自助成果入口代录、代改或代删

### 论文导入与统计

- `POST /api/achievements/papers/import/bibtex-preview/`
  - BibTeX 预览与重复提示
- `POST /api/achievements/papers/import/bibtex-confirm/`
  - BibTeX 确认导入
- `GET /api/achievements/papers/summary/`
  - 论文统计摘要

### 其他成果

- `GET/POST/PATCH/DELETE /api/achievements/projects/`
- `GET/POST/PATCH/DELETE /api/achievements/intellectual-properties/`
- `GET/POST/PATCH/DELETE /api/achievements/teaching-achievements/`
- `GET/POST/PATCH/DELETE /api/achievements/academic-services/`
  - 当前均沿用“教师本人维护、管理员查看验证”的边界

## 4. 画像与看板接口

### 教师画像

- `GET /api/achievements/dashboard-stats/`
  - 当前教师画像主页聚合
  - 管理员支持 `user_id`
- `GET /api/achievements/radar/<user_id>/`
  - 教师画像雷达与维度来源

### 学院看板

- `GET /api/achievements/academy-overview/`
  - 管理员学院级看板
  - 支持学院、年份、教师、职称、合作等筛选

## 5. 推荐接口

### 项目指南管理

- `GET /api/project-guides/`
  - 教师仅看开放指南
  - 管理员可按状态和搜索查看
- `POST /api/project-guides/`
  - 管理员新增指南
- `GET /api/project-guides/<id>/`
- `PATCH /api/project-guides/<id>/`
- `DELETE /api/project-guides/<id>/`

### 项目推荐

- `GET /api/project-guides/recommendations/`
  - 教师获取自己的推荐结果
  - 管理员支持 `user_id`
  - 管理员支持 `compare_user_id`
  - 返回推荐列表、标签、解释维度、管理员分析摘要

## 6. 图谱接口

- `GET /api/graph/topology/<user_id>/`
  - 获取教师学术图谱
  - 教师仅可查看自己，管理员可查看指定教师
  - Neo4j 可用时优先读取图谱
  - Neo4j 不可用时回退到 MySQL
  - 返回节点、边、元信息和轻量图分析结果

## 7. 智能问答接口

- `POST /api/ai-assistant/portrait-qa/`
  - 受控模板问答入口
  - 教师仅可查看自己的问答结果
  - 管理员可查看指定教师问答结果，并可使用学院统计问答
  - 当前支持：
    - `portrait_summary`
    - `portrait_dimension_reason`
    - `achievement_summary`
    - `guide_overview`
    - `guide_reason`
    - `academy_summary`

## 8. 接口边界说明

### 当前已完成

- 主链路接口均已形成稳定路径
- 教师与管理员权限边界基本明确
- 登录后核心页面依赖的主接口都已有测试或回归脚本覆盖

### 当前未完成

- 没有额外的开放 API 文档站点
- 没有版本化 API 管理
- 没有 Swagger / OpenAPI 自动文档

### 后续建议

若后续继续工程化增强，优先顺序建议如下：

1. 为当前清单补参数说明和示例响应
2. 再考虑 OpenAPI 自动生成
3. 若接口稳定后再讨论版本化治理
