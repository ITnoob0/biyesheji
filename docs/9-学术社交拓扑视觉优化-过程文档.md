# 学术社交拓扑视觉优化-过程文档

## 任务背景

学术社交拓扑图在上一轮交互增强后，已经具备：

- 点击节点查看详情
- 点击边查看关系说明
- Neo4j 不可用时自动回退 MySQL
- 图谱异常不拖垮整个画像页

但页面布局上仍有两个问题：

- 节点类型隐藏能力被移除后，缺少教师自行收敛视图的手段
- 图谱图例放在右侧信息栏，会让这一区域显得偏长、偏窄

因此本轮目标是做一次“视觉与布局优化”，不改动主链路，不强化 Neo4j 依赖。

## 设计原则

本轮保持以下约束不变：

- `topology` 接口不改
- Neo4j / MySQL 双路径继续可用
- 图谱失败仍然只影响图谱区域
- 前端增强不能依赖 Neo4j 独有字段

## 实施内容

### 1. 恢复节点隐藏按钮

在图谱画布下方新增了“隐藏节点”控制区，允许对以下类型节点做开关：

- 论文
- 合作学者
- 关键词
- 项目
- 知识产权
- 教学成果
- 学术服务

中心教师节点默认始终保留，不进入隐藏列表。这样可以避免用户误把图谱主体一起关掉。

### 2. 调整控制区位置

原本图谱相关说明更多挤在右侧信息栏。  
本轮把与“图谱本体”直接相关的控制元素下沉到图谱下方：

- 先放“隐藏节点”按钮
- 再放“图谱图例”

这样做的原因是：

- 控制项与图谱本身更接近，操作更直觉
- 右侧信息栏可以只保留“关系速览”和“当前焦点”
- 整体版面不再显得右侧特别狭长

### 3. 重构图谱画布区域

为了让空状态、失败状态与图谱底部控制区互不干扰，本轮将图谱区拆成：

- `chart-area`
- `state-layer`
- `graph-footer`

这样在加载失败或空数据时，覆盖层只作用于图谱画布，不会把底部控制区也覆盖掉。

## 修改文件

- `frontend/src/views/AcademicGraph.vue`

## 风险控制

本轮没有改动：

- 后端 `topology` 接口结构
- Neo4j 回退逻辑
- 教师画像页其它模块

因此风险主要控制在前端展示层。

## 验证方式

执行：

```bash
npm run type-check
```

手工验证：

1. 打开教师画像页中的学术社交拓扑图
2. 确认图谱下方出现“隐藏节点”按钮组
3. 点击任一类型按钮后，确认对应节点与相关边被隐藏
4. 再次点击同一按钮，确认节点恢复显示
5. 确认“图谱图例”位于隐藏按钮下方，而不是右侧
6. 确认右侧区域只保留“关系速览”和“当前焦点”
7. 确认 Neo4j 关闭时，MySQL 回退链路下这些按钮仍然可用

## 结果

本轮完成后，学术社交拓扑图的视觉结构更清晰：

- 图谱相关操作回到图谱下方
- 图例说明位置更合理
- 用户重新获得按类型隐藏节点的能力
- 不破坏当前画像页和回退链路稳定性
