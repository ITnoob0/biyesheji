                       摘要


  在全国多地旱灾不断的现状下，面对着传统灌溉效率低和水资源稀少珍贵的
情况下，同样沙地马铃薯种植中也有水资源利用率低、传统灌溉粗放等问题。现
今，国家推动智慧灌溉等现代化农业技术已成趋势，智慧灌溉技术蓬勃发展。
  本文针对沙地马铃薯智慧灌溉决策功能方面，设计并实现了一种基于 Vue.js
的 web 前端框架和 Spring Boot 和 Mybatis 的后端框架的智慧灌溉决策系统。根据
沙地马铃薯智慧灌溉的主要操作和数据处理，对灌溉相关数据的获得和处理和可
视化进行详细分析，将系统结构主要分为实时监测信息显示、灌溉计划管理、天
气信息查询、智慧决策和用户信息管理。实现土壤温湿度、灌溉状态等数据的可
视化；重点分析了灌溉决策算法，其集成了动态补偿算法，结合实时传感器数据
与气象预测信息，动态生成灌溉决策。
  系统创新性地融合沙地马铃薯生长需求与气象数据等多方面数据的优化算法，
并通过模块化设计支持功能扩展，可以一定程度上节省水资源和灌溉费用。

关键词：沙地马铃薯；智慧灌溉；灌溉决策算法；Spring Boot；Vue.js

                                        ABSTRACT


      Under the current situation of continuous drought in many parts of the country,
facing the situation of low efficiency of traditional irrigation and scarce and precious
water resources, there are also problems such as low utilization rate of water resources
and extensive traditional irrigation in potato planting in sandy land.nowadays, it has
become a trend for the state to promote modern agricultural technologies such as smart
irrigation, and smart irrigation technology is booming.
      This paper designs and implements a smart irrigation decision system based on
vue.js web front-end framework and spring boot and mybatis back-end framework.
According to the main operation and data processing of potato intelligent irrigation in
sandy land, the acquisition, processing and visualization of irrigation related data were
analyzed in detail. The system structure was mainly divided into real-time monitoring
information display, irrigation plan management, weather information query, intelligent
decision-making and user information management. Realize the visualization of soil
temperature, humidity, irrigation status and other data; This paper focuses on the
analysis of the irrigation decision algorithm, which integrates the dynamic
compensation algorithm and combines the real-time sensor data and meteorological
prediction information to dynamically generate the irrigation decision.
      The optimization algorithm of systematically and innovatively integrating the
growth demand of potato in sandy land with meteorological data and other data, and
through modular design support function expansion, can save water resources and
irrigation costs to a certain extent.


Key words: Potato in sandy land; Intelligent irrigation; Irrigation decision algorithm;
Spring Boot；Vue.js

                     1 绪论


1.1 选题的背景及意义

  智慧灌溉作为一个现代化农业技术，可以在提升水资源综合利用率的同时,
有效提升粮食产量, 实现按照农作物类型进行精准化灌溉。因此，智慧灌溉系统
应运而生，为农民提供了智能、高效的灌溉方式和更多实际收益[1]。
  本文研究的沙地马铃薯智慧灌溉决策系统可以简单并有效的实现智慧灌溉的
目的，本设计目的在于开发一个基于 SpringBoot+Vue.js 和 MySQL 技术，通过实
时分析土壤湿度、环境温湿度等参数，结合智慧灌溉模型，实现精准灌溉决策。
系统采用前后端分离架构，前端负责数据可视化与用户交互，后端实现数据处理
与决策分析。基本要求是灌溉决策模型响应时间≤3 秒。前端界面简洁美观，后端
接口高效稳定。
  意义：本系统通过模拟温湿度传感器与简单决策算法，为沙地马铃薯种植提
供定制化灌溉解决方案，降低水资源浪费，提升作物产量。系统还可扩展至其他
干旱地区，推动农业智能化转型，助力乡村振兴。

1.2 国内外发展研究现状

1.2.1 国内发展研究现状

  国内智慧灌溉技术近些年来在政策扶持以及技术不断迭代这两个因素的推动
之下发展得颇为迅速。《数字农业农村发展规划(2019—2025年)》明确指出，推进
数字产业化，提升数字经济在农业产业中的应用[4]，促使智慧农业可更快地落地实
施，像新疆、山东等农业大省率先展开了规模化的试点工。中国农业大学研发的
基于LoRa的土壤墒情监测系统，可实时收集多维度的环境数据，结合模糊控制算
法生成灌溉策略，并且已经在华北小麦种植区验证了其有效性[5]。不过，国内技术
的普及依旧面临着多种挑战：其一，中小型农户受到硬件成本以及技术复杂程度
的限制，推广起来难度比较大，其二，现有的系统大多集中于单一作物，对于跨
作物的适配性以及沙地、干旱等特殊场景的针对性有所欠缺，其三，农业文化遗
产的数字化保护还没有和智能灌溉进行深度的融合，传统经验没能有效地转化为
可以复用的决策参数。而且传感器数据融合的精度、算法的实时性以及边缘计算
能力仍然需要进行优化，以此来契合复杂田间环境的稳定性要求， 和国外成熟的
技术生态相比较而言，国内的研究更加注重低成本以及本地化适配，但是在算法
鲁棒性以及多源数据融合方面还是存在差距的。

1.2.2 国外发展研究现状

   在国外农业技术较为发达的国家，智慧灌溉系统已经形成了一套相对成熟的
技术框架。以色列作为这一领域的先驱，其滴灌技术通过高精度的传感器和动态
决策算法的应用，实现了水资源利用率超过 90%的重大突破[2]。例如，Netafim 公
司所开发的智能滴灌解决方案能够根据实时土壤湿度、气象信息以及作物生长的
具体阶段，灵活调整灌溉策略，从而大幅度减少了水资源的浪费。而在美国，科
研机构在推动相关技术进步方面发挥了重要作用，比如加州大学戴维斯分校研制
的 CIMIS（California Irrigation Management Information System）平台，它集成了来
自气象站的数据、土壤传感网络与作物需水量预测模型，为农户提供了科学合理
的灌溉指导，并已在加利福尼亚州的农业生产中得到了广泛应用[3]。荷兰则充分利
用了其在设施农业方面的优势，将物联网与温室环境管理相结合，发展出了基于
人工智能的灌溉预测工具，该工具通过对历史数据的学习来优化灌溉时间表，进
一步提升了农作物产量及资源利用效率。



1.3 本设计的主要研究内容

   来自国内外的研究都说明智慧灌溉系统的潜力无限意义深远，本设计聚焦沙
地马铃薯种植场景，设计采用 B/S 框架设计一个沙地马铃薯智慧灌溉决策系统。
系统采用前后端分离架构，前端依托 Vue.js 框架构建动态交互界面，实现数据可
视化与用户操作；后端基于 Spring Boot 框架开发，集成简单灌溉决策算法构建灌
溉模型，结合模拟实时采集的土壤温湿度信息、创新点在于结合气象数据及作物
生长需求，动态生成灌溉策略。以下是研究的主要内容：
   （1）实时信息模块：用 Python 程序模拟土壤湿度传感器、温度传感器采集数
据并在前端动态展示土壤温湿度、灌溉模式及设备状态，数据通过 ECharts 可视化
呈现。
   （2）智能决策模块：基于沙地马铃薯生长需求（温度 18~25℃、湿度 60~80%
RH），融合动态补偿算法与气象预测数据，优化灌溉时长与用水量，采用滞后控
制策略减少水泵频繁启停，动态调整灌溉策略。
   （3）灌溉计划管理：用户可以手动设置灌溉参数，也可以选择系统的自动灌
溉功能，系统自动获得用户最新设置的灌溉计划管理的参数传入智慧决策算法里。
还提供对灌溉计划管理中添加的灌溉计划的增删改查。
   （4）天气信息：用户可以查询自己所在城市的天气实时信息和 24 小时天气
预报信息。

2.1 可行性分析

  智慧灌溉决策系统的研发需求源于传统灌溉模式在干旱环境下面临的水资源
浪费严重、作物生长调控粗放等痛点[6]。本设计基于“数据驱动精准调控”的设计
理念，通过技术架构可靠性验证、经济成本效益测算及用户操作友好性评估三维
度，系统性论证方案落地实施的可行性。

2.1.1 技术可行性

  本系统采用成熟的前后端分离架构（Spring Boot + Vue.js）与模块化设计，技
术实现路径清晰：
  后端技术栈：基于 Spring Boot 框架，结合 MyBatis-Plus 优化数据库操作效
率。
  前端技术栈：采用 Vue.js 实现响应式界面，通过 ECharts 完成数据可视化，
并利用 Axios 实现前后端异步通信，保障用户体验流畅性。
  算法层：基于动态补偿模型实现灌溉决策，调用和风天气 api 获得未来 3 小时
的降水概率综合决策，算法实现简单实用。
  扩展性：模块化设计支持功能迭代，预留多作物适配接口（如棉花、小麦），
未来可通过机器学习优化灌溉模型参数。

2.1.2 经济可行性
  经济可行性是指系统投入市场使用后，获得的经济利润大于其本身的成本开
发。本项目的主要成本和预期收益主要包括：
  开发成本：系统采用开源技术栈（Java、Vue.js、MySQL），大幅降低软件授
权费用。
  预期收益：沙地马铃薯灌溉用水量较没有使用此系统前大幅减少，进而降低
沙地特殊环境下用水成本。

2.1.3 操作可行性

  用户界面：前端采用 Vue.js 实现响应式设计，界面简洁直观。普通用户可通
过灌溉计划管理、天气信息等模块快速完成操作，管理员后台功能分类清晰，降
低学习成本。

  系统维护：数据库采用规范化设计，支持数据备份与恢复。
  用户接受度：系统通过角色权限划分（管理员、普通用户）满足不同需求，

                   3 开发环境及相关技术介绍


3.1 Java 语言与 JDK 开发环境

      Java 身为面向对象的跨平台编程语言，依靠其“一次编写，处处运行”的特性，
成为后端开发的关键选择，本系统把 JDK 11 用作开发环境，其长期支持版本有稳
定的垃圾回收机制以及模块化系统，提升了内存管理效率与代码可维护性，开发
工具选用 IntelliJ IDEA 2024.1，集成 Lombok 插件使实体类构建得以简化，借助
Maven 3.8.4 管理项目依赖，保证依赖版本一致性。存在以下技术优势：高并发支
持方面，基于 Java 线程池与 NIO 技术，可支持每秒 1000 多个请求的并发处理，
生态丰富方面，依靠 Spring 生态可快速构建企业级应用，跨平台部署方面，依靠
JVM 达成 Windows/Linux 双平台的无缝迁移，适配云服务器与边缘计算设备。

3.2 Vue 框架

      Vue.js 是一套有渐进式特点的 JavaScript 框架，它借助声明式渲染机制，可高
效地构建动态用户界面，它的核心设计理念以分层架构为中心，基础库着重于视
图层的响应式数据绑定以及组件化开发，同时借助配套工具链，为企业级单页应
用的工程化构建提供支持，开发者可以依据项目的需求，灵活地选择功能模块，
可借助 CDN 快速引入轻量级的核心功能，也可借助 Vue CLI 脚手架搭建全功能的
开发环境。这种渐进式的特性提升了框架的兼容性，让它可和 jQuery、Bootstrap
等传统技术无缝融合，降低有系统的迁移成本，在技术实现方面，Vue.js 基于 ES6
Proxy 构建了高效的响应式系统，借助数据劫持与依赖追踪机制实现 DOM 的自动
更新，结合虚拟 DOM 差分算法，把渲染性能提高了 30%-50%。组件化开发模式
采用单文件组件结构，将模板、逻辑与样式封装成独立单元，组合式 API 的引入
提高了逻辑复用能力，官方工具链覆盖开发、测试与部署的全流程，再加上 Element
Plus、Vant 等第三方组件库，形成了完整的生态系统，研究显示，Vue.js 凭借直观
的模板语法降低了学习门槛，初级开发者能在 8 小时内完成基础功能开发，不过
它在超大规模应用中仍需要依赖服务端渲染或静态站点生成来优化首屏性能。作
为现代 Web 开发的关键工具，该框架在开发效率、运行性能与可维护性之间达成
了有效平衡，但其在微前端架构下的状态管理优化仍是当前研究的重点领域。

3.3     SpringBoot 框架

      Spring Boot 是 Pivotal 团队依托 Spring Framework 开发的开源框架[7]，以“约

定优于配置”作为核心要点，对企业级应用开发给予简化，它借助起步依赖自动

3.4    Websocket 技术

      WebSocket 是一种依托于 TCP 的通信协议，由 HTML5 标准所提出，它可达
成客户端与服务器之间的全双工实时通信[8]，跟传统 HTTP 协议那种单向请求 - 响
应模式不一样，WebSocket 在凭借一次握手构建起持久化连接之后，便支持双向且
低延迟的数据传输，服务器可主动把数据推送给客户端，而不需要客户端不停地
进行轮询。此协议运用轻量级的帧结构来传输数据，其头部开销仅仅 2 到 10 字节，
极大地减少了网络带宽的消耗，适合应用在对实时性要求较高的场景当中，像是
在线聊天、股票行情以及智能设备监控等，WebSocket 的默认端口和 HTTP 一样，
可以兼容现有的防火墙策略，同时还提供安全加密的 wss 协议。在现代 Web 应用
里，它已然成为实现实时数据同步、物联网设备交互以及即时通讯的核心技术，
像 Spring Boot 等框架借助注解和 Handler 机制让其集成开发变得更加简便。

3.5    MySQL 数据库

多线程并发机制达成多用户协同操作，系统资源可依照实际应用场景做弹性配置，
这样的架构优势让硬件设备不用严格匹配就能构建出高性能数据库环境。该系统
拥有完善的异构数据集成能力，支持结构化以及非结构化数据的混合存储，可高
效处理静态数据存储与动态数据流分析的复合需求，基于其模块化架构设计，
MySQL 实现了对主流操作系统平台的全面覆盖，像 Windows、Linux 和 Unix 等系
统环境都囊括在内，同时还提供 C++、Python、Java 等多语言接口，呈现出不错的

4.5 本章小结

   本章围绕系统核心功能设计与数据库架构展开详细阐述。信息论述了系统顺
序图设计部分。数据库设计遵循分层设计原则，从概念模型的 E-R 图抽象到逻辑
结构的具体表实现，建立了以用户和管理员、实时信息与灌溉计划管理和智慧决
策为核心的五大数据库表。其中，管理员表（admin）与用户表（user）通过主键
实现身份标识，实时信息表（realtime）记录温度、湿度及设备状态等多维度传感
数据，灌溉计划管理表（watercontrol）则涵盖计划参数、阈值设置与执行状态等
字段。各表采用规范化设计，数据类型与约束条件明确，如时间戳字段（createdAt）
采用 datetime 类型保障时序精度，水泵状态（tpump/wpump）以二进制标识提升查
询效率。整体设计兼顾功能完备性与扩展性，为系统开发奠定坚实的数据交互与
存储基础。

              6 智慧灌溉决策功能详细设计


6.1 智慧灌溉决策功能设计目标

  灌溉决策功能须具有以下功能特性：
  精准性：基于实时土壤湿度、温度、气象数据动态调整灌溉策略，误差≤3%[9]。
  实时性：决策响应时间≤3 秒，满足田间实时控制需求。
  适应性：支持用户自定义参数（如灌溉阈值、模式等），兼容沙地马铃薯种
植的特殊环境[10]。
  节能性：优化水泵启停逻辑，减少能源消耗与水资源浪费。
  灌溉决策在前端页面表现为选择手动模式时跳转到灌溉计划管理页面，手动
设定灌溉温度阈值和灌溉湿度阈值及灌溉起止时间等；用户也可选择自动模式，
在自动模式下，系统会执行灌溉决策算法，系统后端会收到传感器传来的实时信
息结合沙地马铃薯对温湿度的喜好综合分析是否打开水泵和用水量应该控制在多
少[11]。灌溉决策页面展示灌溉用水量折线图。
