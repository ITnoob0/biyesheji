# 90 教师画像积分总分直接累计与规则版本口径修正记录

## 一、制度文件核查结论

本次核查对象为 `documents/附件2：关于印发《延安大学高质量贡献评价与激励标准（科研部分2022版）》的通知.pdf` 转出的制度文本，以及当前已整理的 `documents/research-evaluation-source.txt`、`documents/research-evaluation-core-rules-final.md`。

制度中明确给出了各类科研成果的积分表、积分分配原则和同一成果不重复计分原则，但未要求对“论文著作、项目、获奖、转化、智库、平台团队、科普类获奖”等成果大类再做固定权重加权后生成总分。

关键依据如下：

1. 制度总则说明科研成果评价按照质量和贡献度进行积分，激励以积分值为依据。
2. 第四条中的“定量评价中学术贡献、社会贡献量化积分占评价总积分的 80%；定性评价占 20%”是定量/定性评价结构，不是当前系统中科研成果大类的二次加权公式。
3. 制度明确“同一成果符合多项激励条件时，不重复计算，按积分就高原则”，说明成果积分应按规则条目确定后进入累计口径。
4. 项目、论文、获奖、成果转化、智库、平台团队和科普类成果均按各自条目、到账金额、完成单位比例、团队分配等内部规则形成积分，未出现大类间固定权重汇总要求。

因此，当前系统中“科研成果积分总分”的正式口径确定为：

> 审核通过且去重后的规则化成果 `final_score` 直接相加。

五维教师画像雷达保留为结构展示和趋势解释，不再作为官方总分的二次加权来源。

## 三、规则版本口径补齐

为满足“过去认同、规则变更后不重算历史成果”的画像要求，本次同步补齐规则版本口径：

1. 教师画像、雷达、深度分析、全部成果、画像报告接口均支持 `rule_version` 查询参数。
2. `rule_version=all` 或不传时，表示全部规则版本累计，历史成果沿用其所属规则版本下已审核冻结的 `final_score`。
3. `rule_version=<版本ID>` 时，仅展示该规则版本下的已审核成果与积分。
4. 历史规则版本下已审核通过的成果禁止继续编辑或删除，返回 `409 rule_achievement_version_frozen`。
5. 教师端和管理员查看教师画像弹窗均增加规则版本选择能力。

## 五、当前论文撰写可采用的表述

系统对教师科研积分采用“规则条目积分直接累计”口径：教师提交成果后，系统根据当前启用评价规则生成预估积分，学院管理员审核确认最终积分；审核通过后成果进入教师画像与统计看板。科研积分总分由所有审核通过且去重后的成果最终积分直接相加，雷达画像仅用于展示成果结构、能力分布和阶段变化，不作为总分的二次加权依据。若评价规则版本发生变化，历史版本下已审核成果的积分冻结保留，新成果按当前启用规则计分并继续累计，从而满足历史认可与规则更新后的连续评价需求。
