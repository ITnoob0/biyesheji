# 成果申报、审批、计分、查看全流程闭环审核测试记录

## 1. 审核目标

本次审核聚焦规则化科研成果主链路是否闭环：

1. 教师端成果申报
2. 系统按规则预估计分
3. 学院管理员待审查看
4. 学院管理员通过或驳回
5. 教师补正后重新提交
6. 审批通过后积分生效
7. 教师端列表、详情、统计与流程日志可查看
8. 非本人教师不可查看他人成果
9. 管理员只能审核，不可代替教师通过自助入口修改申报材料

## 4. 覆盖的闭环行为

### 教师申报

- 教师可获取规则化成果录入配置。
- 教师可对 7 类成果逐类提交。
- 提交后状态统一进入 `PENDING_REVIEW`。
- 提交时同步生成预估分 `provisional_score`。
- 未正式生效前 `final_score` 保持 `0.00`。

### 计分

- 固定积分规则按当前规则条目给出预估分。
- 成果转化到账金额按 `8/万元` 计算，35.75 万元得到 286.00 分。
- 平台团队按负责人分配的本人积分计入，本例为 180.50 分。
- 论文无 DOI/ISBN/检索号时仍可提交，并使用题名、刊物、时间等复合事实识别同一成果。

### 审批

- 学院管理员可查看本学院教师的待审成果。
- 学院管理员可驳回成果并写入驳回原因。
- 教师可查看被驳回成果及驳回意见。
- 教师补正材料后重新提交，成果回到 `PENDING_REVIEW`。
- 学院管理员再次审核通过后，`final_score` 与 `provisional_score` 生效一致。

### 查看

- 教师可查看本人全部成果列表。
- 教师可按 `APPROVED`、`REJECTED` 等状态筛选。
- 教师可查看成果详情。
- 教师可查看流程日志，日志包含 `CREATE`、`SUBMIT_REVIEW`、`REJECT`、`UPDATE`、`APPROVE` 等关键节点。
- 教师端成果统计可反映审批通过后的成果数量。

### 权限边界

- 非本人教师查看他人成果详情返回 `404`。
- 学院管理员只能通过待审、通过、驳回接口处理成果。
- 本次发现并修复：管理员原本可通过 `PATCH` 走教师自助入口修改成果材料；现已禁止，返回 `403`。

## 5. 修复点

修复文件：`backend/achievements/rule_views.py`

修复内容：

- `PUT` 原本已禁止管理员代改教师成果。
- `PATCH` 缺少同样限制，本次补齐。
- 现在管理员不能通过教师自助入口编辑成果，只能审核通过或驳回。

新增测试已覆盖该边界：

- 学院管理员对教师成果执行 `PATCH`。
- 预期返回 `403_FORBIDDEN`。
- 教师本人仍可在被驳回后补正并重新提交。

## 7. 审核结论

成果申报、审批、计分、查看主流程已经形成闭环：

- 教师端可以按 7 类成果提交真实业务材料。
- 系统可以在提交时按规则生成预估分。
- 管理员可以查看待审成果并完成通过/驳回。
- 被驳回成果可以由教师补正后重新进入审核。
- 审批通过后积分生效，并能在教师端列表、详情、统计和日志中查看。
- 权限边界已经补齐，管理员不能绕开审批职责代改教师申报内容。

后续如果继续做画像系统，应直接基于审批通过后的 `final_score`、成果大类、规则条目和事实字段进行画像计算，不应使用待审或驳回成果作为正式画像输入。
