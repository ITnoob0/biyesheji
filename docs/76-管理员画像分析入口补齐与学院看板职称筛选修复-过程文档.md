# 76-管理员画像分析入口补齐与学院看板职称筛选修复-过程文档

## 1. 本次问题
- 学院管理员学院看板的“职称”筛选项缺少“副教授”。
- 两类管理员在管理端“查看画像”弹窗内未看到：
  - 年份切片
  - 同侪基准线叠加（本院/全校）

## 2. 修复内容
### 2.1 学院看板职称筛选补齐
- 后端文件：`backend/achievements/academy_dashboard_analysis.py`
  - 在 `build_filter_options` 中加入默认职称集：`教授 / 副教授 / 讲师`。
  - 与动态职称并集去重后返回，保证前端至少可筛选这三类职称。
- 前端文件：`frontend/src/views/AcademyDashboardView.vue`
  - 新增 `teacherTitleOptions`，对接口返回值再做一次默认并集兜底（双保险）。

### 2.2 管理端画像弹窗能力补齐
- 前端文件：`frontend/src/components/teacher/TeacherRadarPreviewDialog.vue`
  - 从原先 `/api/achievements/radar/<id>/` 升级为调用 `/api/achievements/portrait/analysis/`。
  - 新增“年份”切片下拉。
  - 系统管理员新增“本院平均/全校平均”切换。
  - 雷达图传入双图层 `seriesData`，展示管理员侧基准线叠加。
  - 显示当前实际生效范围标签（处理后端强制降级后的真实 scope）。
  - 保持学院管理员仅院级基准，系统管理员可切换校级基准。

## 3. 联调验证
- 后端检查：`python manage.py check` 通过。
- 后端接口测试：`python manage.py test achievements.test_portrait_snapshot_benchmark --keepdb` 4/4 通过。
- 前端类型检查：`npm run type-check` 通过。
- 前端构建：`npm run build` 通过（仅保留既有 chunk size 警告，不影响功能）。

## 4. 结果
- 学院管理员学院看板可见“副教授”筛选项。
- 系统管理员与学院管理员在管理端“查看画像”弹窗均可按年份切片查看雷达画像。
- 管理端雷达图已显示同侪基准线；系统管理员可切换本院/全校基准，学院管理员固定本院基准。
