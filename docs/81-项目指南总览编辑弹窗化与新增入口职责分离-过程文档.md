# 81-项目指南总览编辑弹窗化与新增入口职责分离-过程文档

## 1. 需求目标
- 项目总览列表操作列收口为：`定向推送`、`编辑`、`删除`。
- 编辑改为“页面弹出卡片（Dialog）”交互，不再依赖切换到“新增/编辑”二级菜单。
- 二级菜单从“新增 / 编辑”改为“新增项目”，并在功能职责上只负责新增。

## 2. 改动文件
- `frontend/src/views/ProjectGuideManagementView.vue`
- `frontend/src/router/workspaceRoutes.ts`

## 3. 关键实现

### 3.1 项目总览操作列收口
- 在总览视图中移除原先的“归档/重新开放”按钮，仅保留：
  - `定向推送`
  - `编辑`
  - `删除`
- 生命周期治理动作（归档/重新开放）仍保留在“生命周期管理”二级菜单中。

### 3.2 编辑改为弹窗卡片
- 新增编辑对话框：
  - `editDialogVisible`
  - `editingGuideId`
  - `editForm`、`editFormRef`
  - `submitEditForm()`
- 点击总览操作列 `编辑` 时：
  - 使用当前行数据填充 `editForm`
  - 弹出 `el-dialog` 执行修改
- 保存后调用 `PATCH /api/project-guides/{id}/`，成功后关闭弹窗并刷新列表。

### 3.3 新增入口职责收口
- 管理页表单取消“编辑态”逻辑（移除 `editingId` 分支）。
- `submitForm()` 固定执行 `POST`，仅负责新增项目。
- 表头文案固定为“新增项目指南”，按钮固定为“创建指南”。

### 3.4 菜单命名更新
- 路由菜单文案：
  - `project-guide-management-manage` 从“新增 / 编辑”改为“新增项目”。

## 4. 验证结果
- 前端类型检查：
  - 命令：`npm run type-check`
  - 结果：通过（`vue-tsc --build` 无报错）。

## 5. 结果结论
- 已实现新增与编辑职责分离。
- 编辑反馈由“无感切换”改为“当前页弹窗编辑”，交互路径更清晰。
- 项目总览操作列已收口为指定三项功能。
