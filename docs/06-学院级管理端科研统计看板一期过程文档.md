# 学院级管理端科研统计看板 一期过程文档

## 1. 任务背景

本轮任务目标是在当前系统中新增一个面向管理员的“学院级/管理端科研统计看板”，但只做克制版聚合统计，不扩展成复杂 BI 系统。

约束如下：

- 当前已有教师维度画像
- 学院级综合看板不属于当前一期硬验收主链路
- 数据源以 MySQL 主业务数据为主
- 只做 4 到 6 个最有演示价值的聚合指标

## 2. 方案取舍

### 2.1 为什么不新建复杂分析模块

当前系统主链路仍以教师个人画像为核心。  
如果直接引入复杂 BI 页面、筛选体系和钻取分析，会明显超出本阶段目标，并增加主链路被影响的风险。

因此本轮采用：

- 一个轻量管理端聚合接口
- 一个管理员专用统计页面
- 两个简单图表 + 若干核心指标卡片

### 2.2 为什么将聚合接口单独拆出

为避免继续把 `achievements/views.py` 堆成多职责文件，本轮将管理端聚合接口单独拆到：

- `backend/achievements/management_dashboard.py`

这样做的原因：

- 教师画像接口与学院级聚合接口职责分离
- 后续若继续扩展管理端统计，改动边界更清楚

## 3. 一期展示内容

本轮学院级看板展示了 5 类信息：

1. 指标卡片
   - 教师总数
   - 论文总数
   - 总成果数
   - 合作关系数
   - 项目与知识产权总量

2. 年度成果趋势
   - 按年份聚合总成果、论文、项目

3. 院系统计分布
   - 按教师所属院系聚合

4. 高活跃教师排行
   - 按成果总量排序

5. 合作活跃度概览
   - 合作关系总数
   - 有合作记录的教师数
   - 有合作的论文数
   - 平均每篇论文合作人数

## 4. 后端实现

新增文件：

- `backend/achievements/management_dashboard.py`

修改文件：

- `backend/achievements/urls.py`
- `backend/achievements/tests.py`

新增接口：

- `GET /api/achievements/academy-overview/`

权限策略：

- 仅管理员可访问
- 普通教师访问返回 `403`

数据来源：

- `users_customuser`
- `achievements_paper`
- `achievements_project`
- `achievements_intellectualproperty`
- `achievements_teachingachievement`
- `achievements_academicservice`
- `achievements_coauthor`

全部基于 MySQL 实时聚合，不依赖 Neo4j。

## 5. 前端实现

新增文件：

- `frontend/src/views/AcademyDashboardView.vue`
- `frontend/src/views/academy-dashboard/overview.ts`

修改文件：

- `frontend/src/router/index.ts`
- `frontend/src/views/DashboardView.vue`

前端设计原则：

- 风格延续当前教师画像页
- 页面独立，不侵入现有画像布局
- 继续复用 ECharts
- 图表 helper 单独拆文件，避免页面同时承担过多图表配置逻辑

路由接入：

- `/academy-dashboard`

权限控制：

- 路由增加 `requiresAdmin`
- 非管理员在前端守卫中直接回退到教师画像页
- 后端接口仍继续做管理员校验，形成双保险
