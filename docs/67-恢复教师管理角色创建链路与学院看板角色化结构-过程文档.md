# 67-恢复教师管理角色创建链路与学院看板角色化结构-过程文档

## 背景

本次修改属于会话历史成果恢复，不新增需求，不改技术路线，只把已经在历史里确认过、但当前工作区尚未完全恢复的内容补回。

## 本次恢复目标

1. 恢复教师管理的最终角色创建链路。
2. 恢复学院看板按系统管理员 / 学院管理员分区后的菜单与页面结构。
3. 恢复学院看板的学院间对比与学院范围隔离能力。

## 恢复内容

### 1. 教师管理恢复

- 系统管理员不再通过教师管理创建教师，改为创建学院管理员。
- 学院管理员恢复“创建教师账号”入口，且仅允许创建本学院教师。
- 教师管理列表与汇总按角色收口：
  - 系统管理员查看全校教师与学院管理员账户，不包含系统管理员自身。
  - 学院管理员仅查看本学院教师账户。

### 2. 学院看板菜单恢复

- 系统管理员：
  - 总览
  - 学院对比
  - 学院钻取
- 学院管理员：
  - 总览
  - 教师分析

### 3. 学院看板页面恢复

- 系统管理员总览恢复为：
  - 年度成果趋势单独一行
  - 范围结构分布与全校管理摘要同一行，左宽右窄
- 系统管理员学院对比恢复为：
  - 主学院与对比学院双学院对比
  - 学院结构分布在双学院场景下仅展示这两个学院
- 系统管理员学院钻取恢复为：
  - 去除旧版辅助板块
  - 保留“学院钻取与教师排行”主卡片
  - 操作列改回普通同层列，不做浮层固定
  - 操作按钮恢复为“钻取 / 画像 / 全部成果”
- 学院管理员总览恢复为：
  - 本院教师速览
  - 近期成果记录
  - 近期成果记录最多展示 5 条
- 学院管理员教师分析恢复为本院教师分析表格，操作列保留画像预览。
