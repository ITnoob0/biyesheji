# 43-跨模块联动增强-过程文档

## 1. 本次任务背景

本轮增量开发聚焦“资料 / 成果 / 画像 / 图谱 / 推荐 / 问答 / 看板”的跨模块联动增强。

目标不是新增大量独立页面，而是在现有严格守卫、现有路由和现有数据链路上，把高价值链路补成“可跳转、可定位、可解释、可回退”的证据闭环。

## 2. 扫描后确认的现状

### 2.1 已有能力

- 成果页已能进入画像主页、推荐页和问答页，但多数跳转停留在模块首页。
- 画像页已具备维度解释、代表性成果、图谱、推荐入口和问答入口。
- 推荐页已具备推荐理由、画像联动说明、管理员教师对比、来源链接和问答入口。
- 智能问答已具备来源说明、边界提示和失败回退。
- 学院看板已具备画像和推荐跳转，以及院系/教师钻取。

### 2.2 关键断点

- 现有跳转大多缺少统一 query 协议，无法表达“从哪来、要看哪块证据、要高亮哪条记录”。
- 问答来源卡片只有说明，没有真正可回跳的证据链接。
- 推荐页虽然有画像联动说明，但缺少真实支撑成果列表。
- 看板到问答的链路不完整。
- 成果到画像、画像到推荐、推荐到成果，多数还是“能到页面，不能定位证据”。

## 3. 本次完成的联动闭环

### 3.1 统一联动参数与定位逻辑

新增前端工具：

- `frontend/src/utils/crossModuleLinking.ts`

统一约定以下联动参数：

- `source`：来源模块
- `page`：目标页面语义
- `section`：目标证据区
- `dimension_key`：画像维度键
- `record_type` / `record_id`：成果类型与成果记录
- `guide_id`：项目指南
- `question_type`：问答场景
- `department` / `year`：学院看板问答范围
- `note`：页面提示说明

同时补充统一能力：

- 管理员目标教师视角下自动带 `user_id`
- 画像页根据管理员/教师身份自动解析到 `/profile/:id` 或 `/dashboard`
- 证据区滚动定位与高亮

### 3.2 已打通的高价值链路

#### A. 成果 -> 画像维度 / 推荐 / 问答

- 成果页最近成果卡片新增：
  - 跳转画像对应维度
  - 跳转推荐证据区
  - 跳转问答说明
- 论文列表中的联动入口不再只是模块首页，而是带上下文跳转。

#### B. 画像 -> 推荐原因 / 支撑成果 / 问答

- 画像页维度卡片新增：
  - 查看推荐理由
  - 问答说明
- 画像页代表性成果新增：
  - 教师视角下回到成果证据区
  - 跳到推荐解释
  - 跳到问答说明
- 画像页新增联动来源提示和证据区高亮。

#### C. 推荐 -> 支撑成果 / 画像维度 / 指南详情 / 问答

- 推荐服务层新增 `supporting_records`，来源于当前教师真实成果。
- 推荐页每张卡片新增：
  - 查看画像维度
  - 查看支撑成果
  - 原有来源链接继续作为指南详情入口保留
  - 问答说明
- 推荐页支持按 `guide_id` / `dimension_key` / `record_type + record_id` 回跳并定位卡片。

#### D. 看板 -> 画像 / 推荐 / 问答

- 学院看板排行表保留画像、推荐入口，并新增教师问答入口。
- 学院看板顶部新增“看板问答”，可直接带当前 `department` / `year` 进入学院问答摘要。
- 学院看板新增对排行区、钻取区的证据定位回跳。

#### E. 问答 -> 对应页面证据区

- 问答来源卡片 `source_details` 新增结构化 `link`。
- 前端在来源卡片上直接渲染“回到证据区”按钮。
- 当前已支持回跳到：
  - 画像维度证据区
  - 画像成果证据区
  - 画像说明区
  - 推荐证据区
  - 成果记录区
  - 学院排行区
  - 学院钻取区

## 4. 权限边界处理

### 4.1 教师视角

- 教师仍只能访问本人画像、本人推荐、本人问答、本人成果录入。
- 教师视角下不会生成学院看板入口。
- 教师视角下不会生成管理员教师对比入口。
- 教师从推荐页不会获得越权进入他人成果录入的路径。

### 4.2 管理员视角

- 管理员可查看指定教师画像、推荐和问答，并继续通过 `user_id` 保持联动上下文。
- 管理员在推荐页和画像页不会出现“进入成果录入页代教师操作”的链路。
- 管理员到学院问答与学院看板联动仅在管理员链路内闭环。

## 8. 最小人工验证步骤

### 链路 1：成果 -> 画像维度 -> 推荐 -> 成果

1. 教师登录后进入成果录入页。
2. 在“最近成果”或论文列表点击“画像维度”。
3. 确认画像页出现联动提示，并定位到对应维度证据卡片。
4. 在该维度卡片点击“查看推荐理由”。
5. 确认推荐页定位到对应推荐卡片。
6. 在“支撑成果”中点击“查看支撑成果”。
7. 确认成果页出现联动提示，并定位到成果证据区。

### 链路 2：推荐 -> 问答 -> 推荐 / 画像 / 成果

1. 教师或管理员进入推荐页。
2. 在任一推荐卡片点击“智能解读”。
3. 确认问答页自动带入场景并生成答案。
4. 在问答来源卡片点击“回到该指南推荐卡片”或“查看画像维度”或“查看支撑成果”。
5. 确认页面定位到对应证据区，而不是停留在模块首页。

### 链路 3：看板 -> 问答 -> 看板

1. 管理员进入学院看板。
2. 点击“看板问答”。
3. 确认问答结果基于当前学院范围返回。
4. 在问答来源卡片点击“回到学院排行区”或“回到学院钻取区”。
5. 确认学院看板定位到对应区域。

## 9. 本次明确未完成的边界

- 当前没有新增复杂的“全页面互跳矩阵”，仍坚持高价值主链路优先。
- 当前推荐支撑成果以轻量规则匹配和近期记录为主，不引入复杂语义检索或向量检索。
- 当前问答回跳以来源卡片为主，不扩展为多轮对话态下的上下文证据树。
- 当前画像维度趋势仍是实时估算口径，不引入历史快照冻结版证据定位。
- 当前学院看板仍是轻量分析入口，不扩展为完整 BI 多维切片平台。
