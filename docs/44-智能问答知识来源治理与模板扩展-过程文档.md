# 44-智能问答知识来源治理与模板扩展-过程文档

## 1. 本次任务范围

本轮在现有“系统内真实数据驱动的模板化问答”基础上，补齐以下一期最小闭环能力：

- 扩展问答模板覆盖范围，新增画像治理、成果联动、图谱状态类问题。
- 增强来源卡片治理信息，明确来源模块、可验证页面、当前可用状态和核验方式。
- 增强边界说明与失败回退，覆盖无数据、数据不足、权限不足、推荐降级、图谱降级、画像快照未持久化等场景。
- 让部分问答结果可直接回跳到画像、成果、推荐、看板中的真实证据区。

本次仍严格保持以下边界：

- 仍然是系统内真实数据问答，不引入 RAG、向量检索、外部知识库。
- 仍然是模板化、可解释、可回退能力，不包装成无边界通用问答平台。
- 不重写原有问答主链路，不改变登录态、严格守卫、推荐路线、图谱可选链路和管理员/教师轻量角色边界。

## 3. 本次实现内容

### 3.1 模板扩展

新增问答类型：

- `portrait_data_governance`
  - 说明画像当前仍是运行时分析视图。
  - 解释基础档案完整度、成果样本量、历史快照未持久化边界。
- `achievement_portrait_link`
  - 说明成果类型如何支撑画像维度。
  - 让成果页、画像维度区和画像成果区形成可回跳闭环。
- `achievement_recommendation_link`
  - 说明成果如何支撑当前推荐。
  - 复用推荐服务中的 `supporting_records` 和 `portrait_dimension_links`。
- `graph_status`
  - 说明图谱采用 Neo4j 优先、MySQL 回退的轻量链路。
  - 在成果不足时明确回退为边界提示。

### 3.2 来源治理增强

来源卡片新增治理字段：

- 来源模块
- 可验证页面
- 当前可用状态
- 核验提示

当前前端会优先展示后端显式下发的治理字段；若后端未显式下发，则按联动页面自动推导基础标签，保证老模板仍可兼容。

新增 `source_governance` 聚合结构：

- `answer_mode`
- `scope_label`
- `verification_note`
- `degraded_flags`
- `unavailable_flags`

这部分会集中承接：

- 画像快照未持久化
- 推荐没有开放指南
- 推荐暂无高匹配
- 图谱处于 MySQL 回退口径
- 图谱当前无数据
- 成果样本偏少
- 教师基础档案缺口

### 3.3 证据回跳补强

新增并打通的问答回跳：

- 问答 -> 画像维度证据区
- 问答 -> 画像成果证据区
- 问答 -> 画像说明区
- 问答 -> 画像页图谱板块
- 问答 -> 成果录入页记录区
- 问答 -> 推荐页指南证据区
- 问答 -> 学院看板排行区 / 钻取区

本次额外补了 `portrait-graph` 联动协议，并在画像页图谱卡片上新增真实锚点区，避免“能跳但没有证据区”的假联动。

## 4. 失败回退与边界策略

### 4.1 已覆盖场景

- 无数据
  - 没有成果时，不输出成果支撑画像/图谱等伪解释，直接回退为缺口说明。
- 数据不足
  - 成果少于 2 项时，保留说明，但明确提示样本偏少。
- 权限不足
  - 教师请求学院统计类问答时仍返回 403，由前端统一回退为权限边界说明。
- 推荐降级
  - 无开放指南：回退为推荐边界说明。
  - 有开放指南但无高匹配：回退为规则边界解释。
- 图谱降级
  - Neo4j 未启用时，明确按 MySQL 回退链路说明。
  - 无成果时，明确无法形成图谱证据。
- 画像快照缺失
  - 问答明确说明当前为运行时分析视图，不宣称已有正式历史快照。

### 4.2 仍然明确不做的部分

- 不输出系统外部知识。
- 不做通用自然语言事实问答。
- 不做复杂图挖掘结论。
- 不把规则增强推荐包装成机器学习推荐。

## 6. 最小人工验证步骤

### 6.1 教师视角

1. 进入“智能辅助与问答说明”页。
2. 依次选择：
   - `画像数据口径与缺口说明`
   - `成果如何支撑画像`
   - `成果如何支撑推荐`
   - `图谱链路与降级状态`
3. 检查结果页是否出现：
   - 来源模块标签
   - 可验证页面标签
   - 当前可用状态标签
   - 来源治理说明区
4. 点击来源卡片按钮，确认可回跳到画像维度区、画像成果区、画像说明区、图谱板块、成果记录区或推荐证据区。

### 6.2 管理员视角

1. 以管理员进入问答页。
2. 使用指定 `user_id` 查看其他教师的画像类、推荐类和图谱类问答。
3. 选择 `学院统计概览`，检查来源卡片是否仍回到学院看板真实统计区。
4. 确认管理员可查看教师画像/推荐/问答，但不会因为问答跳转拿到教师私有录入编辑权限。

### 6.3 边界验证

1. 选择一个成果为空或样本极少的教师。
2. 触发 `成果如何支撑画像` 或 `图谱链路与降级状态`。
3. 确认结果回退为缺口说明，且包含失败原因、边界提示和证据回跳。

## 8. 历史遗留问题与本次边界

历史遗留但非本次新增问题：

- 仓库当前存在较多其他模块改动与未提交文件，本次未做清理或回滚。
- 画像历史快照仍未落库，这属于现有基线边界，不是本次问答增强新增缺陷。
- 图谱当前仍是轻量增强能力，不是复杂图分析平台，这属于既定产品边界。

本次已明确交付但故意未扩大的边界：

- 只做高价值模板扩展，不做任意自然语言问题泛化。
- 只做已有真实页面的证据回跳，不造新页面承接不存在的证据区。
- 只做轻量治理展示，不为问答能力引入重型知识中台。
