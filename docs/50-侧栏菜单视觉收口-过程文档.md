# 50-侧栏菜单视觉收口-过程文档

## 1. 本轮目标

本轮不再调整一级菜单 / 二级菜单逻辑、权限逻辑、默认首页逻辑和业务页面数据逻辑，只针对前端布局表现做视觉收口：

- 将左侧导航从“模块轨道 + 浅色二级面板”改为统一深色侧栏
- 一级菜单与二级菜单改为同一侧栏中的折叠式摆放
- 菜单摆放方式参考 Vben 风格的后台信息架构，但未引入任何后台模板
- 顶部工具栏与右侧内容区仅做配套样式收口，不改业务主链路

## 2. 本轮实际修改

### 2.1 MainLayout 结构收口

文件：

- `frontend/src/layout/MainLayout.vue`

调整内容：

- 侧栏品牌区固定在顶部
- 一级模块菜单改为整行深色导航项
- 当前模块的二级子视图直接在该模块下展开
- 二级菜单使用缩进、圆点和高亮条表示当前子页
- 去掉上一轮“左窄轨道 + 右浅色二级面板”的分裂结构

### 2.2 全局变量补充

文件：

- `frontend/src/assets/workspace-system.css`

补充变量：

- `--workspace-sidebar-bg`
- `--workspace-sidebar-border`
- `--workspace-sidebar-text`
- `--workspace-sidebar-text-strong`
- `--workspace-sidebar-hover`

说明：

- 仍复用现有 `workspace-system.css` 作为全局视觉变量入口
- 没有新建独立主题系统

## 3. 当前效果

本轮完成后，左侧导航已变为更接近参考图的信息架构表现：

- 一级菜单纵向排列
- 当前一级菜单展开其稳定二级菜单
- 一级、二级菜单都位于同一深色侧栏
- 右侧内容区继续保持独立滚动

## 4. 本轮未改动的部分

- 未调整路由守卫
- 未调整 sessionStorage 登录态
- 未调整管理员 / 教师默认首页逻辑
- 未调整业务页面的数据获取、接口调用和统计逻辑
- 未引入 tags-view、keep-alive、多页签

## 5. 验证

执行命令：

- `cd frontend && npm run type-check`
- `cd frontend && npm run verify:ui`
- `cd frontend && npm run verify:baseline`

本轮以这三项前端校验作为收尾验证。
