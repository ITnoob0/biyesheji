# 基础档案编辑与教师管理拆分-过程文档

## 1. 任务背景

当前系统中，“编辑基础档案”和“教师管理”两个按钮实际上会进入同一个混合页面。  
该页面同时包含：

- 当前登录人的基础档案编辑
- 管理员创建教师账号
- 教师账号总览与编辑

这会带来两个问题：

1. 功能语义不清晰  
2. 管理员点击“编辑基础档案”时会看到与自身资料无关的管理面板

因此本轮任务目标是：

- 将“编辑基础档案”与“教师管理”彻底拆成两个独立页面
- 保持现有后端接口不变，尽量只做前端职责重构
- 同时优化两个页面的版式，让界面更聚焦、居中、便于演示

## 2. 拆分方案

### 2.1 基础档案编辑页

新增页面：

- `frontend/src/views/TeacherProfileEditorView.vue`

职责：

- 只负责当前登录账号的基础档案维护
- 不再展示教师管理、账号创建、教师总览等内容

适用对象：

- 普通教师
- 管理员本人

对应路由：

- `/profile-editor`

### 2.2 教师管理页

重构页面：

- `frontend/src/views/TeacherManagementView.vue`

职责：

- 只负责管理员侧教师账号创建、教师资料维护、教师密码重置和画像跳转
- 不再混入“我的基础信息”表单

适用对象：

- 仅管理员

对应路由：

- `/teachers`

## 3. 入口调整

### 3.1 画像主页

`DashboardView.vue` 中：

- “编辑基础档案” 现在跳转到 `/profile-editor`
- “教师管理” 继续跳转到 `/teachers`

这样两个按钮语义和结果已经完全分离。

### 3.2 顶部会话栏

`App.vue` 中新增了：

- “基础档案”快捷入口

便于登录后从任意页面快速进入个人档案编辑页。

## 4. 路由与权限

修改文件：

- `frontend/src/router/index.ts`

改动：

- 新增 `/profile-editor`
- 将 `/teachers` 调整为 `requiresAdmin: true`

这样做的原因：

- “基础档案编辑”属于所有已登录用户的基础功能
- “教师管理”是管理员专用功能，应该在路由层明确限制

## 5. 页面设计调整

### 5.1 基础档案编辑页

设计方向：

- 居中单主卡片
- 顶部保留简洁说明区
- 中间突出“当前登录人身份卡”和“基础档案表单”
- 操作按钮置中，减少视觉分散

### 5.2 教师管理页

设计方向：

- 页面改为管理员管理工作台
- 顶部单独管理说明区
- 主体居中布局
- 保留：
  - 创建教师账号
  - 教师总览列表
  - 编辑弹窗
  - 重置密码
- 移除原先“我的基础信息”面板

## 6. 修改文件清单

- `frontend/src/views/TeacherProfileEditorView.vue`
- `frontend/src/views/TeacherManagementView.vue`
- `frontend/src/views/DashboardView.vue`
- `frontend/src/router/index.ts`
- `frontend/src/App.vue`

## 7. 验证结果

已执行：

1. `cmd /c npm run type-check`

验证目标：

- 新页面可正常编译
- 新路由无类型错误
- 原有受保护路由体系未被破坏

建议手工验证：

1. 普通教师登录后点击“编辑基础档案”
   - 应进入独立基础档案页
   - 不应看到教师管理内容

2. 管理员登录后点击“编辑基础档案”
   - 应进入独立基础档案页

3. 管理员登录后点击“教师管理”
   - 应进入独立教师管理页
   - 页面只展示管理功能，不再出现个人资料编辑面板

4. 普通教师直接访问 `/teachers`
   - 应被前端路由守卫拦回

## 8. 数据库说明

本轮仅调整前端页面与路由。  
未修改数据库模型，也无需迁移。

## 9. 遗留问题

本轮未继续扩展：

- 教师管理页搜索与筛选能力
- 教师资料编辑页的更细粒度分区
- 管理员批量导入教师账号

这些内容可在后续阶段按需继续增强，但不影响当前“功能拆分清晰化”的目标达成。
