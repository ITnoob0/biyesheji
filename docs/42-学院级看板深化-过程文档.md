本轮目标是在现有学院级轻量看板基础上，增强管理员的管理分析能力，重点补齐更完整的筛选、更细的钻取、一类稳定趋势/对比分析、排行榜切换和后端导出。

## 3. 本次落地内容

### 3.1 更完整的筛选维度

在原有院系、年份、教师筛选基础上，补充了：

- 教师职称筛选
- 合作情况筛选
- 成果类型筛选
- 排行榜维度切换

当前前后端都已承接这些筛选项。

### 3.2 更细的院系 / 教师 / 成果多级钻取

本次把看板钻取增强为：

- 院系层：院系统计分解表，可直接钻取某个院系
- 教师层：教师排行表可直接钻取某位教师
- 成果层：在当前院系或教师范围下展示近期成果记录

当前实现方式是基于现有聚合体系和筛选联动，不引入独立 BI 维表。

### 3.3 趋势与对比分析增强

在原有“年度成果趋势”基础上，新增：

- 范围趋势对比图：当前筛选范围 vs 全校范围

当前对比图主要展示：

- 当前范围总成果
- 全校总成果
- 当前范围论文
- 全校论文

用于帮助管理员快速判断“当前筛选范围与全校整体走势”的关系。

### 3.4 排行榜切换增强

支持按以下维度切换教师排行：

- 总成果
- 论文数
- 项目数
- 总被引
- 合作关系

看板会同步返回：

- 当前排行维度
- 当前排行标签
- 每位教师的排行值

### 3.5 导出能力增强

新增后端导出接口：

- `/api/achievements/academy-overview/export/`

当前支持导出目标：

- 教师排行
- 院系统计分解
- 趋势对比
- 当前范围近期成果

导出方式仍为 CSV，强调轻量和可复核，不引入报表任务系统。

### 3.6 管理摘要展示增强

新增管理摘要区，集中展示：

- 当前成果口径
- 当前排行维度
- 趋势摘要
- 导出边界
- 钻取边界
- 统计边界

这样管理员更容易理解当前看板“在看什么、能钻到哪里、能导出什么”。

## 4. 当前统计口径说明

当前看板统计口径如下：

- 统计数据来源于 MySQL 实时业务表
- 成果口径包含：
  - 论文
  - 项目
  - 知识产权
  - 教学成果
  - 学术服务
- 时间口径优先使用 `date_acquired`
- 合作关系口径基于论文合作作者记录
- 院系、教师、职称口径来自当前教师基础数据

当前强调：

- 实时聚合
- 可解释
- 可回溯
- 不引入重型离线分析依赖

## 5. 当前已支持的钻取 / 导出边界

### 当前已支持的钻取

- 院系 -> 教师 -> 近期成果
- 教师排行 -> 教师摘要 -> 近期成果
- 院系分解 -> 院系摘要 -> 院系内重点教师与近期成果

### 当前未支持的钻取

- 任意自定义多维切片组合
- 复杂跨表明细联邦钻取
- BI 式拖拽分析

### 当前已支持的导出

- 教师排行 CSV
- 院系统计分解 CSV
- 趋势对比 CSV
- 当前范围近期成果 CSV

### 当前未支持的导出

- Excel 多工作表报表
- PDF 报表
- 定时任务离线导出

## 6. 历史遗留问题与本次新增问题区分

### 历史遗留

- 后端已有部分筛选字段，但前端未完整承接
- 导出仅前端临时生成当前排行 CSV
- 钻取深度不足，更偏“看板展示”而非“管理入口”

### 本次新增

- 完整接通扩展筛选
- 新增范围趋势对比图
- 新增院系 / 教师 / 成果多级钻取
- 新增排行榜切换
- 新增后端导出接口
- 新增管理摘要边界说明

## 8. 最小人工验证步骤

1. 使用管理员账号进入学院级看板。
2. 依次切换院系、职称、成果类型、合作情况和排行维度，确认看板会稳定刷新。
3. 在教师排行中点击“看板钻取”，确认右侧钻取面板展示该教师摘要和近期成果。
4. 在院系统计分解表中点击“钻取院系”，确认钻取面板切换为院系摘要与院系近期成果。
5. 查看“范围趋势对比”图，确认能同时看到当前范围和全校范围走势。
6. 依次导出教师排行、院系统计分解和趋势对比 CSV，确认文件可下载且内容包含当前筛选结果。

## 9. 后续建议

- 如果下一阶段继续深化，可优先补“学院级跨时间窗口对比”而不是直接做复杂 BI。
- 如果导出需求继续增加，建议先补 Excel 多工作表导出，再评估是否需要报表任务化。
- 如果院系管理方需要更细运营视角，可在当前基础上扩展到“按职称 / 学科 / 团队”的对比看板。
