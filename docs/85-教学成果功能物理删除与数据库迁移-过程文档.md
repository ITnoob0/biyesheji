# 85-教学成果功能物理删除与数据库迁移-过程文档

## 1. 背景

- 在上一轮收口中，系统已经完成“科研画像去教学成果化”，即教学成果不再参与科研画像评分、五维雷达、趋势解释与推荐联动。
- 本轮目标进一步升级为“物理删除”：
  - 从后端运行时彻底移除教学成果功能
  - 删除对应 Django 模型与迁移状态
  - 删除后端路由、序列化器、图谱同步与统计逻辑
  - 前端继续保留稳定可运行状态，但不再开放教学成果入口

## 2. 本次目标

- 删除 `TeachingAchievement` 业务模型及其运行时引用
- 删除 `/api/achievements/teaching-achievements/` 相关路由
- 删除图谱、学院看板、推荐、问答、演示初始化脚本中对教学成果的依赖
- 补充数据库迁移，正式删除教学成果表

## 3. 后端删除范围

### 3.1 模型与迁移

- 文件：
  - `backend/achievements/models.py`
  - `backend/achievements/migrations/0015_delete_teachingachievement.py`
- 处理：
  - 删除 `TeachingAchievement` 模型
  - 删除 `AchievementOperationLog` 中 `teaching-achievements` 选项
  - 新增迁移 `0015_delete_teachingachievement.py`

### 3.2 路由与接口

- 文件：
  - `backend/achievements/urls.py`
  - `backend/achievements/views.py`
  - `backend/achievements/serializers.py`
- 处理：
  - 删除 `TeachingAchievementViewSet`
  - 删除 `TeachingAchievementSerializer`
  - 取消注册 `teaching-achievements` 路由

### 3.3 画像、统计与图谱

- 文件：
  - `backend/achievements/scoring_engine.py`
  - `backend/achievements/academy_dashboard_analysis.py`
  - `backend/achievements/management_dashboard.py`
  - `backend/graph_engine/views.py`
  - `backend/graph_engine/services.py`
  - `backend/graph_engine/utils.py`
- 处理：
  - 删除教学成果计数与统计字段
  - 删除学院看板中的教学成果分项
  - 删除图谱中 `TeachingAchievement` 节点与关系
  - 删除 Neo4j 同步/删除教学成果的工具方法

### 3.4 推荐、问答与初始化

- 文件：
  - `backend/project_guides/services.py`
  - `backend/ai_assistant/services.py`
  - `backend/users/management/commands/init_demo_teachers.py`
  - `backend/scripts/sync_mysql_papers_to_neo4j.py`
- 处理：
  - 推荐系统不再把教学成果纳入近期活跃度与候选证据
  - 智能问答不再输出教学成果相关汇总
  - 演示初始化命令不再同步教学成果数据
  - 图谱全量同步脚本不再同步教学成果节点

## 4. 前端处理策略

- 文件：
  - `frontend/src/views/AchievementEntryView.vue`
  - `frontend/src/views/AchievementReviewView.vue`
  - `frontend/src/views/AcademicGraph.vue`
  - `frontend/src/components/account/PersonalCenterQuickLinks.vue`
  - `frontend/scripts/verify-third-round-ui.mjs`
- 处理：
  - 继续隐藏教学成果录入页签
  - 删除成果审核页中的教学成果审核页签
  - 图谱空状态、快捷入口和验证文案不再出现教学成果

## 6. 当前结论

- 教学成果已从后端运行时与数据库迁移状态中进入删除阶段
- 画像、学院看板、图谱、推荐、问答、初始化脚本均不再依赖教学成果
- 前端当前版本不再开放教学成果入口，系统主路径已与“科研能力画像”题目保持一致

## 7. 后续操作建议

- 执行数据库迁移：

```bash
cd backend
python manage.py migrate
```

- 若生产或演示库中已有教学成果历史数据，执行迁移前建议先备份数据库
- 迁移完成后，教学成果表将从数据库层正式删除
