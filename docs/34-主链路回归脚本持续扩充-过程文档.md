# 34-主链路回归脚本持续扩充-过程文档

## 1. 本轮目标

继续增强统一主链路回归脚本，让当前系统最关键的业务闭环更容易重复执行、记录结果和指导他人回归。

本轮不引入新的测试框架，也不另起一套验证体系，继续沿用：

- `backend/scripts/verify_development_16_24_regression.py`
- `backend/scripts/run_validation_pipeline.py`
- `frontend/npm run verify:baseline`

## 2. 扫描结论

本轮重点扫描了：

- `backend/scripts/verify_development_16_24_regression.py`
- `backend/scripts/run_validation_pipeline.py`
- `README.md`
- `docs/最终答辩演示步骤-第三轮.md`
- `docs/最终验收清单-第三轮.md`

扫描后确认当前问题：

1. 统一主链路回归脚本仍以“状态码存活检查”为主。
2. 对教师资料编辑、成果新增 / 编辑 / 查询、管理员重置密码等关键闭环缺少字段级验证。
3. 登录失效回跳这类前端链路虽然已有 helper 校验，但文档里没有明确说明它和后端回归脚本的分工。

## 3. 本轮改动

### 3.1 扩充统一主链路回归脚本

继续复用：

- `backend/scripts/verify_development_16_24_regression.py`

新增和增强的验证点包括：

- 教师登录返回 token 结构
- 教师当前用户接口返回 `permission_scope`
- 教师资料编辑链路
- 论文新增 / 编辑 / 搜索 / 统计摘要链路
- 教师画像首页和雷达接口关键字段
- 图谱接口返回结构与教师跨账号访问限制
- 推荐接口返回教师快照和推荐结果
- 问答接口返回来源说明
- 未登录访问当前用户接口返回 `401`
- 管理员教师列表、教师详情、密码重置
- 管理员查看指定教师画像 / 图谱 / 推荐 / 问答 / 学院看板
- 管理员推荐对比和管理员分析字段

### 3.2 保持与最小 CI 兼容

本轮没有修改统一流水线入口路径：

- `backend/scripts/run_validation_pipeline.py`

因此当前最小 CI 仍可直接复用扩充后的主链路回归脚本。

## 6. 最小人工验证步骤

1. 进入 `backend`
2. 执行 `python scripts\restore_demo_state.py`
3. 执行 `python scripts\verify_development_16_24_regression.py`
4. 再打开浏览器人工验证：
   - 清空 `sessionStorage`
   - 访问受保护页面
   - 确认自动跳回登录页并保留回跳目标

## 7. 当前边界

- 当前统一主链路回归脚本仍以接口级和字段级验证为主，不是完整 E2E 浏览器自动化
- 当前不会一次性覆盖所有页面和所有分支
- 当前保持“高频、高风险、可演示”的核心闭环优先
