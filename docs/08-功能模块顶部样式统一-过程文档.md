# 功能模块顶部样式统一与教师管理修正-过程文档

## 任务背景

当前系统在学院看板、编辑基础档案两个页面上已经形成了较稳定的顶部视觉风格：居中布局、渐变色 hero 区、标题与说明文案分层明确、右侧操作按钮集中展示。  
但其他功能页面仍存在各自为政的情况，尤其是：

- 教师成果录入中心
- 项目指南管理
- 项目指南推荐

这些页面虽然功能可用，但顶部区域的视觉风格、按钮布局、内容宽度和整体重心与学院看板、基础档案页不一致，影响系统的一体化观感。

本轮任务目标是：

- 不改动现有业务接口与主链路
- 仅统一各功能页面顶部视觉与壳层布局
- 让管理员与教师进入不同小模块后，顶部都保持同一套系统风格

## 现状扫描

本轮实现前，页面情况如下：

- `AcademyDashboardView.vue`：顶部视觉成熟，可作为管理端页头基准
- `TeacherProfileEditorView.vue`：顶部视觉成熟，可作为教师个人功能页头基准
- `TeacherManagementView.vue`：已较接近目标风格，可继续沿用
- `AchievementEntryView.vue`：顶部虽已有 hero 区，但卡片阴影、背景、操作按钮层级与基准页不完全一致
- `ProjectGuideManagementView.vue`：顶部基础结构已接近，但缺少统一的回到画像主页入口，内容卡片样式偏轻
- `ProjectRecommendationView.vue`：仍使用 `el-card` 作为顶部容器，与其他核心功能页面差异明显

## 设计取舍

本轮没有抽一个新的全局 Hero 组件，原因是：

- 当前页面数量还不算多
- 各页功能差异较大
- 一期阶段优先做低风险一致化，不做额外组件抽象

因此本次采用的方案是：

- 统一使用“居中渐变 hero + 右侧操作按钮 + 下方内容壳层”的页面头部结构
- 统一内容最大宽度为 `1180px`
- 统一内容区卡片圆角、阴影和背景质感
- 统一提供“返回画像主页 / 刷新当前页数据”这类高频操作入口

## 实施过程

### 1. 教师成果录入中心

对 `AchievementEntryView.vue` 做了以下调整：

- 保持原有成果录入和多标签页逻辑不变
- 顶部新增“返回画像主页”按钮
- 将内容区卡片改为与学院看板一致的圆角与阴影风格
- 调整标签页头部间距和强调程度，让上部视觉与系统其他模块统一

这样做的原因是：

- 成果录入页是教师最常进入的工作页之一
- 它需要在不影响录入效率的前提下，与画像主页形成统一体验

### 2. 项目指南管理

对 `ProjectGuideManagementView.vue` 做了以下调整：

- 顶部新增“返回画像主页”按钮
- 保留管理员维护指南的原有表单与列表结构
- 将下方内容卡片质感统一到学院看板/基础档案页的视觉层级

这样做的原因是：

- 当前页是管理员扩展功能页
- 统一顶部后，管理员在不同管理模块之间切换时不再有明显割裂感

### 3. 项目指南推荐

对 `ProjectRecommendationView.vue` 做了较大但低风险的结构整理：

- 将原 `el-card` 顶部改为统一 hero 区
- 顶部新增“返回画像主页”按钮与“刷新推荐”按钮
- 为推荐摘要区、推荐列表区增加统一的 `content-shell`
- 对空状态做了居中壳层包裹，避免页面视觉重心漂移
- 将推荐列表卡片风格统一到系统主页面设计

这样做的原因是：

- 推荐页是新增扩展能力，最容易在视觉上脱离主系统
- 本轮需要把它收回到“系统内生页面”的统一体验里

## 二次修正：教师管理页

在首轮统一后，教师管理页顶部虽然已经独立，但视觉上仍没有完全贴近学院看板与基础档案页。  
因此又追加做了一轮结构修正，重点是：

- 将顶部进一步调整为更接近学院看板的 `hero-shell + hero-main` 结构
- 强化管理员页面语义，增加页头标签
- 明确“返回画像主页 / 刷新教师数据”按钮层级

这样做的原因是：

- 教师管理页属于管理员高频功能页
- 如果这页顶部仍偏旧样式，会破坏整个系统的统一感

## 风险控制

本次只改前端展示层，没有改动：

- 路由守卫
- 登录态
- 接口调用地址
- 数据结构
- 表单提交逻辑

因此不会影响现有：

- 教师成果录入
- 项目指南管理
- 项目指南推荐
- 学院看板
- 教师基础档案维护

## 验证方式

执行：

```bash
npm run type-check
```

手工验证重点：

1. 进入教师成果录入中心，确认顶部为统一 hero 风格，并可返回画像主页
2. 进入项目指南管理页，确认顶部与学院看板风格一致
3. 进入项目推荐页，确认顶部不再是独立卡片样式，而是统一 hero 风格
4. 在移动宽度下确认按钮和标题能正常换行，不出现遮挡

## 结果说明

本轮完成后，当前系统主要功能模块的顶部设计已基本收齐到统一视觉语言：

- 学院看板
- 编辑基础档案
- 教师管理
- 成果录入中心
- 项目指南管理
- 项目指南推荐

后续如果继续新增独立功能页，建议直接复用这一轮形成的页面头部模式，避免再次出现各页顶部风格分裂的问题。
