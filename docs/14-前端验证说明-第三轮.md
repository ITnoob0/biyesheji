# 前端验证说明-第三轮

最后更新时间：2026-03-18

## 一、目标

第三轮前端验证不追求一次性引入完整测试框架，而是优先建立“可执行、可复现、能覆盖关键展示路径”的最小验证基线。

## 二、实际命令

### 1. 类型检查

```bash
cmd /c npm run type-check
```

### 2. 准自动化验证脚本

```bash
cmd /c npm run verify:ui
```

## 三、验证脚本覆盖内容

当前脚本位于：

- [frontend/scripts/verify-third-round-ui.mjs](e:/Project/bishe/frontend/scripts/verify-third-round-ui.mjs)

覆盖以下行为：

1. 登录成功后回跳目标选择
2. 成果编辑 / 新增 / 删除后的列表更新
3. 图谱回退来源说明构建
4. 推荐列表筛选与排序

## 四、人工补充验证建议

### 推荐页

1. 教师进入推荐页
2. 观察推荐说明、策略说明和排序说明是否可见
3. 使用搜索、推荐类型筛选和排序切换
4. 点击“智能解读”进入问答页

### 学院看板页

1. 管理员进入学院看板
2. 依次切换学院、年份、教师筛选
3. 点击排行中的画像和推荐按钮
4. 点击“导出当前排行”检查是否下载 CSV

### 智能问答页

1. 进入问答页
2. 分别选择画像总结、成果概括、推荐原因说明
3. 检查结果是否包含：
   - 数据来源
   - 当前边界
   - 非覆盖能力

### 图谱页

1. 进入画像页中的学术拓扑图
2. 检查“数据来源”区块
3. 检查“图分析亮点”和“合作与主题热点”区块
4. 在 Neo4j 不可用时确认页面仍可展示 MySQL 回退说明

## 五、说明

- 当前前端验证已经具备最小可运行脚本。
- 但这并不等于完整前端测试框架已经建立。
- 当前口径应表述为“最小准自动化验证基线已建立”。
