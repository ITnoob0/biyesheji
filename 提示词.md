# Role: 高级全栈工程师（精通 Django, Neo4j, Vue3, ECharts）

# Context (项目背景)
我们正在开发一个“高校教师科研画像与智能辅助系统”。
当前技术栈：Python 3.11, Django 4.x, Django REST Framework, MySQL 8.0, Neo4j, Vue 3 (Composition API, TypeScript), Element-Plus, Apache ECharts。
我们的目标是开发“学术社交拓扑图”模块。目前的 `achievements` 模块中已经有 `Paper`, `CoAuthor`, `ResearchKeyword` 等 MySQL 模型，现在需要引入 Neo4j 来存储和查询复杂的网络关系，并在前端进行可视化展示。

# Tasks (任务清单)
请按照以下步骤逐步完成代码编写。如果任务过长，你可以分步骤向我确认后再继续。

## 步骤 1：完善 MySQL 数据模型
修改 `backend/achievements/models.py`：
为 `CoAuthor` 模型添加一个外键字段 `internal_teacher`，指向自定义用户模型 `settings.AUTH_USER_MODEL`。允许为空（null=True, blank=True）。这个字段用于将论文中提取的合作者字符串与系统中真实的校内教师账号关联起来。

## 步骤 2：Neo4j 数据同步脚本 (核心引擎)
在 `backend/graph_engine` App 下：
1. 请确保系统中安装官方 `neo4j` Python 驱动包。
2. 编写一个 Django Management Command (例如 `backend/graph_engine/management/commands/sync_neo4j.py`)。
3. 该脚本需要读取 MySQL 中的 `TeacherProfile`, `Paper`, `CoAuthor`, `PaperKeyword` 数据，使用 Cypher 语句将它们同步到 Neo4j 图数据库中。
4. **Neo4j 图谱 Schema 定义要求：**
   - 节点 (Nodes)：`Teacher` (属性: user_id, name), `ExternalScholar` (属性: name), `Paper` (属性: paper_id, title), `Keyword` (属性: name)。
   - 关系 (Edges)：
     - `(Teacher)-[:PUBLISHED {is_first: true/false}]->(Paper)`
     - `(ExternalScholar)-[:CO_AUTHORED]->(Paper)`
     - `(Paper)-[:HAS_KEYWORD]->(Keyword)`

## 步骤 3：开发图谱查询 API (DRF)
在 `backend/graph_engine/views.py` 和 `urls.py` 中：
1. 编写一个 API 接口 (例如 `/api/graph/topology/<int:user_id>/`)，接收一个教师的 User ID。
2. 使用 Cypher 语句查询该教师为中心的 2 度社交网络（合作过的学者、共同关注的关键词等）。
3. **关键约束：** 返回的数据格式必须严格符合 ECharts Graph 系列的要求。
   - 响应格式示例：`{"nodes": [{"id": "1", "name": "张三", "category": 0, "symbolSize": 50}], "links": [{"source": "1", "target": "2", "name": "合作"}]}`

## 步骤 4：前端 Vue3 + ECharts 组件
在 `frontend/src/views/` (或 components) 下创建 `AcademicGraph.vue`：
1. 使用 Vue 3 `<script setup lang="ts">`。
2. 引入 `echarts`。
3. 调用步骤 3 编写的接口获取图谱数据。
4. 渲染一个关系图谱 (series type: 'graph')。
5. **样式要求：** 开启 `force` 力引导布局，节点支持拖拽，不同类型的节点（如中心教师、校内合作者、校外学者、关键词）需要用不同的颜色和 `symbolSize` 区分，并在 `tooltip` 中显示详细信息。

# Constraints (约束)
- 严禁修改现有的 JWT 认证逻辑和跨域配置。
- 所有的 Python 代码必须包含清晰的类型提示 (Type Hints) 和中文注释。
- Neo4j 的连接配置请读取 Django `settings.py` 中的环境变量（给出配置示例）。
- 前端使用 Axios 发送请求，注意携带 Bearer Token。
```

# Role: Expert Full-Stack Developer (Django, Neo4j, Vue3, ECharts)

# Context
We are developing a "University Teacher Research Profile & Intelligent Assistant System".
Tech Stack: Python 3.11, Django 4.x, Django REST Framework, MySQL 8.0, Neo4j, Vue 3 (Composition API, TypeScript), Element-Plus, Apache ECharts.
The current goal is to build the "Academic Social Network Topology" module. We already have `Paper`, `CoAuthor`, and `ResearchKeyword` models in MySQL. We need to introduce Neo4j to store and query complex graph relationships and visualize them on the frontend.

# Tasks
Please implement the following steps sequentially. Feel free to ask for confirmation between steps if the output is too long.

## Step 1: Refine MySQL Models
Update `backend/achievements/models.py`:
Add a new foreign key field `internal_teacher` to the `CoAuthor` model. It should point to `settings.AUTH_USER_MODEL`, allowing null and blank values (`null=True, blank=True`). This links raw co-author strings to actual internal user accounts.

## Step 2: Neo4j Data Sync Engine
Inside the `backend/graph_engine` app:
1. Ensure the official `neo4j` Python driver is used.
2. Create a Django Management Command (e.g., `backend/graph_engine/management/commands/sync_neo4j.py`).
3. This script should read data from MySQL (`TeacherProfile`, `Paper`, `CoAuthor`, `PaperKeyword`) and sync it to Neo4j using Cypher queries.
4. **Neo4j Schema Requirements:**
   - Nodes: `Teacher` (props: user_id, name), `ExternalScholar` (props: name), `Paper` (props: paper_id, title), `Keyword` (props: name).
   - Edges:
     - `(Teacher)-[:PUBLISHED {is_first: true/false}]->(Paper)`
     - `(ExternalScholar)-[:CO_AUTHORED]->(Paper)`
     - `(Paper)-[:HAS_KEYWORD]->(Keyword)`
   - *Use MERGE statements to avoid duplicate nodes/edges during repeated syncs.*

## Step 3: Graph Query API (DRF)
In `backend/graph_engine/views.py` and `urls.py`:
1. Create a REST endpoint (e.g., `/api/graph/topology/<int:user_id>/`) that accepts a teacher's user ID.
2. Write a Cypher query to fetch the 2-hop graph network centered around this teacher (including co-authors, papers, and keywords).
3. **Crucial Constraint:** The API response MUST strictly match the format required by Apache ECharts Graph series.
   - Example shape: `{"nodes": [{"id": "1", "name": "Dr. Smith", "category": 0, "symbolSize": 50}], "links": [{"source": "1", "target": "2", "name": "co-author"}]}`

## Step 4: Frontend Vue3 + ECharts Component
Create a new component `AcademicGraph.vue` in `frontend/src/views/` (or components):
1. Use Vue 3 `<script setup lang="ts">`.
2. Import `echarts` and initialize it on an `onMounted` hook.
3. Fetch data from the API created in Step 3.
4. Render an ECharts graph (`series type: 'graph'`).
5. **Styling Requirements:** Use the `force` layout. Make nodes draggable. Differentiate node types (Center Teacher, Internal Co-author, External Scholar, Keyword) using distinct colors and `symbolSize`. Provide a rich `tooltip` for nodes and edges.

# Constraints
- Do NOT modify existing JWT authentication or CORS setups.
- All Python code must include Type Hints and standard docstrings.
- Provide a clear example of how to configure Neo4j credentials in Django `settings.py`.
- On the frontend, ensure Axios requests include the Authorization Bearer token.
```
