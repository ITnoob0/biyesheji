# 认证与账户体系增强-过程文档

## 本次任务目标

在现有“工号登录、教师注册、忘记密码、统一失效跳转”的基础上，继续做最小必要的认证与账户体系增强，重点补齐：

- 登录后主动修改密码；
- 管理员重置教师账户后的安全提示与状态标记；
- 教师与管理员角色身份展示；
- 权限不足、异常登录、会话失效后的统一提示；
- 关键权限场景测试；
- 当前权限模型说明与后续多角色扩展预留说明。

## 仓库现状梳理

### 真实技术栈

- 前端：Vue 3 + TypeScript + Vite + Element Plus + Axios
- 前端验证脚本：`npm run type-check`、`npm run verify:ui`
- 后端：Django + Django REST Framework + SimpleJWT
- 后端入口：`backend/manage.py`
- 后端用户模块：`backend/users`

### 既有账户链路

本次修改前，系统已经具备这些能力：

- 登录：前端走 `/api/token/`，登录态保存在 `sessionStorage`
- 当前用户信息：`/api/users/me/`
- 教师自助注册：`/api/users/register/`
- 忘记密码：`/api/users/forgot-password/`
- 管理员教师管理：`/api/users/teachers/`、`/api/users/teachers/<id>/`
- 管理员重置教师密码：`/api/users/teachers/<id>/reset-password/`
- 路由守卫：前端 `router.beforeEach` 已拦截未登录和管理员页访问

### 发现的问题

- 已有“忘记密码”和“管理员重置密码”，但没有“登录后主动修改密码”能力。
- 管理员重置密码后，前后端都没有持续标记“当前密码是临时密码，需尽快修改”。
- `/api/token/` 仍使用默认登录视图，无法对“账户停用”给出更明确的提示。
- 管理员页和普通教师页虽然有 `is_admin` 判断，但缺少统一角色文案和统一安全提示。
- 权限不足时的前端反馈部分依赖静默跳转，部分依赖各页面自己写提示，口径不够统一。
- 教师管理列表只排除了 `is_superuser=False`，如果后续存在 staff 型管理员，理论上会被混入教师列表。

## 本次完成内容

### 1. 后端账户生命周期增强

- 为 `CustomUser` 增加两个字段：
  - `password_reset_required`
  - `password_updated_at`
- 新增统一密码写入服务 `set_user_password(...)`，收口以下流程：
  - 管理员初始化教师密码
  - 管理员重置教师密码
  - 教师忘记密码
  - 教师登录后主动修改密码
- 用户序列化统一返回：
  - `role_code`
  - `role_label`
  - `password_reset_required`
  - `password_updated_at`
  - `security_notice`

### 2. 统一权限入口与角色边界

- 新增 `backend/users/access.py`，统一收口：
  - 管理员身份判断
  - “仅本人或管理员”访问判断
  - “仅可管理教师账户，不可通过教师管理入口操作管理员账户”判断
- 管理员教师列表现在统一排除管理员账户，只返回教师账号。
- 管理员重置教师密码时，明确拒绝管理员账号被当成教师账号操作。

### 3. 密码修改与安全提示

- 新增接口：`POST /api/users/me/change-password/`
- 接口要求：
  - 校验当前密码
  - 校验新旧密码不能相同
  - 校验两次新密码一致
  - 继续沿用 Django 密码强度校验
- 修改成功后会自动清除 `password_reset_required` 标记，并更新时间。
- 管理员重置教师密码后：
  - 密码重置为默认密码
  - 自动设置 `password_reset_required = true`
  - 前后端统一显示“当前密码为管理员初始化或重置后的临时密码，请登录后尽快修改”

### 4. 异常登录、失效恢复、权限不足提示统一

- 自定义 SimpleJWT 登录视图，对停用账户返回明确提示：
  - “账户已停用，请联系管理员处理。”
- 前端新增统一认证展示工具：
  - `resolveRoleLabel`
  - `buildPasswordSecurityNotice`
  - `resolveApiErrorMessage`
  - `resolveLoginFailureMessage`
- 路由守卫对权限不足跳转增加统一一次性提示：
  - 教师访问管理员页
  - 教师尝试查看其他教师画像
- 登录页增加“失效恢复”说明：
  - 如果有 `redirect`，提示登录后会回到刚才页面

### 5. 前端界面调整

- 登录页：
  - 保留工号登录主流程
  - 增加临时密码安全提示
  - 增加会话失效后回跳说明
- 忘记密码页：
  - 统一错误提示
  - 增加安全说明
- 基础档案页：
  - 保留原档案编辑能力
  - 新增独立的“账户安全”面板
  - 支持登录后主动修改密码
  - 明确显示角色身份、密码更新时间、临时密码状态
- 教师管理页：
  - 列表增加“身份”“账号状态”“密码状态”“安全提示”列
  - 管理员重置密码后弹窗展示临时密码与安全提示
  - 保持现有工号创建与教师资料编辑主链路不变
- 全局头部：
  - 用户身份展示改为统一角色文案，而不是页面各自推断

## 当前权限模型说明

### 当前角色模型

当前系统仍是轻量双角色模型，不做大规模 RBAC 重构：

- 教师账户：
  - `is_staff = False`
  - `is_superuser = False`
- 管理员账户：
  - `is_staff = True` 或 `is_superuser = True`

### 当前权限边界

- 教师可以：
  - 使用工号登录
  - 自助注册
  - 使用忘记密码找回
  - 查看和编辑自己的基础档案
  - 修改自己的登录密码
  - 查看自己的画像与业务页面
- 教师不可以：
  - 进入教师管理页
  - 管理其他教师账户
  - 查看其他教师画像
  - 通过教师管理入口操作管理员账户
- 管理员可以：
  - 创建教师账号
  - 编辑教师基础资料与启停状态
  - 重置教师密码
  - 访问管理员专属页面
- 管理员当前不通过“教师管理入口”处理管理员账号密码，避免角色边界混乱。

## 后续多角色扩展预留说明

本次没有引入复杂 RBAC，只保留了后续扩展所需的最小接口面：

- 后端统一通过 `role_code` / `role_label` 对外输出角色展示信息；
- 权限判断收口到 `users/access.py`，后续如果新增“学院管理员”“科研秘书”等角色，可在这里扩展；
- 前端角色展示和安全提示收口到共享工具函数，后续新增角色时不需要所有页面重复改文案；
- 当前仍以 Django 原生 `is_staff/is_superuser` 为管理员边界，不把“预留扩展”表述成“已完整上线多角色系统”。

## 最小人工验证建议

1. 管理员登录后进入教师管理页，确认列表只展示教师，不混入管理员。
2. 管理员重置某教师密码，确认弹窗显示临时密码与“尽快修改密码”提示。
3. 使用该教师账号登录，确认登录成功后有临时密码安全提醒。
4. 教师进入“基础档案”页，确认可在“账户安全”面板修改密码。
5. 教师修改密码成功后刷新页面，确认“待修改密码”状态消失。
6. 教师直接访问 `/teachers`，确认被守卫拦回且提示权限不足。
7. 停用一个教师账户后尝试登录，确认提示“账户已停用，请联系管理员处理”。
