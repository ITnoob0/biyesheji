# 69-成果审批流管理员可见性与全成果审批收口-过程文档

## 1. 背景与问题
- 管理员（系统管理员/学院管理员）在成果中心读取教师成果时存在可见性异常，出现“读不到教师成果信息”的情况。
- 学院看板部分图表和统计面板出现空数据，实际由查询口径与审批状态不一致触发。
- 需要确认并收口“审批流是否覆盖所有成果类型”，而不仅是论文。

## 2. 本次改动目标
- 修复管理员查询范围，确保两类管理员都能在各自权限范围内读取教师成果。
- 将教师成果首轮提交统一纳入审批流（创建即进入待审核）。
- 统一“图谱发布/正式展示”口径：仅已通过（`APPROVED`）成果进入正式统计、画像、全部成果和学院看板口径。
- 保持学院管理员学院级数据隔离。

## 3. 实施内容

### 3.1 后端审批与可见性收口
文件：`backend/achievements/views.py`

- 管理员读取范围修复（`TeacherOwnedAchievementViewSet.get_queryset`）
  - 教师：仅本人。
  - 管理员 + 指定 `teacher_id`：按 `can_access_teacher_scope` 做权限校验，通过后返回目标教师数据。
  - 管理员 + 未指定 `teacher_id`：
    - 系统管理员：全校教师（排除管理员账号）的成果范围。
    - 学院管理员：本学院教师（排除管理员账号）的成果范围。

- 创建与更新审批状态收口
  - 通用成果创建（`perform_create`）默认写入 `status='PENDING_REVIEW'`。
  - 通用成果更新（`perform_update`）后统一回到 `PENDING_REVIEW`。
  - 论文创建/更新（`PaperViewSet.perform_create/perform_update`）同步收口为 `PENDING_REVIEW`。
  - BibTeX 导入写入（`bibtex_confirm`）改为 `PENDING_REVIEW`。

- 图谱发布时机收口
  - 统一改为通过 `sync_graph_by_status` 控制：
    - `APPROVED` 才同步图谱。
    - 非 `APPROVED` 删除图谱快照。
  - 审批通过/驳回动作后均执行该策略。

- 待审核列表兼容旧数据
  - `pending-review` 接口临时兼容 `status in ('PENDING_REVIEW', 'DRAFT')`，避免历史草稿记录无法进入审批处理。

- 日志归属修复
  - `log_operation` 优先使用成果归属教师作为日志主体（而非审批管理员），保证审核时间轴按教师成果可追踪。

- 论文关键词链路兼容
  - 论文在创建/更新/BibTeX 导入时继续执行关键词提取（不依赖图谱发布），避免论文治理与关键词展示退化。

### 3.2 正式展示口径统一为 `APPROVED`
文件：`backend/achievements/views.py`、`backend/achievements/management_dashboard.py`

- `get_portrait_data_meta` 改为基于已通过成果计算更新时间。
- `build_all_achievements` 改为仅汇总已通过成果。
- 学院看板协作筛选（有无合作）改为基于已通过论文的合作者关系计算。

### 3.3 审批流覆盖性测试补强
文件：`backend/achievements/tests.py`

- 新增：
  - `test_teacher_create_all_achievement_types_defaults_to_pending_review`
    - 覆盖论文、项目、知识产权、教学成果、学术服务五类创建后状态均为 `PENDING_REVIEW`。
  - `test_admin_without_teacher_id_can_list_scoped_teacher_achievements`
    - 覆盖学院管理员与系统管理员未传 `teacher_id` 时的作用域查询行为。

- 既有用例口径更新：
  - 更新与图谱立即发布相关断言，改为待审核阶段“不发布图谱”。
  - 更新“全部成果”测试夹具为 `APPROVED` 口径数据。
  - 更新学院看板扩展过滤测试夹具，补全已通过状态。

## 4. 验证结果

- `npm run type-check`：通过

### 4.2 后端验证（通过）
- `python manage.py check`：通过
- `python manage.py test achievements.tests --noinput`：通过（42 tests）

- 管理员读取相关定向回归：

  - `test_admin_can_filter_academy_dashboard_by_department_and_year`：通过

### 4.3 系统级验证说明
- 尝试执行 `python manage.py test --keepdb --noinput` 与 `python manage.py test users` 时，当前本地 MySQL 测试库存在环境级不稳定因素：
  - 死锁（`1213 Deadlock found when trying to get lock`）
  - 测试库冲突与残留数据导致的唯一键冲突（如 `TeacherProfile.user_id` 重复）
- 该问题与本次审批流改动链路不直接耦合，已通过“成就模块全量 + 管理员定向回归 + 前端构建”完成本次改动正确性验证。

## 5. 结果结论
- 两类管理员已恢复可在权限范围内读取教师成果。
- 审批流已覆盖五类成果，不再仅限论文。
- 待审核成果不会进入正式展示与图谱发布，已通过成果才进入画像、全部成果、学院看板等正式口径。
