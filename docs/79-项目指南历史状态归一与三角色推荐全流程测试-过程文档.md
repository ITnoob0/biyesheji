# 79-项目指南历史状态归一与三角色推荐全流程测试-过程文档

## 1. 变更背景
- 当前 `project_guides` 已切换到新状态体系：`DRAFT / ACTIVE / URGENT / ARCHIVED`。
- 历史库内仍可能存在旧状态值（`OPEN / CLOSED`）以及旧版范围数据（学院管理员历史创建但仍为 `GLOBAL`），会导致管理与推荐口径不一致。
- 本次目标：在不改动业务架构的前提下，完成历史数据归一，并对项目推荐业务进行三类角色（教师 / 学院管理员 / 系统管理员）全流程验证。

## 2. 本次改动清单

### 2.1 数据迁移
- 新增迁移文件：
  - `backend/project_guides/migrations/0005_normalize_legacy_guide_status_and_scope.py`
- 迁移处理内容：
  - `ProjectGuide.status`：
    - `OPEN -> ACTIVE`
    - `CLOSED -> ARCHIVED`
  - `ProjectGuideRecommendationRecord.guide_status_snapshot` 同步映射：
    - `OPEN -> ACTIVE`
    - `CLOSED -> ARCHIVED`
  - 约束清洗：
    - `scope=GLOBAL` 的记录统一 `academy_id=NULL`
  - 历史学院管理员创建数据收口：
    - 若由学院管理员创建、且仍为 `GLOBAL` 且无学院绑定，则强制改为 `ACADEMY` 并按创建人 `department` 绑定学院
    - 若 `scope=ACADEMY` 但无学院绑定，也按创建人 `department` 自动回填

### 2.2 三角色推荐全流程测试
- 更新文件：
  - `backend/project_guides/tests.py`
- 新增测试类：
  - `ProjectRecommendationThreeRoleFlowTests`
- 覆盖场景：
  - 教师：
    - 查看本人推荐（只含全局+本院，排除外院与归档）
    - 收藏与反馈闭环
    - 禁止查看他人推荐
  - 学院管理员：
    - 可查看本院教师推荐并进行教师对比
    - 禁止查看外院教师推荐
  - 系统管理员：
    - 可查看任意教师推荐
    - 可执行跨学院教师对比
