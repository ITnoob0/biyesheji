# 82-项目推荐与成果认领消息通知闭环-过程文档

## 1. 目标
- 打通“项目推荐”业务闭环：管理员定向推送后，教师能在右上角消息中心看到具体推送内容并一键跳转推荐页。
- 打通“成果认领”业务闭环：认领邀请与学院催办提醒进入同一消息中心，点击可直接跳转成果认领页。
- 按需求移除教师个人中心中的认领未读徽标展示，统一收口到右上角消息通知入口。

## 2. 后端改动

### 2.1 用户通知模型
- 文件：`backend/users/models.py`
- 新增模型：`UserNotification`
  - 字段：`recipient`, `sender`, `category`, `title`, `content`, `action_path`, `action_query`, `payload`, `is_read`, `read_at`, `created_at`
  - 类型分类：
    - `PROJECT_GUIDE_PUSH`
    - `ACHIEVEMENT_CLAIM`
    - `CLAIM_REMINDER`

### 2.2 通知服务能力
- 文件：`backend/users/services.py`
- 新增方法：
  - `create_user_notification(...)`
  - `bulk_create_user_notifications(...)`

### 2.3 通知 API
- 文件：`backend/users/views.py`, `backend/users/urls.py`, `backend/users/serializers.py`
- 新增接口：
  - `GET /api/users/notifications/`：获取通知列表（支持 `unread_only`, `limit`）
  - `GET /api/users/notifications/unread-count/`：获取未读数
  - `POST /api/users/notifications/<id>/read/`：标记单条已读
  - `POST /api/users/notifications/read-all/`：全部标记已读

### 2.4 项目指南定向推送接入通知
- 文件：`backend/project_guides/views.py`
- `targeted_push` 执行后：
  - 按关键词匹配到教师时创建 `PROJECT_GUIDE_PUSH` 通知
  - 通知携带跳转路径：`/project-recommendations`
  - 跳转参数带 `guide_id`，支持定位推荐证据卡片
  - 返回字段新增：`notification_count`

### 2.5 成果认领接入通知
- 文件：`backend/achievements/services.py`, `backend/achievements/views.py`
- 在认领邀请生成/重开待认领时：
  - 创建 `ACHIEVEMENT_CLAIM` 通知
  - 跳转到 `/profile-editor/achievement-claims`
- 在学院管理员“长期未认领提醒”时：
  - 创建 `CLAIM_REMINDER` 通知

### 2.6 迁移
- 新增迁移：`backend/users/migrations/0006_usernotification.py`
- 已执行：`python manage.py migrate`

## 3. 前端改动

### 3.1 顶部消息中心（Bell）落地
- 文件：`frontend/src/layout/MainLayout.vue`
- 改造点：
  - 右上角“通知预留”改为真实消息下拉
  - 展示未读数、通知标题、正文、类型、发送人、时间
  - 支持单条已读、全部已读
  - 点击通知自动跳转到 `action_path + action_query`

### 3.2 推荐页通知跳转定位
- 文件：`frontend/src/views/ProjectRecommendationView.vue`
- 增强 `guide_id` 路由参数识别，支持消息点击后定位到对应指南卡片。

### 3.3 个人中心认领徽标下线
- 文件：`frontend/src/views/TeacherProfileEditorView.vue`
- 移除个人中心“待认领成果”未读徽标与计数请求，避免双入口重复展示。

### 3.4 类型补齐
- 文件：`frontend/src/types/users.ts`
  - 新增通知相关类型：`UserNotificationRecord`、`UserNotificationListResponse` 等
- 文件：`frontend/src/types/projectGuides.ts`
  - `GuideTargetedPushResponse` 增加 `notification_count`

## 5. 闭环结果
- 管理员定向推送 -> 教师右上角收到消息 -> 点击直达推荐页面对应指南。
- 成果认领邀请/提醒 -> 教师右上角收到消息 -> 点击直达成果认领页面处理。
- 消息入口统一到顶部 Bell，避免个人中心与消息中心重复认知负担，形成可执行闭环。
