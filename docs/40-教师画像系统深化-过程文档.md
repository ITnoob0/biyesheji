# 40-教师画像系统深化-过程文档

## 1. 任务背景

本轮目标是在不推翻现有教师画像主链路的前提下，把画像页从“当前状态展示”深化为“持续分析视图”。实现重点放在解释层、趋势层、对比层和报告化展示层，不盲目新增画像快照表。

## 3. 本次落地内容

### 3.1 画像解释能力增强

- 在评分引擎中补充了维度权重结构化说明。
- 新增权重卡片字段：维度名称、权重、简式公式、主要输入、形成原因、当前得分。
- 新增计算摘要字段：总分、最强维度、相对薄弱维度、固定权重模式说明。

### 3.2 阶段对比能力

- 新增“最近一年 vs 上一年”的阶段对比能力。
- 当前阶段对比以成果发生年份为锚点回溯估算，不宣称等同于历史冻结快照。
- 返回字段包括：
  - 当前阶段与基线阶段标签
  - 综合得分变化
  - 成果总量变化
  - 变化最明显的维度列表
  - 覆盖说明

### 3.3 快照边界最小方案

- 当前没有新增快照表。
- 当前画像快照方案定义为：`runtime_projection`
- 当前承接方式：
  - 后端实时生成 `snapshot_boundary`
  - 前端展示生成时间、锚点年份、当前边界说明、后续建议
  - 报告导出会包含快照边界说明

当前明确边界：

- `persistence_status = not_persisted`
- `freeze_status = not_frozen`
- 当前不是正式历史档案
- 当前不支持回看某一天的冻结画像

后续如需落库，建议字段：

- `generated_at`
- `teacher_id`
- `dimension_values`
- `total_score`
- `weight_spec_version`
- `data_scope_note`
- `source_range`

### 3.4 报告化展示与导出

- 新增画像报告接口：`/api/achievements/portrait-report/<user_id>/`
- 默认返回结构化 JSON 报告。
- 支持 `export=markdown` 导出 Markdown 报告文件。
- 报告当前覆盖：
  - 画像摘要
  - 权重说明
  - 阶段对比
  - 快照边界

### 3.5 前端页面深化

- 在现有画像页中直接增加深化分析面板，不新增独立画像系统。
- 新面板包括：
  - 阶段对比
  - 权重与形成逻辑
  - 快照边界
  - 报告化展示与导出

## 4. 重要设计决策与权衡

### 4.1 为什么当前不落库快照

当前画像评分仍以实时业务聚合为主，快照口径、冻结时机、生成触发机制和版本治理尚未完全定稿。如果此时直接新增快照表，容易形成“已有快照体系上线”的误导，也会增加维护成本。

因此本轮先完成：

- 快照口径定义
- 接口边界定义
- 前端展示承接
- 报告导出承接

### 4.2 为什么阶段对比先做年对年

当前数据天然具备成果年份字段，直接基于年份做最近阶段对比，成本最低、解释最清楚、回归风险最小。后续如果业务需要季度对比、学期对比或自定义区间对比，再在当前边界上扩展。

### 4.3 为什么报告先用 Markdown

Markdown 导出足够轻量，便于：

- 快速演示
- 复核文字说明
- 后续扩展为 PDF 或模板报表

当前不引入离线报表任务链路，避免过度建设。

## 5. 当前已落地部分与边界说明

### 已落地

- 权重与形成逻辑说明
- 阶段对比
- 快照边界说明
- 报告 JSON 返回
- Markdown 导出
- 页面结构化展示

### 仍在边界设计中

- 正式画像快照是否落库
- 快照何时生成
- 快照版本号如何管理
- 历史画像如何按时间点回看
- 报表是否需要升级为离线归档任务

### 关于后续是否需要落库

当前结论：需要保留落库设计边界，但本轮不强行落地。

建议触发条件：

- 学院侧需要固定周期留痕
- 需要考核口径回溯
- 需要对外输出正式年度画像档案

## 6. 历史遗留问题与本次新增问题区分

### 历史遗留

- 既有画像页更偏“当前状态展示”
- 趋势和解释能力已有基础，但缺少权重卡片、阶段对比和报告导出
- 画像快照尚无正式持久化方案

### 本次新增

- 增加结构化权重解释
- 增加阶段对比能力
- 增加快照边界接口与页面承接
- 增加画像报告接口与 Markdown 导出

## 8. 最小人工验证步骤

1. 使用教师账号登录系统，进入教师画像页。
2. 确认页面新增“阶段对比 / 权重与形成逻辑 / 快照边界 / 报告化展示”区域。
3. 检查阶段对比中是否展示“最近一年 vs 上一年”的综合得分变化和维度变化。
4. 检查权重卡片是否展示权重、公式、主要输入和当前得分。
5. 点击“导出 Markdown 报告”，确认浏览器成功下载 `.md` 文件。
6. 打开导出的 Markdown，确认其中包含画像摘要、维度权重、阶段对比和快照边界说明。
