# 71-知识产权与教学成果承担角色扩展及全成果设为审核通过-过程文档

## 1. 需求背景
- 教师端中“知识产权”“教学成果”希望与“科研项目”一致，支持填写承担角色（负责人/参与人）。
- 需要将当前数据库里全部教师的全部成果统一设为“审核通过”，用于后续人工验证。

## 2. 本次目标
- 为 `IntellectualProperty` 与 `TeachingAchievement` 增加 `role` 字段（`PI/CO_PI`）。
- 前后端同步支持角色录入、回显、列表展示。
- 保持现有审批流与权限架构不重构，仅做增量扩展。
- 将教师账号下五类成果状态统一更新为 `APPROVED` 并完成复核。

## 3. 实施内容

### 3.1 后端模型与迁移
文件：
- `backend/achievements/models.py`
- `backend/achievements/migrations/0010_intellectualproperty_role_teachingachievement_role.py`

变更：
- `IntellectualProperty` 新增：
  - `ROLE_TYPES = (PI, CO_PI)`
  - `role` 字段，默认 `PI`
- `TeachingAchievement` 新增：
  - `ROLE_TYPES = (PI, CO_PI)`
  - `role` 字段，默认 `PI`
- 生成并执行迁移 `0010`，数据库结构已落地。

### 3.2 后端序列化与成果聚合
文件：
- `backend/achievements/serializers.py`
- `backend/achievements/views.py`

变更：
- `IntellectualPropertySerializer` 增加 `role`、`role_display`。
- `TeachingAchievementSerializer` 增加 `role`、`role_display`。
- 操作日志快照补充两类成果的角色字段。
- 全部成果聚合（用于全部成果展示/筛选）中：
  - 知识产权与教学成果的 `detail` 增加角色显示。
  - `author_rank_category/author_rank_label` 改为按角色映射（负责人=lead，参与人=participating）。

### 3.3 前端录入与列表展示
文件：
- `frontend/src/types/achievements.ts`
- `frontend/src/views/AchievementEntryView.vue`

变更：
- 类型定义补充：
  - `IpRecord` / `TeachingRecord` 增加 `role`、`role_display`
  - `IpFormState` / `TeachingFormState` 增加 `role`
  - `IpMutationPayload` / `TeachingMutationPayload` 增加 `role`
- 录入表单补充“承担角色”下拉（复用项目角色选项：负责人、参与人）。
- 知识产权与教学成果列表新增“角色”列。
- 新增表单默认值、校验规则、重置逻辑、编辑回填逻辑，保证录入/编辑链路一致。

### 3.4 数据库状态批量调整为审核通过
执行：
- 面向教师账号（`is_staff=False` 且 `is_superuser=False`）批量更新五类成果状态为 `APPROVED`。

执行结果：
- `papers`: 13
- `projects`: 3
- `intellectual_properties`: 4
- `teaching_achievements`: 5
- `academic_services`: 5

复核结果（非 `APPROVED` 计数）：

- `projects_non_approved`: 0
- `ips_non_approved`: 0
- `teaching_non_approved`: 0
- `services_non_approved`: 0
