# 智能问答与知识辅助增强过程文档

## 1. 本次任务目标

本次工作在现有“基于真实系统数据的轻量问答演示链路”基础上做增量增强，目标是把问答能力升级为：

- 受控
- 可解释
- 可回退
- 可持续扩展

同时明确保持以下边界：

- 不包装成通用知识平台。
- 不引入重型知识库工程。
- 不引入大量外部语料。
- 问答继续只基于当前系统真实数据。

## 2. 扫描范围与现状梳理

本次优先扫描了以下模块：

- 后端：
  - `backend/ai_assistant/views.py`
  - `backend/ai_assistant/services.py`
  - `backend/ai_assistant/serializers.py`
  - `backend/ai_assistant/tests.py`
  - `backend/ai_assistant/urls.py`
- 前端：
  - `frontend/src/views/AssistantDemoView.vue`
  - `frontend/src/types/assistant.ts`

扫描结果确认：

- 当前问答链路已经是模板化服务，不是开放式生成。
- 当前主要支持三类问答：
  - 画像总结
  - 成果概括
  - 推荐原因说明
- 当前问答结果已有基础说明，但来源展示、边界提示和失败回退还不够清晰。
- 当前问答页面更像单点演示，还没有形成“受控模板集合”的产品感。

## 3. 本次增量设计

### 3.1 模板集合增强

在现有模板基础上，扩展为以下受控模板集合：

- `portrait_summary`
  - 教师科研画像总结
- `portrait_dimension_reason`
  - 教师画像如何形成
- `achievement_summary`
  - 教师近年成果结构概括
- `guide_overview`
  - 当前推荐概览
- `guide_reason`
  - 项目指南推荐原因说明
- `academy_summary`
  - 学院统计概览（仅管理员）

这些模板均来自现有真实系统数据，不依赖外部检索。

### 3.2 来源说明增强

回答结果新增：

- `data_sources`
  - 用于展示本次答案引用了哪些系统内数据类型
- `source_details`
  - 用于展示更细颗粒度的来源说明
  - 例如：
    - 教师资料
    - 成果总量
    - 画像优势维度
    - 推荐分数
    - 学院统计范围

这样可以让用户更清楚“答案是怎么来的”。

### 3.3 边界与透明性增强

回答结果统一补充：

- `scope_note`
- `non_coverage_note`
- `boundary_notes`
- `acceptance_scope`

目的不是把问答描述得“更像大模型”，而是让用户清楚：

- 当前只回答系统内已有数据
- 当前不能回答系统外开放知识
- 当前不能进行复杂自由推理
- 当前更适合做系统内说明与辅助

### 3.4 失败回退增强

后端问答视图增加了受控异常兜底：

- 当问答服务构造答案失败时，不再直接让页面只收到异常
- 而是返回结构化回退结果

前端也同步增加了页面级回退：

- 当接口异常或环境不完整时，页面会显示回退说明
- 不影响画像、成果、推荐、学院看板等主链路页面继续使用

本次回退能力强调“安全说明”，不是伪造回答。

## 4. 当前问答的数据来源

当前问答仅使用以下真实系统数据：

- 教师基础档案
- 教师画像评分结果与维度说明
- 论文、项目、知识产权、教学成果、学术服务等成果聚合结果
- 项目指南推荐结果与推荐理由
- 学院看板实时聚合统计（管理员）

当前没有接入：

- 外部知识库
- 互联网开放语料
- 文档向量检索
- 非系统内的实时百科知识

## 5. 当前不覆盖的能力

本次明确不覆盖以下能力：

- 通用开放问答
- 多轮长链路推理
- 外部政策全文问答
- 面向 PDF/Word/网页的大规模知识检索
- 自动构建的完整 RAG 问答体系
- 跨系统知识源联合查询

## 6. 如果未来进入真正 RAG，需要先明确的前置条件

如果后续要从当前模板化问答进入更完整的 RAG 路线，至少需要先明确：

### 6.1 知识来源

- 哪些数据允许进入知识索引
- 哪些数据只允许实时查库，不进入检索索引
- 是否包含项目指南原文、成果全文、制度文档、FAQ 文档等

### 6.2 索引策略

- 按教师、成果、指南、学院、制度文档分别建索引还是统一索引
- 索引粒度是整篇文档还是分段切片
- 标签和元数据如何绑定到检索单元

### 6.3 更新机制

- 新成果录入后何时更新索引
- 项目指南变更后如何重建索引
- 教师档案更新后如何处理缓存与召回一致性

在这些问题没有明确前，不适合把当前问答路线直接升级为“完整 RAG”。

## 7. 验证命令

本次实际执行的验证命令包括：

```bash
cmd /c npm run type-check
cmd /c npm run verify:ui
python -m py_compile backend\ai_assistant\views.py backend\ai_assistant\services.py backend\ai_assistant\serializers.py backend\ai_assistant\tests.py
python manage.py check
python manage.py test ai_assistant
```

## 8. 最小人工验证步骤

建议按以下步骤人工验证：

1. 登录系统并进入智能问答页面。
2. 依次切换以下模板：
   - 教师科研画像总结
   - 教师画像如何形成
   - 教师近年成果结构概括
   - 当前推荐概览
   - 项目指南推荐原因说明
3. 检查每类回答是否展示：
   - 数据来源
   - 来源细项
   - 使用边界
   - 当前验收说明
4. 管理员切换到“学院统计概览”，选择院系或年份后生成回答。
5. 在异常场景下检查是否会显示“回退说明”，并确认其他主链路页面不受影响。

- `rest_framework_simplejwt` 缺失

这属于本地运行环境问题，不属于本次问答模板增强本身的逻辑设计问题。
